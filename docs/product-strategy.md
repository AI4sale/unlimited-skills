# Product Strategy

Unlimited Skills is currently a local-first public alpha. The next product
question is whether users can discover it, install it, reach first value, and
tell us what blocked them.

## Strategy After v0.5.1-alpha

1. Keep the public core local-first and useful without registration.
2. Prove the PyPI install path and Claude Code MCP setup from the package.
3. Measure adoption with manual public signals and voluntary GitHub feedback.
4. Route feedback into visible backlog decisions within 24-48 hours.
5. Track marketplace/listing submissions only with owner-approved evidence.
6. Defer new hosted/team/trust implementation until demand is real.

## What We Are Not Doing Now

- no paid product claim;
- no payment link;
- no sales promise;
- no hosted readiness claim;
- no team readiness claim;
- no Enterprise readiness claim;
- no tracking pixels;
- no analytics SDK;
- no user-level behavior collection.

## Trust-Layer Decision Rule

No new E28+ hosted/team/trust implementation unless:

- real user feedback demands it;
- a team or customer asks for it;
- adoption data shows trust is blocking use;
- the owner explicitly reopens the Enterprise track.

## A3.4 Decision Rule

A3.4 actual submission evidence is blocked pending owner approval. The owner
must approve exact destinations, exact submission owner, exact listing copy,
and whether Codex may submit externally or only prepare evidence.

## #119 Decision Rule

PR #119 / E19 remains background unless the adoption queue is clean or the
owner explicitly reopens it.
