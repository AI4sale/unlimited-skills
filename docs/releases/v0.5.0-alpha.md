# v0.5.0-alpha

Status: public-alpha packaging gate; PyPI publication is blocked until Trusted
Publishing and post-publish smoke pass.

This release prepares Unlimited Skills for a package-first public alpha. The
local-first MIT core remains free and offline. There is nothing for sale in this
release.

## What Changed

- Package version is `0.5.0`.
- Wheel builds now include the bundled ECC and Superpowers packs under
  `unlimited_skills/bundled_packs/`.
- `unlimited-skills quickstart` imports bundled packs from package resources
  when the command runs from an installed wheel and no repository checkout is
  available.
- Package metadata now includes Homepage, Repository, Issues, Changelog, and
  Documentation project URLs.
- The deprecated MIT Trove license classifier was removed while keeping the
  SPDX-style `license = "MIT"` field.
- `scripts/run-v050-alpha-package-smoke.py` builds sdist/wheel, runs
  `twine check`, installs the wheel into a clean virtual environment, and
  verifies `unlimited-skills --version`, `quickstart`, `suggest`, and
  `mcp savings` from the installed package.

## Boundaries

- The repository docs have been flipped to the PyPI install command, but the
  release tag remains blocked until the package is actually uploaded and
  verified from PyPI.
- No release tag is pushed until the v0.5 verifier passes in
  `--package-availability published` mode on the selected release commit.
- No paid, hosted, Team, Pro, Business, or Enterprise capability is described
  as release-ready.
- Hosted registry, team sync, policy sync, catalog submissions, and billing
  diagnostics remain gated alpha surfaces, not products for sale.
- No telemetry is added.

## Required Verification

```bash
python -m pip install build twine pytest httpx
python scripts/run-v050-alpha-package-smoke.py --json
python scripts/verify-skill-effectiveness-gate.py --gate v0.5-release
python scripts/verify-v050-alpha-publication.py --expected-sha "$(git rev-parse HEAD)" --package-availability prepublish
```

Run the final verifier with `--package-availability published` only after the
package is available from PyPI. Before upload, that mode must fail with
`release_blocked_pypi_unavailable`; this is expected and keeps the tag blocked.
