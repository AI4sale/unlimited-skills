# v0.3.5-alpha Upgrade Notes

Upgrade from a GitHub clone:

```bash
git fetch --tags origin
git checkout <v0.3.5-alpha-tag-or-release-branch>
python -m pip install -e ".[all]"
unlimited-skills --version
unlimited-skills reindex
unlimited-skills doctor --json
unlimited-skills community list --channel canary --json
```

PyPI is not the supported `v0.3.5-alpha` distribution path. GitHub clone remains the official alpha distribution path because router skills, installer scripts, schemas, release docs, and bundled repo assets must be available beside the checkout.

## From v0.3.4-alpha

- Existing local libraries remain local and do not require registration.
- Existing registered installation state remains the hosted-service boundary for community operations.
- `community submit <path> --dry-run` remains local-only and uploads nothing.
- Hosted community list, search, preview, install, submission status, withdraw, and review-notes require registration.
- Community preview and install reject unsigned hosted responses and reject items that are not `approved` or `published`.
- Review-gated canary publication is available as the community catalog alpha path; stable promotion remains explicit and is not automatic.
- Full catalog distribution remains disabled.
- No marketplace storefront, billing, checkout, revenue share, card collection, bank collection, or live payment provider behavior is included in this release.
