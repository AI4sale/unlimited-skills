# v0.6 Local ROI Receipt Spec

Status: development-ready specification only. This PR does not implement the
runtime command, publish a package, submit to a marketplace, add analytics, or
touch #119/E19.

The local ROI receipt is a proposed v0.6 public-alpha command surface for a
small, screenshot-friendly proof of value generated on the user's own machine.
It turns local aggregate adoption signals into a shareable summary without
telemetry, upload, private data collection, sales claims, or hosted/team
promises.

Required claim language:

> This receipt is a local estimate from your own machine. It is not telemetry,
> not a benchmark guarantee, and not a paid ROI promise.

## Proposed Command Surface

The command is not implemented yet. The frozen spec for future implementation
is:

```bash
unlimited-skills roi receipt
unlimited-skills roi receipt --format markdown
unlimited-skills roi receipt --format json
unlimited-skills roi receipt --since 7d
unlimited-skills roi receipt --out roi-receipt.md
```

Default output should be a human-readable receipt suitable for a screenshot.
`--format markdown` writes the same receipt as Markdown. `--format json` emits
the machine-readable schema described below. `--since` limits aggregation to a
local time window when event data supports it. `--out` writes the selected
format to a local file instead of stdout.

## Screenshot-Friendly Receipt

The receipt should fit in a compact terminal or Markdown block:

```text
Unlimited Skills local ROI receipt
Generated: 2026-06-14T00:00:00Z
Window: last 7 days

Version: 0.6.x
Library: 267 indexed skills
Quickstart: completed
MCP context savings: measured locally, 98.6% lab fallback or user-configured summary
Skill routing: 42 suggests, 18 views, 9 uses
Suggest to use: aggregate conversion only
Feedback report: paste-safe report available

This receipt is a local estimate from your own machine.
It is not telemetry, not a benchmark guarantee, and not a paid ROI promise.
```

Exact wording may change before runtime implementation, but the safety
boundary and allowed/forbidden field classes below are the contract.

## Allowed Fields

Allowed receipt fields are aggregate, local-safe, or derived:

| Field | Class | Notes |
| --- | --- | --- |
| `schema_version` | safe aggregate | Starts at `1` for JSON output. |
| `report_type` | safe aggregate | Expected value: `local_roi_receipt`. |
| `generated_at` | safe aggregate | Timestamp generated locally. |
| `window` | safe aggregate | Requested window, for example `all` or `7d`. |
| `unlimited_skills_version` | safe aggregate | Installed CLI/package version. |
| `library_skill_count` | safe aggregate | Local indexed skill count only. |
| `quickstart_status` | derived | Completed/skipped/unknown/error summary only. |
| `mcp_savings_summary` | derived | Local `mcp savings` summary: bytes/tokens/percent/status only. |
| `suggest_count` | safe aggregate | Count of local suggest events. |
| `skill_view_count` | safe aggregate | Count of local view events. |
| `skill_use_count` | safe aggregate | Count of local use events. |
| `suggest_view_use_funnel` | derived | Aggregate conversion counts/rates only. |
| `learning_summary_events` | derived | Aggregate metrics from `learning-summary --events`. |
| `feedback_prepare_status` | derived | Whether paste-safe feedback report generation is available. |
| `privacy_notice` | safe aggregate | Required measured-not-promised notice. |

## Forbidden Fields

The receipt must never include:

- prompts;
- raw queries;
- raw tasks;
- tool inputs;
- tool outputs;
- skill bodies;
- MCP schemas;
- raw `events.jsonl`;
- raw `feedback.jsonl`;
- raw `.mcp.json` or `.claude.json`;
- environment names or values;
- tokens, keys, or proofs;
- local absolute paths;
- user identifiers;
- tracking identifiers.

## JSON Contract

`unlimited-skills roi receipt --format json` should emit valid JSON on success:

```json
{
  "schema_version": 1,
  "report_type": "local_roi_receipt",
  "generated_at": "2026-06-14T00:00:00Z",
  "window": {"requested": "7d", "effective": "7d"},
  "unlimited_skills_version": "0.6.x",
  "library": {"skill_count": 267},
  "quickstart": {"status": "completed"},
  "mcp_savings_summary": {
    "source": "local_mcp_savings",
    "measured_servers": 0,
    "savings_pct": 98.6,
    "status": "benchmark_fallback"
  },
  "skill_routing": {
    "suggest_count": 42,
    "view_count": 18,
    "use_count": 9,
    "suggest_to_view_rate": 0.429,
    "suggest_to_use_rate": 0.214
  },
  "learning_summary_events": {
    "available": true,
    "aggregate_only": true
  },
  "feedback_prepare": {"available": true},
  "privacy_notice": "This receipt is a local estimate from your own machine. It is not telemetry, not a benchmark guarantee, and not a paid ROI promise."
}
```

Optional fields may be added later when they follow the allowed-field classes.
The JSON contract must not add raw identifiers, raw task text, raw local logs,
or local absolute paths.

## Markdown Contract

`unlimited-skills roi receipt --format markdown` should render the same values
as a paste-safe Markdown report:

- a short title;
- generated timestamp and window;
- version and library count;
- quickstart status;
- MCP savings summary;
- skill routing aggregate counts;
- suggest-to-view/use aggregate conversion;
- feedback report availability;
- the required measured-not-promised notice.

Markdown output must stay screenshot-friendly and must not include raw local
logs, prompts, tool data, skill bodies, MCP schemas, env values, tokens, keys,
proofs, local absolute paths, user identifiers, or tracking identifiers.

## Legacy Pre-v0.5.3 Handling

The future implementation must use the post-v0.5.3 local event privacy
contract where possible. Legacy pre-v0.5.3 local logs are not automatically
rewritten and must be handled conservatively:

- do not read raw legacy fields into the receipt;
- prefer aggregate `learning-summary --events` values;
- if a safe aggregate cannot be computed without reading unsafe legacy content,
  mark that metric as `unavailable_legacy_logs`;
- do not ask users to share raw legacy files.
