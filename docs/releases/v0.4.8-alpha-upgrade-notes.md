# v0.4.8-alpha Upgrade Notes

`v0.4.8-alpha` is a GitHub-clone alpha release. It publishes the managed MCP
profile trust store for the local stdio Unlimited Tools gateway.

## Upgrade

From a local clone:

```powershell
git fetch --tags origin
git checkout v0.4.8-alpha
python -m pip install -e ".[all]"
```

For a working branch that tracks `main`:

```powershell
git fetch origin
git checkout main
git pull --ff-only
python -m pip install -e ".[all]"
```

## What Changes

- `unlimited-skills mcp trust status|list|import|revoke|doctor` manages the
  local trust store used by signed MCP profile bundles.
- The local trust store keeps E14 `trusted-keys.json` and `crl.json` formats as
  the verification backend and adds a store-only metadata sidecar for display,
  scope notes, validity windows, and audit provenance.
- `trust import` accepts public Ed25519 keys only and refuses private-looking
  material before writing.
- `trust revoke` appends key ids or bundle SHA-256 values to the local CRL.
- `trust doctor` checks malformed files, corrupt trust store state, unreadable
  CRLs, revoked key state, expired key state, and missing trust store behavior.
- `mcp gateway --profile-bundle` defaults to the managed trusted-keys file when
  no explicit `--trusted-keys` file is provided and the managed file exists.

## Compatibility

This release does not add hosted trust fetch, registry sync, production signing
keys, private key storage, OAuth, remote upstreams, MCP resources, or MCP
prompts.

MCP v1 schemas/configs/profiles/bundles/trust-store files are alpha and may
break before v0.6. The trust-store metadata sidecar is informational only; it
cannot widen bundle scope, audience, or profile permissions.

## Boundaries

- No hosted trust fetch.
- No registry sync.
- No production signing keys.
- No private key storage.
- No OAuth.
- No remote upstreams.
- No MCP resources or prompts.
- No hosted gateway.
- No production hosted calls.
- No automatic telemetry.
- No arbitrary shell execution.
- No full catalog distribution.
- No auto-publish.
- No live billing.
- No PyPI publication.

## Publication

After the final publication verifier passes on the selected `main` SHA:

```powershell
git tag -a v0.4.8-alpha <verified-main-sha> -m "v0.4.8-alpha"
git push origin v0.4.8-alpha
```
