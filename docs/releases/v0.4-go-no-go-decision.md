## Non-Negotiable Boundaries

- No automatic telemetry.
- No prompt upload.
- No skill body upload.
- No automatic rewriting.
- No auto-publish.
- No live billing.
- No PyPI publication.
- No full catalog distribution.
- MIT local core remains registration-free.
- Signed hosted manifests remain required.
- No automatic install, update, or remove.
- No production hosted calls in public tests.

## Review Gates

- Release owner approves this GO decision.
- Runtime review before any public client mutation-adjacent UX.
- Registry operator review before any private registry production flow.
- Security and privacy review before hosted metadata expansion.
- PR hygiene cleanup before each release layer.

## Residual Risks

- GO authorizes implementation epics, not production rollout. Each implementation PR still needs its own release gates and security/privacy review.
- Public fixture mode can diverge from private registry production behavior. External local-registry mode and private registry gates remain mandatory before release.

## Branch Cleanup

All prior v0.4 blocker and readiness branches were merged and deleted before this decision branch started. Delete `release/v04-go-no-go-decision` after merge if no downstream PR depends on it.
