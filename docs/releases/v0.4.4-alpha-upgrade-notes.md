# v0.4.4-alpha Upgrade Notes

`v0.4.4-alpha` is a GitHub-clone alpha release. It publishes MCP
permissioned tool profile runtime enforcement for the local stdio Unlimited
Tools gateway.

## Upgrade

From a local clone:

```powershell
git fetch --tags origin
git checkout v0.4.4-alpha
python -m pip install -e ".[all]"
```

For a working branch that tracks `main`:

```powershell
git fetch origin
git checkout main
git pull --ff-only
python -m pip install -e ".[all]"
```

## What Changes

- MCP tool profiles are enforced at runtime when the gateway is started with
  `--profiles FILE`.
- Selected profiles are default-deny: tools are hidden until the profile makes
  them visible.
- `tools_search` returns only visible tools and marks view-only tools as not
  callable.
- `tools_schema` refuses hidden tools with `TOOL_NOT_VISIBLE` / `-32011`
  before existence checks or upstream spawn.
- `tools_call` refuses visible-but-not-callable tools with
  `TOOL_NOT_CALLABLE` / `-32012` before upstream spawn.
- Missing selected profiles fail closed with `PROFILE_NOT_FOUND` / `-32013`.
- Invalid profile files fail closed with `PROFILE_INVALID` / `-32014`.
- `extends` inheritance is restriction-only: child profiles can narrow but
  cannot widen parent grants.
- Audit rows carry the active profile name, and startup records a
  `profile_loaded` row with the profile file SHA-256.

## Compatibility

No-profiles mode remains the unchanged open default until v0.6. Profiles are
loaded once at gateway startup; changing a profile file requires a gateway
restart.

MCP v1 schemas/configs/profiles are alpha and may break before v0.6.

## Boundaries

- No OAuth.
- No remote upstreams.
- No MCP resources or prompts.
- No hosted gateway.
- No production hosted calls.
- No automatic telemetry.
- No arbitrary shell execution.
- No full catalog distribution.
- No auto-publish.
- No live billing.
- No PyPI publication.
