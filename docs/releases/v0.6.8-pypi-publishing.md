# PyPI Trusted Publishing for v0.6.8

The release workflow uses PyPI OIDC Trusted Publishing. It has no package token
or local `.pypirc` fallback.

| PyPI field | Required value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |

Manual workflow inputs:

```text
version: 0.6.8
expected_sha: <exact current main SHA>
confirm_pypi_publish: publish unlimited-skills 0.6.8 to PyPI
```

The workflow verifies that `expected_sha` is exactly `origin/main`, runs the
full product/frozen-contract suite, builds wheel and sdist once, runs `twine
check`, tests the exact wheel for inherited retrieval and signed receipt
transport, proves daemon rollover, and then publishes with OIDC.

After upload, a clean public-index install and local artifact digests must
match. Only then may the workflow create GitHub tag/release `v0.6.8` at the
same SHA.

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is
allowed in this path.
