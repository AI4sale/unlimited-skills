# v0.4.1-alpha Known Issues

`v0.4.1-alpha` is an alpha reliability milestone, not a production SLA.

- Codex does not create or push the final release tag. The release owner must verify the final `main` SHA and run the human tag command.
- Hosted production calls are intentionally disabled in tests.
- Production signing keys are not required by this gate.
- Full catalog distribution remains disabled.
- Live billing remains disabled.
- PyPI publication remains disabled.
- Automatic telemetry, automatic rewriting, and auto-publish remain disabled.
- `v0.4.0-alpha` publication verification now has an explicit post-tag mode for release smoke. The strict pre-tag verifier still blocks Codex from creating or overwriting tags by default.
- Local recommendation context preview work must wait until the v0.4.1-alpha reliability publication gate is merged, tagged by the release owner, or explicitly blocked with owner, action, and fallback.
