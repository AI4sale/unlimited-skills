# v0.4.3-alpha Upgrade Notes

`v0.4.3-alpha` is a GitHub-clone alpha release. It publishes the MCP upstream
enforcement gate and includes the E09 permissioned tool profile design
documents, but it does not enable profile runtime enforcement.

## Upgrade

From a local clone:

```powershell
git fetch --tags origin
git checkout v0.4.3-alpha
python -m pip install -e ".[all]"
```

For a working branch that tracks `main`:

```powershell
git fetch origin
git checkout main
git pull --ff-only
python -m pip install -e ".[all]"
```

## What Changes

- MCP upstream configs are enforced at runtime for local stdio upstreams.
- Disabled upstreams and future remote placeholders are refused before spawn or
  indexing.
- Commands, environment forwarding, schema size, response size, startup timeout,
  request timeout, audit rotation, and audit redaction now have release evidence.
- Permissioned MCP tool profile design/schema/docs are included for review and
  future runtime work.

## Compatibility

MCP v1 schemas/configs/profiles are alpha and may break before v0.6.

Profile runtime enforcement is not part of `v0.4.3-alpha`; agents still need the
next E10 runtime layer before profile visibility/callability is enforced by the
gateway.

## Boundaries

- No OAuth.
- No remote upstreams.
- No MCP resources or prompts.
- No hosted gateway.
- No production hosted calls.
- No automatic telemetry.
- No arbitrary shell execution.
- No full catalog distribution.
- No auto-publish.
- No live billing.
- No PyPI publication.
