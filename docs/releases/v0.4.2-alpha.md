# v0.4.2-alpha

Status: final publication gate for the v0.4.2-alpha Unlimited Tools MCP milestone.

`v0.4.2-alpha` publishes the E06 Unlimited Tools MCP server/gateway core and the E07 MCP upstream security model contract. This final publication package adds the release smoke, publication verifier, upgrade notes, final manifest, known issues, and human tag instructions on top of the integration evidence. It does not implement OAuth, remote upstreams, MCP resources, MCP prompts, a hosted gateway, automatic telemetry, E08 runtime enforcement, or a final release tag.

Codex must not create or push the final `v0.4.2-alpha` tag. The release owner runs the human tag command after this publication PR is merged and the selected `main` SHA is verified.

## Integrated MCP Work

- E06 MCP core: `unlimited-skills mcp serve` exposes `skills_search`, `skills_view`, and `skills_use` over stdio JSON-RPC.
- E06 Unlimited Tools gateway: `unlimited-skills mcp gateway` exposes `tools_search`, `tools_schema`, and `tools_call` over one local stdio MCP endpoint.
- Fixture stdio handshake coverage for skills server and gateway startup.
- Metadata-only `skills_search`, one-body `skills_view`, and local safe/dry `skills_use` event logging.
- Compact `tools_search` that never dumps full tool schemas.
- Single-tool `tools_schema` that lazily spawns/reuses an upstream.
- Fixture `tools_call` routing through a fake upstream without production hosted calls.
- Structured refusal evidence for malformed requests, unknown tools, and unavailable upstreams.
- Redacted MCP audit evidence with no token, private key, proof, prompt, skill body, search query, private pack body, or local path leakage.
- E07 upstream security model evidence: default `local-restricted`, no shell, `env_allowlist` names only, wildcard forwarding impossible, size limits refuse instead of truncate, timeout caps, no OAuth/remote upstream, and no MCP resources/prompts.
- Publication verifier evidence for version `0.4.2`, final release manifest, tag-target SHA, no private material in release docs, no full schema dump into agent context, and exact human tag instructions.

## Human Tag Command

After this PR is merged and the release owner has verified the final `main` SHA:

```powershell
git checkout main
git pull --ff-only origin main
$sha = git rev-parse HEAD
python scripts/verify-v042-alpha-publication.py --expected-sha $sha
git tag -a v0.4.2-alpha $sha -m "v0.4.2-alpha"

```
