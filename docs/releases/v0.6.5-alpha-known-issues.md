# v0.6.5-alpha Known Issues

Status: known issues and claim boundaries for the retrieval/learning reliability
release package.

## Retrieval and Learning Are Better, Not Perfect

`v0.6.5-alpha` reduces zero-candidate failures and aligns search, suggest, hook,
and learning paths around one candidate family. It does not claim that search is
perfect, that every useful skill ranks top-1, or that all ambiguous prompts are
resolved without user judgment.

Low-confidence candidates may still appear. That is intentional: recall-first
delivery sends candidate names/hints instead of suppressing them to zero.

## Local Learning Is Local Ranking, Not Hosted Intelligence

Accepted usage and rejected/wrong feedback influence future local ranking
through privacy-safe query fingerprints and token hashes. This does not upload
prompts, train a hosted model, create a marketplace, or enable hosted/team/
business/enterprise rollout.

## Money and Billing Claims Are Out of Scope

This release is not a Money Saved feature release. It makes no exact revenue,
exact money claim, exact provider token claim, billing reconciliation claim, or
guaranteed bill reduction claim.

## Prior v0.6.4 State

The old `unlimited-skills==0.6.4` remains yanked because it reported
context/token proxy savings only and did not compute dollar savings. This
release does not unyank or overwrite `0.6.4`.
