# v0.3.3-alpha

Status: org/team governance and private-pack diagnostics integration gate.

`v0.3.3-alpha` connects the public client diagnostics to the private registry organization, team, namespace, entitlement, and private-pack ownership model. The public MIT local core remains usable without registration.

## Highlights

- Package version is `0.3.3`.
- GitHub clone remains the supported alpha distribution path.
- `org status` reads local cached organization/team governance status without hosted calls.
- `org status --refresh` is reserved for registered hosted refresh.
- `private-packs access-check <pack_id>` reports redacted entitlement and policy denial reasons.
- `private-packs doctor` reports local private-pack setup without contacting hosted private-pack endpoints.
- Support bundle includes redacted org/team/private-pack entitlement summaries.
- Cross-repo fixture validation verifies public CLI behavior against the private registry org/team namespace model.

## Security Boundary

- MIT local commands remain unregistered.
- `org status` is local/cache-only unless `--refresh` is passed.
- `org status --refresh` requires registration, hosted token, and device proof.
- `private-packs access-check <pack_id>` requires registration, hosted token, and device proof.
- Access-check output uses hashed pack references and stable denial codes such as `no_entitlement`, `not_team_member`, `wrong_agent`, `wrong_channel`, `revoked`, `policy_denied`, and `service_unavailable`.
- Support and diagnostics do not expose private pack names by default, private skill names, private skill bodies, archive URLs, local paths, tokens, proofs, or private keys.
- No private registry skill bodies, signing private keys, hosted tokens, or device private keys are shipped in the public repo.
