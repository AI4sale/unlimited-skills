# v0.6.4-alpha

Status: release execution package ready for final Opus review and release
operator execution. This document sets the intentional package version state for
`unlimited-skills==0.6.4`, but it does not upload, tag, publish, yank, or create
a GitHub release by itself.

`v0.6.4-alpha` ships the local Money Saved Meter release package. The release
claim is command-based: users can run the local Free meter, Registered export,
Team rollup, Business admin export, Enterprise evidence pack, clean verifier,
and tamper fail-closed verifier.

## User-Facing Value

- Free: run `unlimited-skills money-saved meter --json`.
- Registered: write a local Money Saved export with
  `unlimited-skills money-saved registered-export`.
- Team: aggregate registered exports with
  `unlimited-skills money-saved team-rollup`.
- Business: write local admin JSON/CSV with
  `unlimited-skills money-saved admin-export`.
- Enterprise: write and independently verify a tamper-evident local evidence
  pack with `unlimited-skills money-saved evidence-pack` and
  `unlimited-skills money-saved verify-evidence-pack`.

All surfaces are local-only. They do not upload data, call hosted services,
enable telemetry, reconcile provider billing, or claim exact money/token
savings. Counts stay counts, measured bytes stay measured bytes, tokens are
explicit estimates, dollars remain disabled by default, and the 100-call frame
is cadence rather than billing math.

## Release Execution Gate

Before any tag or PyPI publish:

1. Merge C064-REL-01 into `main`.
2. Obtain Opus `O064-REL-FINAL-R1` PASS.
3. Run the smoke verifier, release execution verifier, frozen-contract verifier,
   feedback boundary verifier, package smoke, focused tests, full suite, and
   `git diff --check` from the selected `main` SHA.
4. Dispatch `.github/workflows/publish-pypi.yml` with:
   `version=0.6.4`, the exact selected SHA, and
   `confirm_pypi_publish=publish unlimited-skills 0.6.4 to PyPI`.
5. After the workflow succeeds, install `unlimited-skills==0.6.4` from PyPI in a
   clean environment and rerun the post-publish smoke.
6. Create `v0.6.4-alpha` only after clean PyPI install and post-publish verifier
   pass.

Release execution decision for this PR: deferred to the release operator because
Trusted Publishing workflow dispatch, PyPI upload, post-publish clean install,
tag, and GitHub prerelease creation are external release actions. The package is
prepublish-ready when all C064-REL-01 gates pass.

## Boundaries

- No exact money savings, no exact token savings, and no guaranteed bill reduction.
- No hosted dashboard, hosted telemetry, live sync, automatic upload, SSO/SCIM,
  hosted governance, billing, or signature-enforced claim.
- No PyPI publish, tag, GitHub prerelease, yanking, marketplace submission, or
  hosted rollout happens inside this PR.
- No hosted/team/business/enterprise service rollout is claimed. The above-Free
  tier surfaces are local command artifacts only.
- No docs-only tier value claim. Tier value must be tied to runnable commands and
  verifier evidence.
- No #195 merge-as-is.
- No #119/E19 reopening or release inclusion.
