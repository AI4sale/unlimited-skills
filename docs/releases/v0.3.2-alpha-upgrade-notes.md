# v0.3.2-alpha Upgrade Notes

Upgrade from a GitHub clone:

```bash
git fetch --tags origin
git checkout <v0.3.2-alpha-tag-or-release-branch>
python -m pip install -e ".[all]"
unlimited-skills --version
unlimited-skills reindex
unlimited-skills doctor --json
```

PyPI is not the supported `v0.3.2-alpha` distribution path.

## From v0.3.0-alpha

- Existing local libraries should remain in place.
- Native sync and hosted update flows must not delete pre-existing local `local/` skills.
- Private team packs install under `registry/private/<pack_id>` and are tracked in `.unlimited-skills-private-packs.json`.
- `private-packs remove` refuses to delete paths outside registry-owned private pack metadata.
- Private pack diagnostics are local and redacted by default.
- Private pack hosted operations require registration, device proof, trusted private-pack manifest keys, and registry-side entitlement or Business/Enterprise plan.

## Verify After Upgrade

```bash
python scripts/run-private-team-pack-cross-repo-e2e.py --fixture-mode --temp-home
python scripts/run-v0.3.2-alpha-private-packs-smoke.py
python scripts/verify-v0.3.2-alpha-private-packs.py
```

Registered hosted flows still require a registered installation, hosted token, signed device proof, trusted signed manifests, and registry-side authorization.
