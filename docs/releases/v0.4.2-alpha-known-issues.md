# v0.4.2-alpha Known Issues

`v0.4.2-alpha` is an alpha MCP integration gate, not a production SLA.

- Codex does not create or push the final release tag. The release owner must verify the final `main` SHA and run the human tag command.
- Hosted production calls are intentionally disabled in tests.
- The MCP gateway is local stdio only; there is no hosted gateway.
- OAuth and remote upstreams are out of scope.
- MCP resources and prompts are out of scope.
- E07 is a security model and schema contract; E08 runtime enforcement is not included yet and lands in a later implementation gate.
- MCP v1 schemas/configs are alpha and may break before v0.6.
- `tools_search` intentionally returns compact metadata only. Full schemas are available one tool at a time through `tools_schema`.
- Size-limit and timeout-limit behavior is verified at the security model/schema layer in this gate; full runtime enforcement is tracked by the E08 implementation work.
- Full catalog distribution, live billing, PyPI publication, automatic telemetry, automatic rewriting, and auto-publish remain disabled.
