# v0.3.4-alpha

Status: release integration gate for plan, entitlement, and sandbox billing diagnostics.

`v0.3.4-alpha` adds public client visibility for registered plan and billing lifecycle state while keeping the MIT local core usable without registration. Billing support is diagnostic and sandbox-only. The public client does not create checkout sessions, collect payment data, store card or bank data, or enable live payment providers.

## Highlights

- Package version is `0.3.4`.
- GitHub clone remains the supported alpha distribution path.
- `plan status`, `plan explain`, and `plan doctor` remain local/cache-only.
- `plan refresh` requires registration and uses `/v1/hub/entitlements`.
- `billing status` and `billing doctor` are local/cache-only.
- `billing refresh` requires registration and uses `/v1/hub/billing-status`.
- Shared denial vocabulary now includes `past_due`, `suspended`, and `expired`.
- Support bundle includes a redacted billing lifecycle summary.
- Cross-repo fixture validation verifies `subscription_active`, `payment_failed`, and `subscription_canceled` against the private registry sandbox billing ledger.
- No production hosted calls are required for release tests.

Private registry PRs included or required:

## Security Boundary

- MIT local commands remain unregistered.
- `billing status` and `billing doctor` are local/cache-only.
- `billing refresh` requires registration, hosted token, and device proof.
- Billing mode is `sandbox_only`.
- Live payment providers are not enabled.
- Checkout sessions are not created.
- Payment data is not collected.
- Support and diagnostics do not expose checkout URLs, payment links, invoice URLs, card data, bank data, private skill names, private skill bodies, local paths, hosted tokens, device proofs, or private keys.
