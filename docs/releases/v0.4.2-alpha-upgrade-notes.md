# v0.4.2-alpha Upgrade Notes

`v0.4.2-alpha` is the Unlimited Tools MCP milestone on top of `v0.4.1-alpha`.
It publishes the local MCP skills server/gateway and the E07 upstream security
model contract as an alpha GitHub-clone release.

## Recommended Upgrade

Use a GitHub clone, not PyPI, for this alpha:

```powershell
git fetch origin --tags
git checkout v0.4.2-alpha
python -m pip install -e ".[all]"
```

Then reinstall the router/agent integration through the installer for your
agent if needed.

## What Changes

- Package and plugin metadata move to `0.4.2`.
- `unlimited-skills mcp serve` exposes `skills_search`, `skills_view`, and
  `skills_use` over local stdio JSON-RPC.
- `unlimited-skills mcp gateway` exposes `tools_search`, `tools_schema`, and
  `tools_call` as a compact front door for many upstream MCP servers.
- `tools_search` stays metadata-only and does not dump input schemas into the
  agent context.
- `tools_schema` retrieves one selected tool schema at a time.
- `tools_call` routes one selected upstream call in fixture-tested local mode.
- The E07 upstream security model documents the next config generation:
  default `local-restricted`, no shell, names-only `env_allowlist`, impossible
  wildcard forwarding, refusal-not-truncation size limits, timeout caps, and no
  OAuth/resources/prompts.

## Boundaries

- Alpha only.
- MCP v1 schemas/configs may break before v0.6.
- No OAuth.
- No remote upstreams.
- No MCP resources or prompts.
- No hosted gateway.
- No production hosted calls.
- No automatic telemetry.
- No arbitrary shell execution.
- No E08 runtime enforcement yet.
- No PyPI publication.
- No tag push by Codex.

## Release Owner Step

Codex prepares this publication package but must not push `v0.4.2-alpha`.
After the final publication PR is merged, the release owner verifies the final
`main` SHA and creates the tag manually:

```powershell
git checkout main
git pull --ff-only origin main
$sha = git rev-parse HEAD
python scripts/verify-v042-alpha-publication.py --expected-sha $sha
git tag -a v0.4.2-alpha $sha -m "v0.4.2-alpha"
git push origin v0.4.2-alpha
```
