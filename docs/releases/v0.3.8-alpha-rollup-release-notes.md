# Unlimited Skills v0.3.8-alpha Rollup Release Notes

Status: release backlog merged and ready for release-owner review.

This rollup covers the public core/router repository and the registered private registry work merged during the v0.3.1 through v0.3.8 alpha train. The release turns Unlimited Skills from a local router prototype into a governed skill delivery system with registration, team/private pack controls, catalog review, browsing, feedback, and signed quality signals.

## What Changed

### Public Core and Client

- Added first-run setup, service diagnostics, support bundles, and redacted local status reports.
- Added plan and entitlement diagnostics: `plan status`, `plan refresh`, `plan explain`, and `plan doctor`.
- Added billing lifecycle diagnostics for sandbox status reconciliation: `billing status`, `billing refresh`, and `billing doctor`.
- Added private team pack client flows for registered installs, including list, preview, install, sync, installed-state, removal, access-check, and doctor commands.
- Added organization/team status commands and local cached governance diagnostics.
- Added managed Enterprise Skill Lock policy sync commands with signed policy verification and unmanaged-removal refusal.
- Added community catalog submission, review-status, preview, install, withdraw, and review-notes UX.
- Added signed registered catalog browser commands: browse, search, filters, preview, and dry-run install.
- Added explicit catalog feedback commands and redacted support bundle summaries.
- Added catalog quality and skill evaluation status commands: `catalog quality`, `catalog eval-status`, `catalog explain-risk`, and install risk warnings.
- Added cross-repo E2E runners, smoke tests, release verifiers, schemas, examples, upgrade notes, known limitations, and release manifests through v0.3.8-alpha.

### Registered Private Registry

- Added signed registry artifact generation and validation for registered catalog distribution.
- Added production signing ceremony workflow, signing handoff records, and registry operator acceptance gates.
- Added release channel manifests and signed channel-aware catalog lookup.
- Added private team pack distribution API with signed access enforcement.
- Added private team pack publishing pipeline, validation, signing, audit records, and admin management.
- Added plan and entitlement gates for private team packs, including stable denial codes and Business/Enterprise plan behavior.
- Added organization, team, namespace, membership, and governance models.
- Added admin console MVP shell, read-only data views, audited operator actions, organization/team governance, plan/billing management, and sandbox billing ledger.
- Added community catalog submission review workflow and approved/published distribution path.
- Added signed catalog browser API, metadata filtering, preview payloads, and quality metadata exposure.
- Added explicit catalog feedback API, summary/triage workflow, and admin feedback visibility.
- Added skill evaluation harness, catalog quality scoring, quality schemas, evaluation result validation, and aggregate audit reports.

## Security and Privacy Boundaries

- No automatic telemetry is enabled by the public client.
- Public tests use fixture mode and do not call production hosted services.
- Catalog browsing, quality status, and evaluation status are metadata-only surfaces.
- Skill bodies, prompts, task text, customer data, private paths, proof headers, tokens, and private keys must not appear in support bundles or public release artifacts.
- Hosted pack access requires registration and signed manifest verification.
- Private team pack access requires entitlement or an eligible Business/Enterprise plan.
- Managed Enterprise policy sync can remove only policies previously installed by managed sync with matching policy identifiers and hashes.
- Blocked, local-only, and needs-review catalog items remain excluded from automatic hosted distribution.

## Release Notes for Operators

- Registered catalog install/download paths now depend on the hosted registry API and signed registry artifacts.
- A clean install can discover approved community packs, browse signed metadata, preview install plans, and receive quality/evaluation status after registration.
- Team and private-pack flows are now governed by registry-side entitlement checks, team/org metadata, and signed access records.
- The public core remains usable without registration for local routing and local skill library behavior.
- Do not widen full catalog distribution until blocked, local-only, needs-review, and tool-dependency classes have explicit policy decisions and capability checks.
