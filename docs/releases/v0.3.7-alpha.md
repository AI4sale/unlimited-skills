# v0.3.7-alpha

Status: release integration gate for the explicit catalog feedback loop.

`v0.3.7-alpha` connects the public `catalog feedback` client to the private registry catalog feedback API using fixture-only cross-repo verification. The public MIT local core remains usable without registration.

## Scope

- Registered `catalog feedback` and `catalog feedback-status` commands.
- Client-side redaction checks before feedback submit.
- Registry-side feedback submit, summary, triage, admin-console visibility, and catalog quality signals.
- Fixture-only cross-repo E2E between the public client and private registry.

## Security Boundary

- Feedback is explicit only.
- No automatic telemetry is sent.
- Dry-run sends no hosted request.
- Feedback submit requires registration and user confirmation.
- Public tests do not call production hosted services.
- Feedback must not include prompts, task text, skill bodies, local paths, repo paths, customer data, tokens, device proofs, private keys, archive URLs, checkout URLs, or payment links.

## Release Owner Gate

Before tagging, the release owner should run:

```bash
python scripts/run-v0.3.7-alpha-catalog-feedback-smoke.py
python scripts/verify-v0.3.7-alpha-catalog-feedback.py --expected-sha <tag-target-sha>
```

This branch depends on the private registry feedback API PR and the public catalog feedback client PR being reviewed and merged first.
