# v0.5.0-alpha — PyPI-flip UX copy review

Reviewed copy for every user-facing surface that changes at the A3 flip
(the moment `unlimited-skills` is actually installable from PyPI). The flip
is mechanical — every site carries an `A3-PYPI-FLIP` marker — but the words
around the command matter as much as the command. This document fixes the
post-flip wording once so the flip PR is a find-and-replace, not a writing
exercise.

Until the package is live, nothing below ships. The pre-publication verifier
(`scripts/verify-v050-alpha-publication.py`) enforces that markers stay in
place before availability and disappear after it.

Every post-flip command below carries an `A3-PYPI-FLIP` trailer to satisfy
the install-path guard test; strip those trailers when transcribing the copy
into the flip PR.

## 1. README install block (README.md, top "Start here" section)

Post-flip copy:

```markdown
Install the core (light, no vector dependencies):

    pip install unlimited-skills  # A3-PYPI-FLIP: post-flip copy

Want hybrid vector search? Install the extras after trying the core:

    pip install "unlimited-skills[vector]"

Then take the 2-minute golden path:

    unlimited-skills quickstart
```

Review notes:

- Lead with the plain command and nothing else. The git+ fallback should NOT
  remain as an alternative in the same block — two install commands at the
  top is a decision the new user should not have to make. Keep the git+
  source install only in the contributor/development section.
- Delete the `A3-PYPI-FLIP` HTML comment together with the flip (risk R4:
  internal release machinery must not ship in the published long
  description).
- Delete every "the package is not published on PyPI yet" sentence in the
  same PR (risk R5: self-contradicting on the PyPI page). Known sites:
  README install section and the v0.4.2 distribution-path paragraph.

## 2. Runtime hints (cli.py, hub.py, commands/library.py)

Current pre-flip hint (verified in `unlimited_skills/hub.py` and
`unlimited_skills/commands/library.py`) recommends the git+ extras install.
Post-flip copy for the missing-extras hint:

```text
Vector search needs optional dependencies. Install them with:
  pip install "unlimited-skills[vector]"
```

Review notes:

- The hint must quote the extras form, not the bare package — a user who
  sees this hint already has the core installed.
- Keep the hint to two lines. It renders inside CLI error output; long
  prose there reads as noise.

## 3. Plugin surfaces (docs/claude-code-plugin.md, plugin SKILL.md, session_start.py missing-CLI hint)

Post-flip copy for the missing-CLI hint:

```text
unlimited-skills CLI not found. Install it with:
  pip install unlimited-skills  # A3-PYPI-FLIP: post-flip copy
Then restart the session so hooks can find it.
```

Review notes:

- The restart sentence is load-bearing: hooks resolve the CLI at session
  start, and without it users report "installed but still broken".
- `.claude-plugin/marketplace.json` cannot carry a marker comment; its
  install hint is pinned by `tests/test_install_path_docs.py` instead.
  The flip PR must update that test's expectation in the same commit.

## 4. Marketplace listing (docs/adoption/marketplace-listing-copy.md)

The listing draft already carries its own `A3-PYPI-FLIP` marker. Post-flip,
its install section collapses to the plain command plus quickstart. No other
listing copy changes at flip time.

## 5. Built-artifact gate (applies to the flip PR itself)

Before uploading anything, grep the BUILT artifacts (not the repo):

- `A3-PYPI-FLIP` — must be absent from the wheel/sdist METADATA and long
  description.
- `git+https://` — must be absent from install instructions in METADATA.
- "not published on PyPI" / "not on PyPI" — must be absent.

`scripts/run-v050-alpha-package-smoke.py` already builds and `twine check`s;
extend it with these greps in the flip PR (risk R12 closure).
