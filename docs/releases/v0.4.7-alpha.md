# v0.4.7-alpha

Status: signed MCP profile bundle integration gate.

`v0.4.7-alpha` integrates the E13 signed profile bundle design with the E14
runtime verification path. The gate proves that signed MCP profile bundles can
be verified locally before gateway profile loading, and that invalid bundles
fail closed before profile enforcement.

This is an integration gate, not a final publication gate. Codex must not
create or push the `v0.4.7-alpha` tag from this PR. A later final publication
gate must verify the selected `main` SHA before tagging.

## Integrated Work

- Public PR #102 added the E13 signed MCP profile bundle design, schema, and
  example.
- Public PR #103 added the E14 runtime verification prototype for signed MCP
  profile bundles.
- Private registry PR #50 added the Tier B writer inventory readiness gate
  needed by the parallel registry consolidation track.
- The gateway can load `--profile-bundle` with `--trusted-keys` and
  `--audience-id`, verify it once at startup, and then enforce the embedded
  profile map like an E09 profile.
- Every bundle verification failure is fail-closed before profile enforcement:
  bad signature, unknown key, expired or not-yet-valid bundle, revoked bundle
  or key, wrong audience, and namespace ceiling violations.
- Audit provenance records bundle SHA-256, issuer key id/display, audience,
  expiry, and verification status without key material or signature values.
