# Unlimited Skills v0.2.2-alpha

This alpha is the productionization release gate for the registered registry client and signed release-channel update flow. It builds on `v0.2.1-alpha` and must be tagged only after private registry PR #3 through PR #5 and public PR #20 through PR #24 are reviewed and merged in order.

## Highlights

- Production-shaped registry contract E2E coverage now exercises registered device proof, public key discovery, signed catalog/update/enhancement/hub/team/release-channel manifests, hub heartbeat, entitlement refresh, safe retries, signed offline cache fallback, and proof replay rejection without calling production hosts.
- Registered clients can inspect and pin release channels with `release status` and `release pin`.
- Hosted catalog and update checks can use a pinned channel or a one-off `--channel` override.
- Hosted update apply keeps a local rollback snapshot under `registry/.rollbacks/<collection>/`.
- `updates rollback <collection>` restores the latest local hosted collection snapshot.
- The official bundled trusted manifest key scope includes `release-channels`.

## Product Boundary

- Existing `unlimited-skills serve` remains the free MIT local daemon and does not require registration.
- `unlimited-skills hub serve` remains registration-gated.
- Local search, list, view, where, reindex, vector-reindex, adapt, native sync, installers, migration scripts, and public self-update remain local-first and unregistered.
- Local Skill Hub remains alpha and allowlist-only.
- Full catalog distribution remains disabled.
- Hosted registry services do not receive local hub search queries by default.
- The hub does not execute skills.
- Release-channel pinning affects registered hosted catalog/update calls only. It does not enable full catalog distribution.

## Security Notes

Hosted remote manifests must include valid signed manifest envelopes. Manifest signatures verify authenticity and scope; downloaded hosted archive bytes are still accepted only after SHA256 verification and safe extraction.

The public trusted manifest key id for this alpha is `registry-alpha-2026-06`. It is scoped to `hub-allowlist`, `catalog-updates`, `enhancement-manifest`, `team-sync-manifest`, and `release-channels` for `https://unlimited.ai4.sale`. The private registry signing key is not shipped in the public repo.

For serious LAN deployment, run Local Skill Hub behind reverse proxy or network controls that provide TLS, authentication, access logging, and IP allowlisting.
