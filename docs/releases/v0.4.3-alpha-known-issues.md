# v0.4.3-alpha Known Issues

`v0.4.3-alpha` is an MCP upstream enforcement integration gate, not a
production SLA.

- Codex creates and pushes the final `v0.4.3-alpha` release tag only after the
  final publication PR is merged and the publication verifier passes on the
  selected `main` SHA.
- Hosted production calls are intentionally disabled in tests.
- The MCP gateway is local stdio only; there is no hosted gateway.
- OAuth and remote upstreams are out of scope.
- MCP resources and MCP prompts are out of scope.
- Resources/prompts support is intentionally not exposed.
- Future remote upstream entries are placeholders and are refused until the
  OAuth/remote gate opens.
- The command allowlist is intentionally conservative; reviewed local
  upstreams may need `local-trusted` and one of the fixed known runners.
- `env_allowlist` forwards names only. Literal values, wildcard names, and
  the old v1 `env` map are rejected.
- Oversize schemas and responses are refused, never truncated. Agents must
  request narrower tools or smaller upstream outputs.
- Audit is always on. Minimal mode reduces argument/error detail but still
  records success/refusal observability.
- MCP v1 schemas/configs are alpha and may break before v0.6.
- MCP v1 schemas/configs/profiles are alpha and may break before v0.6.
- Permissioned MCP tool profiles are included as design/schema/docs only;
  runtime enforcement is not part of `v0.4.3-alpha`.
- Full catalog distribution, live billing, PyPI publication, automatic
  telemetry, automatic rewriting, and auto-publish remain disabled.
