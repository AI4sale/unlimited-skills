# v0.3.5-alpha Known Issues

`v0.3.5-alpha` is a developer preview release gate for review-gated community catalog submission and consumption. It is not a production SLA.

## Distribution

- GitHub clone is the supported alpha distribution path.
- PyPI remains deferred because router skills, installers, schemas, docs, scripts, and bundled repo assets must be available beside the checkout.

## Hosted Community Catalog

- Hosted community operations require a registered installation and hosted service support.
- Community `submit --dry-run` is local-only and does not upload content.
- Community preview and install require signed hosted responses.
- Community install is limited to signed items whose review status is `approved` or `published`.
- Pending, rejected, withdrawn, deprecated, retired, and unreviewed items must not install silently.
- Full catalog distribution remains disabled.
- Stable promotion is not automatic.

## Marketplace And Billing

- This release does not include a marketplace storefront.
- This release does not include billing, checkout, revenue share, invoices, cards, bank data, refunds, or live payment provider integrations.
- Donations and hosted-service commercial terms remain separate from the MIT local core.

## Signing Status

Production release signing status remains release-owner controlled. Fixture manifests and local test keys are used only for reproducible test evidence. The release owner must confirm production signing or record an explicit override before a broad production release.

## Privacy

- Support bundle and diagnostics are local and redacted by default.
- Diagnostics must not include hosted tokens, device proofs, device private keys, private skill bodies, private skill names, private pack names, raw archive URLs, local paths, prompts, search queries, payment links, checkout URLs, invoice URLs, card data, or bank data.
- Users decide whether to share support bundle output with support.
