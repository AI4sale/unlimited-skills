# v0.3.7-alpha Upgrade Notes

`v0.3.7-alpha` is a catalog feedback milestone. It does not change the local MIT core install requirement, and it does not require registration for local search, list, view, use, reindex, native sync, self-update, local feedback, or local daemon commands.

## From v0.3.6-alpha

1. Pull the public repo release branch or tag after release-owner approval.
2. Run local smoke checks:

```bash
python scripts/run-v0.2x-smoke-tests.py
python scripts/run-v0.3.7-alpha-release-smoke.py
```

3. Registered users can test catalog feedback with `--dry-run` before sending anything:

```bash
unlimited-skills catalog feedback community:browser-qa-pack:0.1.0 --type documentation_issue --dry-run --json
```

4. Submit requires registration and confirmation:

```bash
unlimited-skills catalog feedback community:browser-qa-pack:0.1.0 --type documentation_issue --yes
```

## Privacy Boundary

Catalog feedback is explicit-only. The client rejects or excludes prompts, task text, skill bodies, local paths, repo paths, customer data, tokens, device proofs, private keys, checkout URLs, archive URLs, and payment links.

Support bundles include feedback boundary/status flags only. They do not include raw feedback payloads.
