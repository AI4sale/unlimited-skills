# v0.4.1-alpha

Status: reliability publication gate for the v0.4.1-alpha milestone.

`v0.4.1-alpha` packages the reliability layer that landed after the `v0.4.0-alpha` SkillOps foundation tag. It covers transactional installs, rollback manifests, model-aware vector index safety, CLI command modularization, and `skillops usage-snapshot` compatibility after the CLI split.

This publication package prepares the release notes, manifest, upgrade notes, known issues, smoke runners, and verifiers. Codex must not create or push the final `v0.4.1-alpha` tag automatically; the release owner runs the human tag command after the publication PR is merged and verified on the selected `main` SHA.

## Integrated Reliability Work

- Transactional installs for Codex-adjacent agent installers through shared `InstallTransaction` and schema v2 rollback manifests.
- Rollback support for Claude Code and OpenClaw installers, with Hermes legacy rollback compatibility preserved.
- Backup directory uniquification to prevent same-second reinstall collisions.
- Model-aware vector sidecar and Chroma metadata checks that raise `VectorModelMismatch` on stale embedding models.
- Hybrid search fallback to lexical with a visible warning unless `--require-vector` is set.
- CLI command module split under `unlimited_skills/commands/` while keeping `unlimited_skills.cli` as the public facade.
- `skillops usage-snapshot` routed through `unlimited_skills/commands/skillops.py`.
- Post-tag `v0.4.0-alpha` verifier compatibility: release smoke now verifies that the existing `v0.4.0-alpha` tag points to the expected release-owner commit instead of failing the whole smoke after tag publication.

## Security Boundary

- Alpha only.
- No production rollout.
- No production hosted calls.
- No automatic telemetry.
- No automatic rewriting.
- No automatic install, update, remove, reindex, or publish action outside explicit local installer commands.
- No auto-publish.
- No live billing.
- No PyPI.
- No full catalog distribution.
- No prompt, task text, search query, skill body, local path, repository path, token, proof, private key, customer data, or private registry content is uploaded by this release gate.
- MIT local core remains registration-free.

## Human Tag Command

After this PR is merged and the release owner has verified the final `main` SHA:

```powershell
git checkout main
git pull --ff-only origin main
$sha = git rev-parse HEAD
python scripts/verify-v041-alpha-publication.py --expected-sha $sha
git tag -a v0.4.1-alpha $sha -m "v0.4.1-alpha"
git push origin v0.4.1-alpha
```

`v0.4.1-alpha` hardens the installed-agent runtime path. It makes installs transactional, adds rollback coverage, prevents stale vector embedding-model reuse, and keeps the new modular CLI structure compatible with existing imports and the `skillops usage-snapshot` command.
