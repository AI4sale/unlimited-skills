# v0.5.2-alpha

Status: adoption instrumentation packaging gate; PyPI publication is blocked
until Trusted Publishing and post-publish smoke pass.

This release packages the adoption instrumentation that landed after
`v0.5.1-alpha`: the A4.7 public-alpha signal rollup generator, the A3.1.1
local effectiveness v2 instrumentation for the `suggest -> view -> use`
funnel, and the A3.1.1 privacy-gate follow-up that removed raw task/query text
and unsalted session fingerprints from local events.

## What Changed

- Package, Claude plugin, and marketplace metadata versions are `0.5.2`.
- `scripts/generate-public-alpha-signal-rollup.py` is included as the
  reproducible fixture/live public aggregate rollup path.
- `learning-summary --events` exposes local aggregate effectiveness metrics
  for the suggest/view/use funnel.
- The default `learning-summary` output remains unchanged without `--events`.
- The public `suggest --json` stdout contract remains unchanged.
- Raw session ids are never persisted; session correlation is hashed/truncated
  when a harness exports a session id.
- No eval-gate thresholds are changed.
- `scripts/run-v052-alpha-package-smoke.py` extends the v0.5.1 smoke with the
  signal-rollup fixture and installed-package effectiveness metrics checks.
- `scripts/verify-v052-alpha-publication.py` gates package metadata, release
  docs, manifest requirements, clean-install smoke, and post-publish PyPI
  smoke before tag guidance is emitted.
