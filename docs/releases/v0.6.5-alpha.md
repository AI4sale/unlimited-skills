# v0.6.5-alpha

Status: release execution package ready for review and release operator
execution. This document sets the intentional package version state for
`unlimited-skills==0.6.5`, but it does not upload, tag, publish, yank, or create
a GitHub release by itself.

`v0.6.5-alpha` is a technical-debt / reliability release. It does not introduce
a new product ladder, marketplace surface, hosted rollout, or paid-tier feature
work. It packages the retrieval and learning repairs merged in public PRs
#228-#233.

## User-Facing Value

The agent is much less likely to miss usable skills: search, suggest, and the
UserPromptSubmit hook now deliver candidate names/hints when retrieval has
candidates, and accepted usage teaches future ranking. The claim is bounded to
local retrieval/ranking behavior. It does not claim perfect search, guaranteed
top-1 ranking, exact revenue, exact money saved, billing reconciliation, or
hosted governance.

## What Changed

1. Zero-candidate delivery fixed: suggest/UserPromptSubmit no longer deliver 0
   candidates when retrieval has candidates.
2. Recall-first candidate delivery: low-confidence candidates are delivered as
   candidate names/hints instead of suppressed.
3. Shared candidate family: search, suggest, and UserPromptSubmit now use and
   verify an aligned candidate family.
4. Learning loop repaired: accepted skill usage boosts future exact/similar
   queries; wrong/rejected skills demote; manual search -> view -> use without
   repeating `--query` learns through privacy-safe query fingerprints.
5. Long-run gate: 100-step / 10-phase retrieval-learning verifier with injected
   negative controls.
6. Combined release smoke: one verifier runs the zero-candidate, shared-family,
   learning-loop, and long-run gates.

## Release Execution Gate

Before any tag or PyPI publish:

1. Merge the public v0.6.5-alpha release package PR into `main`.
2. Run the combined release smoke, release execution verifier, package smoke,
   frozen-contract verifier, feedback boundary verifier, focused tests, full
   suite, and `git diff --check` from the selected `main` SHA.
3. Confirm PyPI still has no `unlimited-skills==0.6.5`.
4. Dispatch `.github/workflows/publish-pypi.yml` with:
   `version=0.6.5`, the exact selected SHA, and
   `confirm_pypi_publish=publish unlimited-skills 0.6.5 to PyPI`.
5. After the workflow succeeds, install `unlimited-skills==0.6.5` from PyPI in a
   clean environment and rerun post-publish verification.
6. Create `v0.6.5-alpha` only after clean PyPI install and post-publish verifier
   pass.

Release execution decision for this PR: deferred to the release operator because
Trusted Publishing workflow dispatch, PyPI upload, post-publish clean install,
tag, and GitHub prerelease creation are external release actions.
