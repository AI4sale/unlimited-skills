# v0.4.1-alpha Upgrade Notes

`v0.4.1-alpha` is a reliability milestone on top of `v0.4.0-alpha`.

## From v0.4.0-alpha

1. Fetch the release after release-owner approval:

```powershell
git fetch --tags origin
git checkout v0.4.1-alpha
python -m pip install -e ".[all]"
```

2. Verify the local checkout:

```powershell
python scripts/run-v0.2x-smoke-tests.py
python scripts/run-v041-alpha-release-smoke.py
```

3. Reinstall or resync agent integration if you rely on installer rollback behavior or Claude Code plugin metadata:

```powershell
python -m pip install -e ".[all]"
unlimited-skills --version
```

The package and plugin metadata version is `0.4.1`, while the Git tag is `v0.4.1-alpha`.

## What Changes

- Agent installers use transactional manifests for rollback-aware install steps.
- Rollback manifests use schema version 2; Hermes legacy rollback remains supported.
- Claude Code and OpenClaw installers expose rollback flags.
- Same-second reinstall backups are uniquified.
- Vector search detects embedding model mismatch and raises `VectorModelMismatch`.
- Hybrid search can fall back to lexical results with a warning when vector state is stale.
- The CLI command implementation is split into command modules while preserving the `unlimited_skills.cli` facade.
- `skillops usage-snapshot` works through `unlimited_skills/commands/skillops.py`.
- The v0.4.0 release smoke now supports post-tag verification by checking the existing `v0.4.0-alpha` tag target.

## What Does Not Change

- Local search, list, view, where, use, feedback, reindex, vector-reindex, adapt, serve, installers, native sync, and self-update remain local-first.
- Registration is not required for the MIT local core.
- The release does not enable production rollout, live billing, PyPI publication, full catalog distribution, automatic telemetry, automatic rewriting, or auto-publish.
- The client must not upload prompts, task text, search queries, local paths, repository paths, tokens, proofs, private keys, skill bodies, private-pack bodies, or customer data.

## Tagging

Codex prepares this publication package but must not push `v0.4.1-alpha`. After the PR is merged, the release owner verifies the final `main` SHA and creates the tag manually.
