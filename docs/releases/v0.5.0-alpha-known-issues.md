# v0.5.0-alpha — known limitations (public alpha)

Unlimited Skills v0.5 is the first **public alpha**. It is free, open source
(MIT core), local-first, and there is nothing for sale. This page is the
honest list of what is rough, incomplete, or likely to change — each item
with a workaround where one exists.

Nothing in this product sends telemetry. All search, indexing, suggestion,
and measurement runs on your machine; logs are local files. Feedback reaches
us only when you choose to open a GitHub issue (see
[docs/feedback.md](../feedback.md)).

## 1. Alpha status: interfaces may change before 0.6

CLI commands, flags, JSON output shapes, config file formats, and library
layout may change in any 0.5.x release without a deprecation period. The
project has no legacy-user compatibility constraints yet, and we use that
freedom deliberately to fix design mistakes fast. Compatibility promises
start when the project reaches 0.6.

**Workaround:** pin the exact version you installed; read the changelog
before upgrading; do not build automation against `--json` output shapes
yet without a re-check on upgrade.

## 2. Windows-first testing

Day-to-day development and the measured calibration runs happen on Windows.
The bash installers, migration scripts, and the CLI are written to be
portable and run on macOS/Linux/WSL, but they get less routine testing
there in this alpha.

**Workaround:** everything is dry-run-by-default where it mutates files;
run installers and migrations in dry-run first. If something breaks on
macOS/Linux, please file an install-friction issue — those reports directly
drive the test matrix.

## 3. Repo installer scripts are still repo assets

The supported install today is:

```bash
pip install unlimited-skills
```

The v0.5 wheel includes the CLI and the bundled `ecc` + `superpowers` skill
packs used by `quickstart`, so a clean install has a first-value path without
cloning the repository. Repo installer scripts, release docs, and contributor
workflows still live in the GitHub checkout.

**Workaround:** for repo installer scripts, full docs, and local development,
clone the repository and run `python -m pip install -e ".[all]"` as described
in the README. For vector/hybrid search from a package install, install the
`[all]` extras.

## 4. Vector and hybrid search are optional and need a model download

Lexical search works offline with no extra dependencies. Vector and hybrid
modes require the `[vector]` extras (`chromadb`, `fastembed`), and the first
`vector-reindex` downloads an embedding model to your machine — that one
step needs network access and disk space. After the download, vector search
runs fully offline. Cold-start vector queries also pay a model-load cost per
process.

**Workaround:** stay on `--mode lexical` if you do not want the model
download; use `unlimited-skills serve` (the local warm daemon) to amortize
model loading for repeated vector/hybrid queries.

## 5. MCP gateway (Unlimited Tools) v1 boundaries

The MCP gateway is alpha and deliberately local-only in v1:

- stdio upstream servers only — no OAuth upstreams, no remote (`sse`/`http`)
  upstreams, no MCP resources, no MCP prompts, no hosted gateway mode;
- size and timeout caps on schemas and responses are enforced and fail
  closed;
- permissioned tool profiles are default-deny; a missing or invalid profile
  refuses rather than guesses.

**Workaround:** none for remote upstreams in v1 — keep those servers
registered directly with your host for now and put only stdio servers
behind the gateway.

## 6. `mcp savings` numbers are estimates with known edges

The token figure uses a `bytes / 4` heuristic, not a tokenizer. Remote
servers and opaque shell-string commands are skipped (reported as skipped
rows, never failures). With no MCP servers configured the command shows the
lab benchmark instead of your numbers.

**Workaround:** treat the output as an orientation measurement; the byte
counts themselves are exact local measurements of the `tools/list` payloads.

## 7. Skill suggestion: silence is a designed outcome

The `suggest` probe is tuned so that silence beats noise: a no-match returns
nothing rather than a weak guess. The measured effectiveness run on the
bundled 267-skill library (see
[docs/adoption/skill-effectiveness-standard.md](../adoption/skill-effectiveness-standard.md))
shows top-1 0.933 / top-3 0.967 with a 0.000 false-positive rate — which
also means some genuinely skill-eligible tasks score below the floor and get
no suggestion (a documented honest miss class).

**Workaround:** when `suggest` is silent but you expect a skill, run
`unlimited-skills search "<query>" --mode hybrid --limit 8` for deeper
retrieval — and please file a skill-not-invoked issue so the miss lands in
the eval set discussion. High-confidence ambient skill cards can be turned
off entirely with `UNLIMITED_SKILLS_NO_INJECT=1`.

## 8. Warm daemon is experimental

`unlimited-skills serve` is not yet the default retrieval path and its API
may change.

**Workaround:** treat it as an opt-in accelerator for large libraries.

## 9. Hosted/registered surfaces are early-access, not products

Registration-gated features (hosted catalog, community submissions, team
sync, policy sync, plan/billing diagnostics) exist as early-access and
fixture-verified surfaces with availability limits during alpha. Billing
surfaces are sandbox-only diagnostics: the client creates no checkout
sessions, no payment links, and collects no payment data. Nothing is for
sale in v0.5.

**Workaround:** the local core needs none of this — local search, view,
sync, import, MCP gateway, and self-update all work without registration.

## 10. Public alpha package publication gate

The repository is prepared for the public alpha package, but the release tag
stays blocked until PyPI Trusted Publishing uploads `unlimited-skills==0.5.0`
and a clean install from PyPI passes. The prepublish gate proves build,
`twine check`, bundled-pack inclusion, clean install from the built wheel,
`quickstart`, `suggest`, and `mcp savings`; the published gate must prove the
same commands from the real PyPI package. Treat the CLI and JSON outputs as
alpha interfaces until 0.6.

## 11. Claude Code plugin needs the CLI installed separately

The plugin (`/plugin marketplace add AI4sale/unlimited-skills`) ships the
router skill and session hooks, but retrieval is done by the
`unlimited-skills` CLI, which must be installed first. Without the CLI the
hooks degrade to a one-line install hint (they never fail the session).

## 12. Native sync runs on CLI calls, not in the background

Newly installed agent skills are mirrored into the library when the router
CLI next runs (`search`, `list`, `view`, `use`, `reindex`, ...). There is no
background file watcher.

**Workaround:** run any router command (or `unlimited-skills sync-native
--agent <agent>`) after installing new skills.
