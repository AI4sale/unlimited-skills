# v0.5.1-alpha

Status: adoption tools packaging gate; PyPI publication is blocked until
Trusted Publishing and post-publish smoke pass.

This release packages the adoption toolchain that landed after v0.5.0-alpha:
the Claude Code MCP installer, feedback prepare reports, marketplace/listing
launch pack, and first-week adoption measurement docs/templates.

## What Changed

- Package, Claude plugin, and marketplace metadata versions are `0.5.1`.
- `unlimited-skills mcp install --claude-code` is included for PyPI users and
  must pass clean-install dry-run smoke.
- `unlimited-skills feedback prepare` is included for PyPI users and must
  emit a local-only, redacted public-alpha feedback report.
- Marketplace/listing launch-pack docs are included so public copy points to
  commands that exist in the package.
- First-week adoption measurement docs and issue-template updates are included
  so feedback collection remains manual and telemetry-free.
- `scripts/run-v051-alpha-package-smoke.py` extends the v0.5.0 package smoke
  with the adoption commands that marketplace users are now told to run.
- `scripts/verify-v051-alpha-publication.py` gates package metadata, release
  docs, manifest requirements, clean-install smoke, and post-publish PyPI
  smoke before tag guidance is emitted.

## Boundaries

- #119 / E19 is not included in this release unless explicitly approved later.
- No marketplace submission is performed by this release gate.
- No paid launch, paid CTA, payment link, hosted readiness claim, Team
  readiness claim, or Enterprise readiness claim is added.
- No hosted readiness, no team readiness, and no enterprise readiness is
  claimed by this release.
- No telemetry is added.
- The tag `v0.5.1-alpha` remains blocked until the package is uploaded to PyPI
  and `verify-v051-alpha-publication.py --package-availability published`
  passes on the selected release commit.

## Required Verification

```bash
python -m pytest tests -q
python scripts/verify-mcp-boundaries.py
python scripts/verify-feedback-report-boundaries.py
python scripts/verify-marketplace-listing-launch-pack.py
python scripts/run-v051-alpha-package-smoke.py --json
python scripts/verify-v051-alpha-publication.py --expected-sha "$(git rev-parse HEAD)" --package-availability prepublish --json
python -m compileall -q unlimited_skills scripts tests
git diff --check
```

Run the final verifier with `--package-availability published` only after
`unlimited-skills==0.5.1` is available from PyPI. Before upload, published
mode is expected to block the tag.
