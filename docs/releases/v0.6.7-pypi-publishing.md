# PyPI Trusted Publishing for v0.6.7

Use only `.github/workflows/publish-pypi.yml` from the exact approved `main`
commit. Do not upload locally.

| Trusted Publisher field | Required value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |

Dispatch inputs:

```text
version: 0.6.7
expected_sha: <exact origin/main SHA>
confirm_pypi_publish: publish unlimited-skills 0.6.7 to PyPI
```

The workflow validates the selected SHA against `origin/main`, runs the source
and exact-artifact gates, publishes with OIDC Trusted Publishing, validates
PyPI filenames and SHA-256 digests, and clean-installs from the public simple
index. The verifier retries only the temporary JSON/simple-index propagation
race. Any actual install or smoke defect fails immediately. The matching GitHub
release is created only after public-wheel verification succeeds.

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is used.
