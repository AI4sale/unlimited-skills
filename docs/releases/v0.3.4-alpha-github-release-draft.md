# v0.3.4-alpha GitHub Release Draft

## Title

v0.3.4-alpha

## Summary

`v0.3.4-alpha` is the sandbox billing and entitlement milestone for Unlimited Skills.

This release adds public client plan and billing lifecycle diagnostics, cross-repo billing lifecycle verification, and release gates that keep billing explicitly `sandbox_only`.

## Highlights

- `plan status`, `plan explain`, and `plan doctor` provide local/cache-only plan diagnostics.
- `plan refresh` requires registration and uses hosted `/v1/hub/entitlements`.
- `billing status` and `billing doctor` provide local/cache-only billing lifecycle diagnostics.
- `billing refresh` requires registration and uses hosted `/v1/hub/billing-status`.
- Billing denial vocabulary includes `past_due`, `suspended`, and `expired`.
- Support bundle includes redacted plan and billing summaries.
- Cross-repo fixture tests verify `subscription_active`, `payment_failed`, and `subscription_canceled` through the private registry sandbox ledger and the public CLI.

## Security Boundary

Billing remains `sandbox_only`.

This release does not add:

- live payment providers;
- checkout sessions;
- checkout or payment links;
- real charges;
- invoices or refunds;
- card or bank data collection;
- live billing credentials.

The MIT local core remains unregistered. Hosted services remain registration-gated. Full catalog distribution remains disabled.
