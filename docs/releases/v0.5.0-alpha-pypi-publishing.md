# v0.5.0-alpha PyPI Trusted Publishing

Status: required unblock for `v0.5.0-alpha` publication.

The public package must be published through GitHub Actions Trusted Publishing,
not from a local long-lived PyPI token. This keeps PyPI credentials out of
developer machines and CI secrets. The workflow uses GitHub OIDC and PyPI issues
a short-lived publishing token for the approved job.

## PyPI Configuration

Configure a Trusted Publisher or Pending Trusted Publisher in PyPI with these
exact values:

| Field | Value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |

If the `unlimited-skills` project name is already owned by another publisher,
stop the release. Do not rename the package silently.

## GitHub Configuration

The workflow is `.github/workflows/publish-pypi.yml`.

Required GitHub environment:

```text
pypi
```

Recommended protection:

- Require manual approval for the `pypi` environment.
- Limit approval to release owners.
- Keep repository write access separate from PyPI release approval.

## Workflow Contract

The workflow is manual-only:

```text
workflow_dispatch
```

Required inputs:

```text
version: 0.5.0
expected_sha: <exact main commit SHA>
confirm_pypi_publish: publish unlimited-skills 0.5.0 to PyPI
```

The job validates all inputs before building or uploading. It checks out the
exact `expected_sha`, verifies that the checkout matches, runs the v0.5 package
smoke, runs the prepublish publication verifier, builds the sdist/wheel, runs
`twine check`, and publishes through:

```text
pypa/gh-action-pypi-publish@release/v1
```

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is
required by this release workflow.

## Publish Command

After PyPI Trusted Publishing and the GitHub `pypi` environment are configured,
trigger the workflow from GitHub Actions with:

```text
version = 0.5.0
expected_sha = <selected main SHA>
confirm_pypi_publish = publish unlimited-skills 0.5.0 to PyPI
```

## Required Post-Publish Evidence

After the workflow uploads the package:

```bash
python -m venv /tmp/uls-pypi-smoke
source /tmp/uls-pypi-smoke/bin/activate
python -m pip install --upgrade pip
python -m pip install --no-cache-dir unlimited-skills==0.5.0
unlimited-skills --version
unlimited-skills quickstart --json
unlimited-skills suggest "Design a REST API for a service" --json
unlimited-skills mcp savings --json
deactivate
```

Expected:

- `unlimited-skills --version` prints `unlimited-skills 0.5.0`;
- `quickstart` imports the bundled skill packs;
- `suggest` returns at least one candidate;
- `mcp savings` returns benchmark output with local paths redacted or absent.

Then run:

```bash
python scripts/verify-v050-alpha-publication.py \
  --expected-sha "<selected main SHA>" \
  --package-availability published \
  --json
```

Only after this verifier passes may the release owner create tag
`v0.5.0-alpha` and the GitHub prerelease.

## Stop Conditions

Stop the publication and do not tag if:

- PyPI Trusted Publishing is not configured;
- the workflow requests or uses a long-lived token;
- the selected SHA does not match the workflow input;
- package smoke or `twine check` fails;
- PyPI upload fails;
- clean `pip install unlimited-skills==0.5.0` fails;
- the final published-mode verifier fails.
