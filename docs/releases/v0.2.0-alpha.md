# Unlimited Skills v0.2.0-alpha

This release changes the local library model from "many visible skill packs" to a governed two-container library:

- `registry/` for bundled, hosted, community, and team-delivered collections.
- `local/` for native agent skill mirrors and user-installed local skills.

The practical goal is simple: one router skill stays visible to the agent, while the real skill library remains searchable, deduplicated, and available on demand.

## Highlights

- Added the `registry/` + `local/` library layout.
- Added deduplicated indexing by skill name, so search/list/vector counts represent semantic skills instead of every physical copy.
- Added a query-time vector sidecar fast path while keeping ChromaDB as compatibility storage.
- Made native skill sync non-destructive. Sync overlays new or changed files and does not clear existing `local/` content.
- Changed Codex installs to default to `~/.codex/.unlimited-skills/library` and patch `~/.codex/AGENTS.md` by default.
- Migrated Codex `.system` skills into the router-accessible library instead of leaving them as visible context noise.
- Changed Hermes native skill handling to `local/hermes/skills` instead of treating Hermes as a registry pack.
- Moved community, team, update, and bundled collection installs under `registry/<collection>/`.
- Added regression coverage for AGENTS patching, deduplicated indexing, and non-destructive native sync.

## Why It Matters

Earlier builds could leave multiple native skill packs visible at once or count duplicated skills from ECC, Superpowers, Hermes, and local mirrors as separate search results. That defeated the core idea of Unlimited Skills: one router, many skills, minimal context.

v0.2.0-alpha makes the library shape explicit:

```text
library/
  registry/<collection>/...   # hosted, community, team, or bundled registry packs
  local/skills/...            # Codex-local and local skill-library content
  local/<agent>/skills/...    # native agent mirrors such as Hermes, Claude Code, OpenClaw
```

The files can still exist on disk, but the index now selects one best candidate per skill name.

## Upgrade Notes

For Codex on Windows:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\install-codex.ps1
```

For Codex on macOS/Linux:

```bash
./scripts/install-codex.sh
```

After upgrading, rebuild both indexes:

```bash
unlimited-skills reindex
unlimited-skills vector-reindex
```

`vector-reindex` now creates `.unlimited-skills-vectors.json` for the normal query fast path and `.chroma-skills/` for ChromaDB compatibility. Rebuild the vector index once after upgrading so vector search does not fall back to the slower legacy Chroma query path.

Codex installs now use `~/.codex/.unlimited-skills/library` by default. Other agents keep using `~/.unlimited-skills/library` unless their installer configures a different root.

## Known Limitations

- Hosted registry access is still registration-gated early access.
- Full catalog distribution remains governed by allowlists and review status.
- Local Skill Hub LAN mode requires explicit `--allow-lan` and at least one active hub client token; serious LAN deployment still needs reverse proxy/TLS and normal network controls.
- `remote search`, `remote resolve`, and `remote view` are alpha Local Skill Hub client commands. They require a configured hub URL/token and do not query the hosted registry by default.
- Enterprise Skill Lock is documented as a planned governance feature, not implemented in this release.
- Hosted collection archives are SHA256-verified today; cryptographic signature verification remains planned.
- Warm daemon mode is still experimental and not the default retrieval path.

## Compatibility

Supported agent surfaces in this alpha:

- Codex
- Claude Code
- Hermes
- OpenClaw

v0.2.0-alpha is a layout and safety release. It is intended to make future hosted catalog, community catalog, and team sync work safer by separating registry-delivered content from local/native user content.
