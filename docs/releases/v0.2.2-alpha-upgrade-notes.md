# Upgrade Notes: v0.2.2-alpha

`v0.2.2-alpha` is an alpha upgrade from `v0.2.0-alpha` and `v0.2.1-alpha`. Install from a GitHub clone; PyPI is not the supported v0.2.x distribution path because router skills, scripts, docs, and bundled assets are repo files.

## From v0.2.1-alpha

Run:

```bash
git fetch --tags origin
git checkout v0.2.2-alpha
python -m pip install -e ".[all]"
unlimited-skills reindex
```

Expected behavior after upgrade:

- local MIT Community Core commands remain unregistered;
- `unlimited-skills serve` remains the free local daemon;
- `unlimited-skills hub serve` remains registration-gated and allowlist-only;
- hosted catalog/update/community/team/enhancement commands still require registration;
- release channel commands support signed `stable`, `beta`, and `canary` channel status;
- hosted update apply keeps rollback snapshots under `registry/.rollbacks/<collection>/`;
- service diagnostics are available through the local `service ...` commands;
- Enterprise Skill Lock local policy commands are available, but no policy is installed by default.

## From v0.2.0-alpha

Run the same Git checkout and editable install steps, then reindex:

```bash
git fetch --tags origin
git checkout v0.2.2-alpha
python -m pip install -e ".[all]"
unlimited-skills reindex
unlimited-skills doctor --json
```

The v0.2 two-container library layout remains:

- `registry/` stores hosted, community, team, and bundled packs;
- `local/` stores native agent mirrors and local skill-library content.

Upgrade must not delete existing local library files. The synthetic upgrade smoke creates a v0.2.0-style library and verifies that existing `registry/` and `local/` content survives the v0.2.2 reindex path.

## Registration And Hosted Services

Existing registration state remains local private state. The client may use a hosted-service token plus signed device proof for registered hosted calls. It must not upload local skill bodies, prompts, source code, full paths, repository paths, customer names, environment variable values, tokens, secrets, or device private keys.

Hosted remote manifests must include valid signed manifest envelopes. Archive bytes are accepted only after SHA256 verification and safe extraction. Archive-byte signatures are not implemented in this alpha.

## Enterprise Skill Lock

Enterprise Skill Lock is an opt-in local policy MVP in this release. With no installed policy, Community Core behavior is unchanged. Managed hosted policy sync, SSO, SCIM, billing, organization administration, and enterprise private-registry enforcement are not implemented in this alpha.

## Release Owner Check

Before pushing the tag, the release owner must verify the final tag target:

```bash
python scripts/verify-v0.2.2-alpha-publication.py --expected-sha <tag-target-sha>
git tag v0.2.2-alpha <tag-target-sha>
git push origin v0.2.2-alpha
```
