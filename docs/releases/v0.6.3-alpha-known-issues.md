# v0.6.3-alpha Known Issues

Status: draft.

- `apply-candidate` is dry-run only. It does not edit skill files.
- v0.6.3 can claim local diagnostics, privacy-safe candidates, and dry-run
  previews. It must not claim automatic skill improvement or non-dry-run apply.
- Candidate generation is deterministic and local, but intentionally simple:
  it groups wrong, missed, and rejected local feedback into reviewable
  candidates. Human review is still required before any skill or routing
  change.
- Existing feedback rows created before v0.6.3 may not contain the newer
  `missed` or `wrong` outcomes.
- Raw `.learning` files are local diagnostics, not support artifacts. Share
  `feedback prepare`, `learning-summary --events --json`, or the dry-run
  candidate output instead.
- Hosted catalog improvement metadata remains separate from this local Learning
  Loop and may require registration.
- Registered, Team, Business, and Enterprise v0.6.3 tier artifacts are
  docs-only / future-compatible patterns unless later implementation PRs make
  them live.
