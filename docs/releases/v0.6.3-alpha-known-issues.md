# v0.6.3-alpha Known Issues

Status: draft.

- `apply-candidate` is dry-run only. It does not edit skill files.
- v0.6.3 can claim local diagnostics, privacy-safe candidates, local tier
  artifacts, and dry-run previews. It must not claim automatic skill improvement
  or non-dry-run apply.
- Registered, Team, Business, and Enterprise are local command artifacts in this
  release. They are not hosted services, paid rollout, entitlement enforcement,
  dashboards, support SLAs, SSO/SCIM, license servers, or marketplace features.
- Enterprise evidence packs are tamper-evident through local hashes and verifier
  checks. Do not claim cryptographic signing unless a signing implementation is
  explicitly present.
- Existing feedback rows created before v0.6.3 may not contain the newer
  `missed` or `wrong` outcomes.
- Raw `.learning` files are local diagnostics, not support artifacts. Share
  `feedback prepare`, `learning-summary --events --json`, tier exports, or
  evidence packs instead.
- v0.6.2 router-health tier-debt closure is compatibility evidence only. Do not
  present it as the v0.6.3 Learning Loop value frame.
- v0.6.4 Money Saved Meter is out of scope for this release package.
- This release package does not authorize tag creation, version bump, PyPI
  publishing, yanking, marketplace submission, hosted rollout, or paid-tier
  service rollout.
