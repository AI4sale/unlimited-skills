# v0.5.3-alpha PyPI Trusted Publishing

Status: required unblock for `v0.5.3-alpha` publication. The release tag stays
blocked until Trusted Publishing uploads `unlimited-skills==0.5.3` and the
post-publish verifier passes.

## PyPI Trusted Publisher Values

| Field | Value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |
| Package version | `0.5.3` |

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is used
by the release workflow. Publishing uses GitHub OIDC through PyPI Trusted
Publishing.

## Manual Workflow Inputs

```text
version: 0.5.3
expected_sha: <selected main commit>
confirm_pypi_publish: publish unlimited-skills 0.5.3 to PyPI
```

## Pre-Publish Commands

```bash
python scripts/run-v053-alpha-package-smoke.py --json
python scripts/verify-v053-alpha-publication.py \
  --expected-sha "$(git rev-parse HEAD)" \
  --package-availability prepublish \
  --json
```

## Post-Publish Commands

Run only after the PyPI workflow completes:

```bash
python -m pip install --no-cache-dir unlimited-skills==0.5.3
unlimited-skills --version
unlimited-skills quickstart --json
unlimited-skills feedback prepare --json
python scripts/verify-v053-alpha-publication.py \
  --expected-sha "$(git rev-parse HEAD)" \
  --package-availability published \
  --json
```

Only the published verifier may unlock the `v0.5.3-alpha` tag.

## Boundaries

- No marketplace submission happens inside this release PR.
- No paid/support/SLA claim is added.
- No hosted/team/enterprise readiness claim is added.
- No telemetry, analytics SDK, tracking pixel, private social scraping, prompt
  collection, tool input collection, tool output collection, or hosted query
