# v0.4.6-alpha Known Issues

`v0.4.6-alpha` is an alpha integration gate for the MCP performance benchmark
pack.

## Intentional Limitations

- The benchmark runner is fixture-only. It does not benchmark customer MCP
  servers, hosted gateway mode, OAuth, remote upstreams, MCP resources, or MCP
  prompts.
- Reference numbers are environment-specific. They are useful for regression
  tracking and relative comparisons, not universal performance guarantees.
- Memory reporting is best-effort and covers the gateway process only; spawned
  fixture upstream subprocesses may not be included on every platform.
- The warm-start optimization plan is design only. Persistent index cache,
  opt-in pre-spawn, and import-graph slimming are not implemented in this gate.
- No runtime default changes are included.
- No production hosted calls, hosted gateway, automatic telemetry, full catalog
  distribution, auto-publish, live billing, or PyPI publication are included.
- Codex must not create or push `v0.4.6-alpha` from this integration gate.

## Operator Notes

- Benchmark reports are written to `build/perf/`, which is gitignored.
- Use medians and raw samples together; single Windows cold-start samples can
  be noisy.
- The schema-validated JSON report is the stable artifact for comparison.
