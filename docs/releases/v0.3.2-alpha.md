# v0.3.2-alpha

Status: private team packs integration gate.

`v0.3.2-alpha` integrates the registered private team pack client, setup diagnostics, service diagnostics, support bundle reporting, and cross-repo fixture validation. The public MIT local core remains usable without registration.

## Highlights

- Package version is `0.3.2`.
- GitHub clone remains the supported alpha distribution path.
- PyPI remains deferred because router skills, scripts, docs, schemas, and fixture assets must be present beside the checkout.
- Private team pack commands can list, preview, install, sync, inspect installed packs, and remove registry-owned private packs.
- Setup, service diagnostics, doctor, and support bundle surfaces include redacted private-pack counters only.
- Cross-repo private pack E2E runs against a local fixture registry and does not call production hosted services.

## Security Boundary

- MIT local commands remain unregistered.
- `unlimited-skills serve` remains unregistered.
- Hosted private team pack list, preview, manifest, download, and sync require registration and device proof.
- Private team pack access also requires a private-pack entitlement or Business/Enterprise plan on the registry side.
- Private pack manifests must verify against trusted Ed25519 keys scoped to `private-team-pack`.
- Private pack archives are downloaded through proofed POST endpoints, SHA256-verified, and safely extracted under `registry/private/<pack_id>`.
- `private-packs remove` deletes only registry-owned paths recorded in local private-pack metadata.
- Support and diagnostics do not expose private pack names by default, private skill names, private skill bodies, archive URLs, local paths, tokens, proofs, or private keys.
- No private registry skill bodies, signing private keys, hosted tokens, or device private keys are shipped in the public repo.
