# v0.3.8-alpha

Status: integration gate for skill evaluations and catalog quality scoring.

`v0.3.8-alpha` proves the next catalog quality layer end to end: private-registry skill evaluations produce quality signals, and the public client displays signed quality/evaluation status without uploading user prompts, task text, skill bodies, or customer data.

## Scope

- Fixture-safe skill evaluation cross-repo E2E.
- Public quality/eval status commands from the catalog-quality client.
- Browse/search quality metadata proof.
- Low-score install warning proof.
- Blocked hosted item refusal proof.
- Support bundle redaction boundary for quality/eval summaries.

## Security Boundary

- No automatic telemetry.
- No prompt/task text upload.
- No skill-body leakage.
- No untrusted script execution.
- No automatic skill rewriting.
- No auto-publish.
- No production hosted calls in tests.

## Release Owner Gate

Before this integration milestone can become a publication candidate, run:

```bash
python scripts/run-skill-evals-cross-repo-e2e.py --fixture-mode --temp-home --json
python -m pytest tests/integration/test_skill_evals_cross_repo_e2e.py -q
python scripts/run-v0.3.8-alpha-skill-evals-smoke.py --fixture-mode --temp-home
python scripts/verify-v0.3.8-alpha-skill-evals.py --expected-sha <tag-target-sha>
```

If the private registry checkout is available:

```bash
python scripts/run-skill-evals-cross-repo-e2e.py --registry-repo ../unlimited-skills-registry --temp-home --json
```

This is not a final tag gate. Keep the PR in review until release, catalog, security, and privacy reviewers inspect the evidence.
