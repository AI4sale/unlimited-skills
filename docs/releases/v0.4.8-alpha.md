# v0.4.8-alpha

Status: final publication gate for the managed MCP profile trust store.

`v0.4.8-alpha` integrates the E15 local trust-store layer with the E14 signed
MCP profile bundle verifier. The gate proves that the managed store is only an
offline management layer over the existing E14 `trusted-keys.json` and
`crl.json` formats. Verification semantics still live in the signed bundle
loader.

The integration gate is complete. This final publication gate verifies the
selected `main` SHA and permits Codex to create and push `v0.4.8-alpha` only
after the final publication verifier passes.

## Integrated Work

- Public PR #107 added the managed MCP profile trust store and
  `unlimited-skills mcp trust status|list|import|revoke|doctor`.
- Public PR #108 integrated the trust-store release evidence with the v0.4.8
  alpha manifest, smoke runner, integration test, and docs.
- The trust store lives under `<library root>/.unlimited-skills-trust/` unless
  `--store-dir` is used.
- `trust import` accepts public Ed25519 keys only and refuses private-looking
  material before writing anything.
- `trust list` and `trust status` show key state and abbreviated fingerprints
  without full key bytes.
- `trust revoke` appends key ids or bundle SHA-256 values to the local CRL and
  keeps revocation history append-only.
- `trust doctor` performs an offline self-check and fails closed on malformed
  trusted keys, corrupt trust store files, unreadable CRLs, and unexplained
  revocations.
- `mcp gateway --profile-bundle` defaults to the managed trusted-keys file when
  `--trusted-keys` is omitted and the managed file exists. Explicit
  `--trusted-keys` always wins.

## Security Boundary

- Alpha only; behavior may break before v0.6.
- Fixture-only integration evidence.
- The trust store is local and offline.
- Public keys only.
- No hosted trust fetch.
- No registry sync.
- No production signing keys.
- No private key storage.
- No OAuth implementation.
- No remote upstream implementation.
- No MCP resources.
- No MCP prompts.
- No production hosted calls.
- No full catalog distribution.
- No auto-publish.
- No live billing.
- No PyPI publication.
- Raw local profiles may still work when policy allows them.
- Missing trust store behavior remains fail-closed for signed bundle
  verification with `bundle_key_missing`.
- Wrong scope and wrong audience behavior remains enforced by the signed bundle
  verifier; the trust-store metadata sidecar grants no authority.
