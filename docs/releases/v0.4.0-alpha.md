# v0.4.0-alpha

Status: final publication gate for the first SkillOps platform foundation layer.

`v0.4.0-alpha` is the SkillOps foundation milestone for Unlimited Skills. It proves that the public client and private registry contracts can coordinate recommendation preview, release evaluation, maintainer queue status, and governance summaries without crossing the local-first or metadata-only boundaries.

This publication package prepares the release notes, manifest, upgrade notes, known issues, smoke runner, and verifier. Codex must not create or push the final tag automatically; the release owner runs the human tag command after the publication PR is merged and verified on the selected `main` SHA.

## Integrated Epics

- E01: policy-aware recommendation runtime preview.
- E02: eval release operator workflow.
- E03: maintainer queue runtime API and public status client.
- E04: governance dashboard signed summary API.

## Scope

- Cross-repo E2E runner for E01-E04.
- Fixture mode that works without a private registry checkout.
- External local-registry mode that verifies private registry E02, E03, and E04 validators when a checkout is available.
- Final publication release documentation for the v0.4.0-alpha alpha milestone.
- Release manifest with human tag approval policy.
- Upgrade notes, known issues, release smoke runner, and publication verifier.
- Draft GitHub release notes for the release owner.

## Security Boundary

- Alpha only.
- No production rollout.
- No automatic telemetry.
- No prompt upload.
- No skill body upload.
- No automatic hosted query forwarding.
- No automatic rewriting.
- No automatic install, update, or remove.
- No auto-publish.
- No live billing.
- No PyPI.
- No full catalog distribution.
- MIT local core remains registration-free.
- Signed hosted manifests remain required.
- No private registry content is committed to the public repository.

## Human Tag Command

After this PR is merged and the release owner has verified the final `main` SHA:

```powershell
git checkout main
git pull --ff-only origin main
$sha = git rev-parse HEAD
python scripts/verify-v040-alpha-publication.py --expected-sha $sha
git tag -a v0.4.0-alpha $sha -m "v0.4.0-alpha"
git push origin v0.4.0-alpha
```
