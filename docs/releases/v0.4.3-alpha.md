# v0.4.3-alpha

Status: MCP upstream enforcement integration gate.

`v0.4.3-alpha` integrates the E08 runtime enforcement layer for the Unlimited
Tools MCP gateway after `v0.4.2-alpha` publication. The release proves the
upstream security model is now enforced by runtime code, not only described in
the E07 design contract.

This is an integration gate, not a production hosted rollout and not a new
runtime feature beyond public PR #89.

## Integrated Work

- Public PR #90 published and tagged the `v0.4.2-alpha` MCP baseline.
- Public PR #89 added E08 MCP upstream enforcement for local stdio upstreams.
- Disabled upstream refusal: `enabled: false` and `trust_level: disabled`
  are never indexed, never spawned, and addressed calls fail with
  `UPSTREAM_DISABLED` / `-32005`.
- `future-remote-placeholder` refusal: future remote configs are never
  indexed, never spawned, and all I/O is refused with
  `TRUST_LEVEL_VIOLATION` / `-32010` until the OAuth/remote gate opens.
- Command allowlist enforcement: shell commands, relative commands, unknown
  bare commands, and unsafe temp paths are refused with
  `COMMAND_NOT_ALLOWED` / `-32006`.
- Environment forwarding enforcement: only names in `env_allowlist` are copied
  from the local environment; wildcard forwarding and the removed v1 literal
  `env` map are refused with `ENV_FORWARDING_DENIED` / `-32007` or config-load
  errors.
- Size enforcement: oversize tool schemas fail with `SCHEMA_TOO_LARGE` /
  `-32008`, oversize tool results fail with `RESPONSE_TOO_LARGE` / `-32009`,
  and neither path truncates or embeds payload content in the refusal.
- Startup timeout and request timeout hard bounds are enforced at config load
  and direct runtime construction.
- Audit cannot be disabled. Minimal audit still records success/refusal
  observability, standard audit redacts arguments and path-scrubs errors, and
  audit rotation caps retained local JSONL generations.

## Security Boundary

- Alpha only.
- Local stdio gateway only.
- Fixture-mode verification only.
- No production rollout.
- No production hosted calls.
- No hosted gateway.
- No automatic telemetry.
- No OAuth implementation.
- No remote upstream implementation.
- No MCP resources.
- No MCP prompts.
- No resources or prompts support.
- No shell execution.
- No arbitrary shell execution.
- No automatic hosted query forwarding.
- No automatic rewriting.
- No automatic install, update, remove, reindex, or publish action.
- No auto-publish.
- No live billing.
- No PyPI.
- No full catalog distribution.
- No full MCP tool schema dump.
- No prompt, task text, search query, skill body, local path, repository path,
  token, proof, private key, customer data, or private registry content is
  uploaded by this release gate.
- MCP v1 schemas/configs are alpha and may break before v0.6.
