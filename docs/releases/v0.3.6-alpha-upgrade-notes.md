# v0.3.6-alpha Upgrade Notes

`v0.3.6-alpha` is a GitHub-clone alpha release. PyPI remains deferred because the checkout must include router skills, installers, schemas, docs, scripts, and release smoke assets.

## Upgrade Path

1. Pull or clone the release checkout.
2. Reinstall the router for the agent surface you use.
3. Reindex the local library.
4. Register only if you need hosted catalog browser operations.

Example:

```bash
git fetch origin
git checkout v0.3.6-alpha
python -m pip install -e ".[all]"
unlimited-skills reindex
unlimited-skills catalog browse --source community --compatible-agent codex
```

## What Changes

- `catalog browse`, `catalog search`, `catalog filters`, `catalog preview`, and `catalog install --dry-run` provide registered signed metadata discovery.
- Catalog browser responses must be signed by trusted manifest keys.
- Preview remains metadata-only.
- Dry-run install writes no files.
- Community-source installs delegate to the Community Skills install flow only after signed browser metadata verification.

## What Does Not Change

- MIT local commands remain unregistered.
- `serve` remains the free local daemon.
- Full catalog distribution remains disabled.
- Official and private-visible catalog browser installs remain metadata/dry-run only until dedicated install-plan capability checks are implemented.
- No marketplace storefront, billing, checkout, payment, revenue-share, or PyPI distribution is introduced by this release.

## Verification

Before tagging, the release owner should run:

```bash
python scripts/run-v0.3.6-alpha-release-smoke.py
python scripts/verify-v0.3.6-alpha-publication.py --expected-sha <tag-target-sha>
```
