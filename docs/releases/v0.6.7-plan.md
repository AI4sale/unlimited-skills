# v0.6.7 business context retrieval plan

Status: implementation and release execution

## Capability

One router call can return both the best task skill and relevant, cited
business reference data from an owner-configured provider. The public core
never names or imports a private knowledge system: it ships only a generic,
opt-in JSON-over-stdio client contract. The public core never names or depends
on a private knowledge system.

This release is retrieval-only. Raw assistant prose, PR-looking tokens, URLs,
hashes, and test counts are not accepted completion receipts. Automatic Stop
write-back remains disabled until a later release introduces structured,
independently verifiable acceptance receipts and post-write visibility proof.

## Audit defects addressed

| Priority | Defect | Release invariant |
| --- | --- | --- |
| P0 | Skill routing and company recall were separate manual actions | one `suggest --card` result carries both channels when an owner opts in |
| P0 | Retrieved text could close its own trust delimiter | provider-controlled text is escaped and adversarially tested |
| P0 | Raw internal data was allowed by default | defaults permit only `public` and `internal-sanitized`; raw `internal` requires explicit owner opt-in |
| P0 | Model-written prose could be mistaken for accepted work | the Stop hook never submits prose; structured receipts are a later contract |
| P0 | The exact-wheel gate did not execute the headline provider capability | installed-wheel doctor/retrieve/card/fail-open/kill-switch cases gate publishing |
| P1 | Filtered prefix rows could hide eligible later rows | item limiting happens after policy filtering |
| P1 | A missing lexical manifest forced a full rescan on every prompt | hooks trigger one detached repair and preserve local skills |
| P1 | Skill card and memory text had independent caps | the final hook context has one aggregate budget |
| P1 | PyPI JSON can lead the simple index during propagation | retries cover only that propagation race, never real smoke failures |

## Public contract

- Provider config is absent and silent by default.
- Commands are argument arrays and run with `shell=False`, bounded time/output,
  and a minimal environment.
- Source references must be relative and non-traversing; sensitivities must be
  explicitly allowed.
- Default sensitivities are `public` and `internal-sanitized`. Selecting raw
  `internal` explicitly means the configured agent model will receive it.
- Retrieved excerpts are escaped, cited, and labelled reference data, not
  instructions or authority for external action.
- Automatic retrieval runs only on the card/hook channel. Ordinary text and
  non-card JSON suggest contracts do not change.
- `no_context` never becomes verified absence. Provider failure permits only
  generic work, not company-specific guessing.
- `context retrieve` and `context doctor` expose the same contract to operators.
- `context completion-candidate` remains an experimental manual transport
  surface; no bundled hook calls it in 0.6.7.
- A missing, incompatible, slow, or malformed provider fails open.

## Private provider boundary

The owner implementation must enforce server identity, project/entity walls,
sensitivity, ranking, freshness, and provenance. A local provider should be a
thin client so the same envelope can move to a client/server Company Memory
without changing the public router contract.

## Non-goals

- No automatic completion learning in 0.6.7.
- No hosted dependency, telemetry, prompt upload, or automatic remote sync.
- No bundled company adapter, private path, database, secret, or knowledge body.
- No deletion, replacement, or migration of `library/local`.
- No new paid-tier, SLA, or guaranteed-outcome claim.

## Release gates

1. Provider schema, escaping, sensitivity default, source filtering, timeout,
   environment, aggregate context budget, and kill-switch tests pass.
2. A private thin-client adapter proves business walls, loopback-only transport,
   daemon identity, POST JSON retrieval, and disabled prose write-back.
3. Full pytest, compile, privacy/security scans, frozen contracts, wheel/sdist
   smoke, exact upgrade smoke, and installed-wheel provider E2E pass.
4. Independent review returns GO on the final public/private delta.
5. Public PR CI passes; exact `main` is published through Trusted Publishing;
   public artifacts are independently clean-installed and verified.
6. The installed router is refreshed, reindexed, and proves skill plus cited
   Company Memory delivery against the live thin-client provider.

Rollback is a new patch release; published PyPI artifacts remain immutable.
