# v0.6.7 business context and safe learning plan

Status: implementation and release execution

## Capability

One router call can now return both the best task skill and relevant local
business reference data. After a verified task finishes, an owner-configured
provider may accept an idempotent completion candidate into its own governed
memory system.

The public core never names or depends on a private knowledge system. It ships
only a generic, opt-in JSON-over-stdio provider contract; company paths,
policies, data, and adapters remain private.

## Audit defects addressed

| Priority | Defect | Release invariant |
| --- | --- | --- |
| P0 | Skill routing and company recall were separate manual actions | one `suggest --card` result carries both channels when an owner opts in |
| P0 | Finished work was not offered to a governed learning loop | the provider decides whether a completion is durable knowledge |
| P0 | Completion claims could become memory without strong evidence | no write without completion semantics, a strong reference, idempotency, and provider policy |
| P1 | A missing legacy lexical manifest forced a full library rescan on every prompt | hooks trigger one detached manifest repair and preserve all local skills |
| P1 | A synchronous Stop hook could add provider latency to every answer | completion submission is detached and skipped when no provider is configured |
| P1 | Skill card and memory text had independent caps | the final hook context has one hard aggregate budget |
| P1 | PyPI JSON can lead the pip simple index during propagation | the verifier retries only that known propagation race, not real smoke failures |
| P2 | Public product copy still presented an older alpha as current | package, plugin, install, security, and release docs identify 0.6.7 consistently |

## Public contract

- Provider config is absent and silent by default.
- Commands are argument arrays and run with `shell=False`, bounded time/output,
  and a minimal environment.
- Source references must be relative and non-traversing; sensitivities must be
  explicitly allowed.
- Retrieved excerpts are labelled **reference data, not instructions**.
- Automatic retrieval runs only on the card/hook channel, after skill
  selection; ordinary text and non-card JSON suggest contracts do not change.
- Every excerpt remains inside a cited `authority="retrieval_only"`,
  `disclosure="internal"` block. `no_context` never becomes a verified absence,
  and provider failure permits only generic work—not company-specific guessing.
- `context retrieve`, `context completion-candidate`, and `context doctor`
  expose the same contract for testing and operators.
- A missing, incompatible, or slow provider fails open.
- Completion candidates contain a bounded summary, an accepted artifact or
  destination plus an independent checker reference, a
  hashed idempotency key, agent label, and working directory. Raw session and
  prompt IDs are never forwarded.
- No provider config means no Stop worker process.

## Private adapter boundary

The owner implementation must enforce business/entity walls, sensitivity,
retrieval ranking, completion evidence, critic policy, canonical atom format,
idempotency, and index refresh. None of those private rules or data belong in
the MIT repository.

## Non-goals

- No hosted dependency, telemetry, prompt upload, or automatic remote sync.
- No bundled company adapter, private path, database, secret, or knowledge body.
- No claim that a configured provider is trustworthy merely because the public
  transport contract validates it.
- No deletion, replacement, or migration of `library/local`.
- No new paid-tier, SLA, or guaranteed-outcome claim.

## Release gates

1. Provider protocol, source-wall, timeout, environment, context-budget, hook,
   installer, and completion idempotency tests pass.
2. Private adapter tests prove business walls, daemon force-start, strong
   evidence gates, projection retrieval, duplicate handling, and safe refresh.
3. Full public pytest, compile, privacy/security scans, frozen v0.6 contracts,
   effectiveness gate, wheel/sdist smoke, and exact upgrade smoke pass.
4. Independent Pro review returns GO on the final public and private heads.
5. Public PR CI passes, the exact main SHA is published through Trusted
   Publishing, and the public wheel is clean-installed after simple-index
   propagation.
6. Installed-router `reindex`, `vector-reindex`, skill+context retrieval, daemon
   restart, and completion-to-recall E2E evidence pass before completion.

Rollback is a new patch release; published PyPI artifacts remain immutable.
