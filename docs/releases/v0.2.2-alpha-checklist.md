# Unlimited Skills v0.2.2-alpha Release Checklist

Use this checklist after the stacked registry and public release PRs are reviewed and before tagging `v0.2.2-alpha`.

## Required Merge Order

1. Private registry PR #2: signed registry manifest publisher.
2. Public PR #13 through PR #20: v0.2.1-alpha runtime, trust, lifecycle, and finalization baseline.
3. Private registry PR #3: `feat/staging-signed-registry-api-v1`
4. Public PR #21: `test/staging-signed-registry-e2e-v1`
5. Public PR #22: `feat/hub-plan-heartbeat-entitlement-sync-v1`
6. Private registry PR #4: `feat/registry-production-api-mvp-v1`
7. Public PR #23: `test/production-registry-contract-e2e-v1`
8. Private registry PR #5: `feat/registry-release-channels-rollback-v1`
9. Public PR #24: `feat/client-channel-pinning-update-rollback-v1`
10. Public PR #25: `release/v0.2.2-alpha-final-gate`
11. Private registry PR #6: `feat/production-registry-deployment-ops-v1`
12. Public PR #26: `feat/production-registry-onboarding-diagnostics-v1`
13. Public PR #27: `feat/enterprise-skill-lock-policy-mvp-v1`
14. Publication gate PR: `release/v0.2.2-alpha-publication`

## Release Gate

- [ ] GitHub Actions CI is green on the release-gate PR.
- [ ] Full public pytest suite is green.
- [ ] v0.2.x smoke suite is green.
- [ ] Staging signed registry E2E is green in fixture mode.
- [ ] Production registry contract E2E is green in fixture mode.
- [ ] Cross-repo smoke is green in fixture mode: `python scripts/run-v0.2.2-alpha-cross-repo-smoke.py --fixture-mode --temp-home`.
- [ ] Release verifier is green: `python scripts/verify-v0.2.2-alpha-release.py`.
- [ ] Publication verifier is green: `python scripts/verify-v0.2.2-alpha-publication.py`.
- [ ] Fresh install smoke is green: `python scripts/run-v0.2.2-alpha-fresh-install-smoke.py`.
- [ ] Upgrade smoke from a synthetic v0.2.0 layout is green: `python scripts/run-v0.2.2-alpha-upgrade-smoke.py`.
- [ ] `python -m compileall -q unlimited_skills scripts tests` is green.
- [ ] `git diff --check` is green.
- [ ] Private registry pytest suite is green.
- [ ] Private signed manifest validator is green.
- [ ] Private production registry verifier is green in fixture mode.
- [ ] Tests use temporary HOME and temporary Unlimited Skills roots.
- [ ] Tests make no production hosted registry calls by default.
- [ ] No raw hub tokens, registration tokens, device private keys, signing private keys, or API tokens are present in release artifacts or CI logs.
- [ ] Full catalog distribution remains disabled.
- [ ] Local Skill Hub distribution remains allowlist-only.
- [ ] Hosted remote manifests require valid signed manifest envelopes.
- [ ] Release-channel status uses the `release-channels` signed manifest scope.
- [ ] Release channel naming is fixed to `stable`, `beta`, and `canary`; no `dev` channel remains in release-channel docs/tests/config.
- [ ] Archive bytes still require SHA256 verification and safe extraction.
- [ ] `unlimited-skills serve` remains MIT/free/unregistered.
- [ ] `unlimited-skills hub serve` remains registration-gated.
- [ ] Remote CLI search/resolve/view are documented as alpha Local Skill Hub client commands, not hosted registry search.
- [ ] Enterprise Skill Lock is documented as local policy MVP, with managed hosted policy sync and enterprise org controls still out of scope.
- [ ] Machine-readable release manifest exists and is valid: `docs/releases/v0.2.2-alpha.release-manifest.json`.
- [ ] Machine-readable release manifest has no legacy placeholder SHA.
- [ ] Release owner has checked `git.sha` against the final tag target with `python scripts/verify-v0.2.2-alpha-publication.py --expected-sha <tag-target-sha>`.
- [ ] Release notes reviewed.
- [ ] Security notes reviewed.
- [ ] Tag approval received from the release owner.

## Tagging

```bash
git tag v0.2.2-alpha
git push origin v0.2.2-alpha
```

Create the GitHub release only after the release owner approves the tag.
