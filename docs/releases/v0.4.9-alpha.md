# v0.4.9-alpha

Status: MCP profile rollout simulator integration gate.

`v0.4.9-alpha` integrates the E16 local rollout simulator and policy doctor
with the previous E10 profile, E14 signed bundle, and E15 managed trust-store
layers. The gate proves that users can preview profile/bundle/trust-store
effects before starting the gateway.

This is an integration gate, not the final publication gate. Codex must not
create or push `v0.4.9-alpha` from this PR.

## Integrated Work

- Public PR #110 added `unlimited-skills mcp profiles rollout-plan|doctor`.
- `rollout-plan` reports raw profile rollout plan counts, signed bundle
  rollout plan provenance, trust-store-backed rollout plan behavior, visible /
  hidden / callable / view-only / refused tool counts, upstreams that would
  never spawn, profile inheritance narrowing, E14 verification result, and the
  audit row fields that would be emitted.
- `doctor` reports missing trust store, corrupt trust store, expired key,
  revoked key, wrong audience, namespace violation, hide-all-tools, shadowed
  tool, signed-required unsigned-source, and other rollout findings.
- The fixture smoke proves no upstream spawn, no network, and no mutation.
