# v0.3.6-alpha Known Issues

`v0.3.6-alpha` is a developer preview, not a production SLA.

- Hosted registry access is early-access and may be unavailable during alpha testing.
- Catalog browser commands require hosted `/v1/catalog/browser/*` support and registration.
- Official and private-visible catalog browser items are metadata/dry-run only until dedicated install-plan capability checks are implemented.
- Catalog browser fixture tests use local test signing keys. Production signing status remains `blocked_no_production_signing_key` until the release owner records `production_signed` or an explicit override.
- Full catalog distribution remains disabled.
- Local Skill Hub remains allowlist-only.
- PyPI is not the supported distribution path.
- No marketplace storefront and no billing flow are enabled by this release.
- The public MIT repo does not publish private registered catalog skill bodies.
- Support bundles are local redacted diagnostics; users decide whether to share them.
