# v0.3.9-alpha Upgrade Notes

`v0.3.9-alpha` is the skill improvement workflow milestone. It does not change the local MIT core requirement, and it does not require registration for local search, list, view, use, reindex, native sync, self-update, local daemon, or local diagnostics commands.

## From v0.3.8-alpha

1. Pull the public repo release branch or tag after release-owner approval.
2. Run local smoke checks:

```powershell
.\.venv\Scripts\python.exe scripts/run-v0.2x-smoke-tests.py
.\.venv\Scripts\python.exe scripts/run-v0.3.9-alpha-release-smoke.py
```

3. Registered users can inspect signed improvement metadata:

```powershell
unlimited-skills catalog improvement-status community:browser-qa-pack:0.1.0
unlimited-skills catalog known-issues community:browser-qa-pack:0.1.0
unlimited-skills catalog update-recommendations --json
unlimited-skills catalog update-preview community:browser-qa-pack:0.1.0
```

4. Recommendations are preview-only. They do not download, install, update, remove, rewrite, reindex, sign, publish, or promote skills.

## Privacy Boundary

Skill improvement status is metadata-only. The client must reject or exclude prompts, task text, skill bodies, local paths, repo paths, customer data, tokens, device proofs, private keys, checkout URLs, archive URLs, and payment links.

Support bundles include aggregate improvement counters only. They do not include item ids, issue titles, raw recommendations, prompts, skill bodies, private pack inventory, or private paths.

## v0.4 Boundary

This release includes the v0.4 readiness audit as a NO-GO planning record. It does not authorize v0.4 runtime implementation, live billing, PyPI publication, full catalog distribution, automatic telemetry, automatic hosted query forwarding, automatic skill rewriting, or auto-publish.
