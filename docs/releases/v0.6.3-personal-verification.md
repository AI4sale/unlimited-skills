# v0.6.3 Personal Verification Procedure

This procedure is for an owner/operator validating `v0.6.3-alpha` locally before
or during release execution. It verifies runnable commands, package build, clean
install, and post-release checks, not only review documents.

## 1. Start From A Clean Checkout

```powershell
git checkout main
git pull --ff-only
git status --short
```

Expected: `git status --short` prints nothing.

Confirm the intended release version state:

```powershell
python -c "import tomllib; print(tomllib.load(open('pyproject.toml','rb'))['project']['version'])"
python -c "import unlimited_skills; print(unlimited_skills.__version__)"
```

Expected: both commands print `0.6.3`.

## 2. Run The One-Command Smoke

```powershell
python scripts/verify-v063-tier-release-smoke.py --json
```

Expected:

- `ok` is `true`.
- `release` is `v0.6.3-alpha`.
- `rows` include Free, Registered, Team, Business, Enterprise, and v0.6.2
  router-health compatibility checks.
- `privacy.no_egress_asserted` is `true`.
- `mutation.apply_candidate_dry_run_only` is `true`.
- Work artifacts are written under `.tmp/v063-tier-release-smoke`.

## 4. Run Each Learning Loop Tier Command Manually

Create a fixture library:

```powershell
$root = "$PWD\.tmp\v063-manual-library"
Remove-Item -Recurse -Force $root -ErrorAction SilentlyContinue
New-Item -ItemType Directory -Force "$root\local\skills\python-patterns" | Out-Null
@"
---
name: python-patterns
description: Python implementation patterns.
---

# python-patterns
"@ | Set-Content -Encoding utf8 "$root\local\skills\python-patterns\SKILL.md"
unlimited-skills --root $root reindex
unlimited-skills --root $root feedback record python-patterns --verdict wrong
unlimited-skills --root $root feedback record python-patterns --verdict missed
unlimited-skills --root $root feedback record python-patterns --verdict rejected
```

Run Free/base commands:

```powershell
unlimited-skills --root $root learning doctor
unlimited-skills --root $root improvement-candidates
unlimited-skills --root $root apply-candidate --dry-run <candidate-id>
```

Expected:

- `learning doctor` reports feedback and candidates.
- `improvement-candidates` emits privacy-safe candidate JSON.
- `apply-candidate --dry-run` reports `written=false` and `mutated_files=[]`.

Run above-Free local artifact commands:

```powershell
$out = "$PWD\.tmp\v063-manual-artifacts"
New-Item -ItemType Directory -Force $out | Out-Null
unlimited-skills --root $root learning export --out "$out\registered-learning-export.json" --json-status
unlimited-skills --root $root learning team-rollup --input "$out\registered-learning-export.json" --out "$out\team-learning-rollup.json" --json-status
unlimited-skills --root $root learning admin-export --input "$out\team-learning-rollup.json" --json "$out\business-learning-admin.json" --csv "$out\business-learning-admin.csv"
unlimited-skills --root $root learning evidence-pack --input "$out\business-learning-admin.json" --out "$out\enterprise-learning-evidence-pack"
unlimited-skills --root $root learning verify-evidence-pack --input "$out\enterprise-learning-evidence-pack" --json
```

Expected:

- Registered export JSON exists.
- Team rollup JSON exists.
- Business JSON and CSV exist.
- Enterprise evidence pack contains `manifest.json`, `privacy-proof.json`,
  `non-mutation-proof.json`, and `schema-version-proof.json`.
- Evidence pack verification returns `ok=true`.

## 5. Verify Tamper Fail-Closed

```powershell
$tampered = "$out\enterprise-learning-evidence-pack-tampered"

unlimited-skills --root $root learning verify-evidence-pack --input $tampered --json
```

Expected: command exits non-zero and reports `ok=false`.

## 10. Trusted Publishing And Post-Publish Verification

Use `docs/releases/v0.6.3-alpha-pypi-publishing.md`. Dispatch the PyPI workflow
only after C063-REL-02 is merged, Opus O063-REL-FINAL-R1 returns PASS, and all
pre-publish checks pass on the selected SHA.

After PyPI upload, install `unlimited-skills==0.6.3` in a clean environment and
run the post-publish commands from the PyPI publishing doc. Create
`v0.6.3-alpha` only after the clean PyPI install and post-publish verifier pass.
