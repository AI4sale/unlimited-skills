# v0.4.4-alpha

Status: final MCP permissioned tool profile alpha publication.

`v0.4.4-alpha` integrates the E10 runtime enforcement layer for permissioned
MCP tool profiles after the `v0.4.3-alpha` publication tag. It proves that the
Unlimited Tools gateway can enforce visible/callable tool boundaries in fixture
mode without enabling OAuth, remote upstreams, resources, prompts, hosted
gateway mode, production hosted calls, or automatic telemetry.

This is the final publication gate. It does not add runtime behavior beyond the
already-merged E10 PR. Codex may create and push the `v0.4.4-alpha` tag only
after the publication verifier passes on the selected `main` SHA.

## Integrated Work

- Public PR #94 published and tagged `v0.4.3-alpha`.
- Public PR #93 added E10 runtime enforcement for permissioned MCP tool
  profiles.
- Public PR #96 integrated the v0.4.4-alpha release evidence, verifier, smoke,
  release docs, and CI proof.
- Default-deny profiles hide every tool until a profile explicitly allows it.
- `tools_search` returns only visible tools and marks view-only tools as not
  callable.
- `tools_schema` refuses hidden tools with `TOOL_NOT_VISIBLE` / `-32011`
  before existence checks or upstream spawn.
- `tools_call` refuses visible-but-not-callable tools with
  `TOOL_NOT_CALLABLE` / `-32012`.
- Missing selected profiles fail closed with `PROFILE_NOT_FOUND` / `-32013`.
- Invalid profile files fail closed with `PROFILE_INVALID` / `-32014`.
- `extends` inheritance is restriction-only; children can narrow but never
  widen parent grants.
- Audit rows carry the active profile name, and startup records a
  `profile_loaded` row with the profile file SHA-256.

- No full catalog distribution.
