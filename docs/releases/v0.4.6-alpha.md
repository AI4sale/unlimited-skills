# v0.4.6-alpha

Status: MCP performance benchmark integration gate.

`v0.4.6-alpha` integrates the E12 MCP performance benchmark pack after the
`v0.4.5-alpha` publication tag and the PR #99 benchmark pack merge. It proves
that operators can measure Unlimited Tools MCP gateway cold start, spawn vs
reuse, schema indexing, `tools_search`, audit overhead, context bytes, and
best-effort gateway RSS in fixture-only mode.

This is an integration gate, not a final publication gate. Codex must not
create or push the `v0.4.6-alpha` tag from this PR. A later final publication
gate must verify the selected `main` SHA before tagging.

## Integrated Work

- Public PR #100 published and tagged `v0.4.5-alpha`.
- Public PR #99 added the E12 MCP performance benchmark pack and warm-start
  optimization plan.
- `scripts/run-mcp-performance-benchmarks.py --fixture-mode --json` produces a
  JSON report validated by `schemas/mcp-perf-report.schema.json` and a Markdown
  report under `build/perf/`.
- The benchmark pack records raw samples and medians for cold start, first
  touch lazy spawn, reused `tools_schema`/`tools_call`, `tools_search`, schema
  indexing, audit overhead, context bytes, and best-effort memory.
- `docs/mcp-performance.md` documents measured reference tables and the
  warm-start optimization plan.
- The warm-start optimization plan is design only: persistent tool-index cache,
  opt-in pre-spawn, and import-graph slimming are not implemented in this gate.
