# v0.6.4-alpha Known Issues

Status: post-publish emergency correction.

## Money Saved Meter Does Not Compute Dollars

Known issue: `v0.6.4` reports context/token proxy savings only; it does not
compute dollar savings.

The shipped `money-saved` commands keep dollar value disabled by default and do
not bind a measurement to a provider/model price table. They also do not
separate skill-body savings from MCP schema savings in the Free meter output.
This means the released package can honestly report local counts, measured MCP
schema bytes, and estimated tokens, but it cannot honestly answer "how much
money was saved" yet.

Required hotfix direction:

- bind reports to an explicit model/provider/currency/price basis before
  computing money;
- keep skill savings and MCP savings separate in the JSON contract;
- compute estimated input money saved from estimated input tokens avoided and a
  public or locally configured input-token price;
- keep output-token savings at zero unless a future artifact measures output
  reduction;
- propagate compatible money basis through Registered, Team, Business, and
  Enterprise artifacts;
- make Enterprise verification recompute the formula and fail closed on model,
  price, formula, or total tampering.

- Money Saved Meter tokens are estimates, not measured provider billing tokens.
- Dollar value is disabled by default and must not be presented as exact money
  saved.
- The 100-call frame is reporting cadence, not billing math.
- Local tier artifacts are files produced on the user's machine. v0.6.4 does not
  include hosted dashboards, live sync, automatic upload, hosted telemetry,
  SSO/SCIM, billing reconciliation, or hosted governance.
- Public PR #195 remains HOLD/stale/superseded-needed and is not part of this
  release package.
- #119/E19 remains parked and is not part of v0.6.4-alpha.
