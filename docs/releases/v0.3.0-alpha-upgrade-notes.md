# v0.3.0-alpha Upgrade Notes

Upgrade from a GitHub clone:

```bash
git fetch --tags origin
git checkout <v0.3.0-alpha-tag-or-release-branch>
python -m pip install -e ".[all]"
unlimited-skills --version
unlimited-skills reindex
unlimited-skills doctor --json
```

PyPI is not the supported `v0.3.0-alpha` distribution path.

## From v0.2.2-alpha

- Existing local libraries should remain in place.
- Native sync and hosted update flows must not delete pre-existing local `local/` skills.
- Managed Enterprise Skill Lock sync adds a new local state file under `policy/managed-policy-state.json`.
- Registry-driven policy removal is guarded and refuses to remove unmanaged local policies.
- Existing manually installed Enterprise Skill Lock policies remain local unless they were installed by managed sync and match the managed state.

## Verify After Upgrade

```bash
python scripts/verify-v0.3.0-alpha-package-assets.py
python scripts/run-v0.3.0-alpha-packaging-smoke.py
python scripts/run-managed-policy-sync-e2e.py --fixture-mode --temp-home
python scripts/run-v0.3.0-alpha-release-smoke.py
```

Registered hosted flows still require a registered installation, hosted token, signed device proof, and trusted signed manifests.
