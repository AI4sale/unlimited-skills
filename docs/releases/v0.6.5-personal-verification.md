# v0.6.5 Personal Verification Procedure

This procedure is for an owner/operator validating `v0.6.5-alpha` locally before
or after release execution. Use the actual current CLI flags; do not invent
flags if the command help differs.

## Version

```powershell
unlimited-skills --version
python -m unlimited_skills --version
```

Expected after publish: both commands report `0.6.5`.

## Retrieval and Hook Candidate Delivery

```powershell
unlimited-skills suggest "напиши пост для линкедин" --json --card --limit 5
unlimited-skills search "напиши пост для линкедин" --mode hybrid --json --limit 5
```

Expected: the local library returns usable LinkedIn/social/content candidates
when those skills are installed. Candidate sources and ranks should be visible
on the card/search path. Low confidence is acceptable; zero candidates when
retrieval has candidates is not.

## Release Gates

```powershell
python scripts/verify-v065-retrieval-learning-release-smoke.py --json
python scripts/verify-v065-learning-loop.py --json
python scripts/verify-v065-long-run-retrieval-learning.py --json
python scripts/verify-v065-alpha-release-execution.py --json
python scripts/run-v065-alpha-package-smoke.py --json
```

Expected:

- combined release smoke reports `ok=true`;
- learning loop includes manual search -> view -> use without `--query`;
- long-run gate reports 100 steps, 10 phases, zero candidate losses, and all
  injected negative controls detected;
- release execution verifier reports `status=passed`;
- package smoke builds wheel/sdist, clean-installs `0.6.5`, and runs installed
  CLI retrieval smoke.

## Release Execution Boundary

This procedure does not authorize release execution by itself. PyPI Trusted
Publishing, tag creation, and GitHub prerelease creation are separate
release-operator actions and remain blocked until selected-main-SHA gates pass.

Do not include marketplace, hosted rollout, paid-tier work, #195, or #119/E19 in
this release.
