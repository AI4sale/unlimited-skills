# v0.3.1-alpha Upgrade Notes

Upgrade from a GitHub clone:

```bash
git fetch --tags origin
git checkout <v0.3.1-alpha-branch-or-tag>
python -m pip install -e ".[all]"
unlimited-skills --version
unlimited-skills reindex
unlimited-skills doctor --json
```

PyPI is not the supported `v0.3.1-alpha` distribution path.

## From v0.3.0-alpha

- Existing local libraries should remain in place.
- Native sync and hosted update flows must not delete pre-existing `local/` skills.
- Registry-driven Enterprise Skill Lock policy removal remains guarded and refuses to remove unmanaged local policies.
- Existing manually installed Enterprise Skill Lock policies remain local unless they were installed by managed sync and match the managed state.

## From v0.2.2-alpha

- The v0.3 managed policy sync state may add local state under `policy/managed-policy-state.json` only when managed sync is used.
- Existing `local/` and `registry/` library content is preserved by the synthetic upgrade smoke.
- Hosted features remain registration-gated.

## Verify After Upgrade

```bash
python scripts/run-v0.3.1-alpha-post-release-smoke.py
python scripts/verify-v0.3.1-alpha-stabilization.py
```
