# v0.4.7-alpha Known Issues

`v0.4.7-alpha` is an alpha integration gate for signed MCP profile bundle
verification.

## Intentional Limitations

- Behavior is alpha and may break before v0.6.
- The evidence is fixture-only. It does not fetch trust data from hosted
  services and does not synchronize with the registry.
- The local MIT core may still allow unsigned profiles by policy.
- Registered/business signed-required behavior is future-gated unless a later
  gate explicitly implements it.
- Trust is a local trusted-keys file. There is no PKI, no hosted trust fetch,
  and no production signing key material in this repository.
- Revocation is local CRL only. A declared but unreadable CRL fails closed.
- No OAuth, remote upstreams, MCP resources, MCP prompts, hosted gateway mode,
  production hosted calls, full catalog distribution, auto-publish, live
  billing, or PyPI publication are included.
- Codex must not create or push `v0.4.7-alpha` from this integration gate.

## Operator Notes

- Use `--profile-bundle`, `--trusted-keys`, and `--audience-id` together for
  signed bundle verification.
- Use `--require-signed-profiles` only when the host policy should reject raw
  unsigned profile files.
- Without signed-bundle flags, raw local profiles keep the existing local MIT
  core behavior.
