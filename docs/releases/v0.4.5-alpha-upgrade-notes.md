# v0.4.5-alpha Upgrade Notes

`v0.4.5-alpha` is a GitHub-clone alpha release. It publishes the MCP audit
inspector integration for the local stdio Unlimited Tools gateway audit log.

## Upgrade

From a local clone:

```powershell
git fetch --tags origin
git checkout v0.4.5-alpha
python -m pip install -e ".[all]"
```

For a working branch that tracks `main`:

```powershell
git fetch origin
git checkout main
git pull --ff-only
python -m pip install -e ".[all]"
```

## What Changes

- `unlimited-skills mcp audit-report` renders a local read-only report over
  the redacted MCP audit JSONL log.
- The report covers summary, refusal codes, upstream health, optional profile
  evidence, and redaction self-checks.
- JSON output validates against `schemas/mcp-audit-report.schema.json`.
- Rotated audit logs are read oldest first, then the active audit log.
- Recent refusals include timestamp, tool, upstream, code, and name only: no
  argument values and no error text.
- The redaction self-check reports suspect file, line, field, and reason only.
  Suspect values are never printed.
- Missing audit logs return a clear exit code 1 message instead of a traceback.

## Compatibility

This release does not change MCP audit writer format, gateway defaults, or
profile behavior. The inspector is local and read-only: it does not write,
rotate, or mutate audit logs.

MCP v1 schemas/configs/profiles/audit reports are alpha and may break before
v0.6.

## Boundaries

- No audit writer changes.
- No audit log writes from the inspector.
- No OAuth.
- No remote upstreams.
- No MCP resources or prompts.
- No hosted gateway.
- No production hosted calls.
- No automatic telemetry.
- No arbitrary shell execution.
- No full catalog distribution.
- No auto-publish.
- No live billing.
- No PyPI publication.
