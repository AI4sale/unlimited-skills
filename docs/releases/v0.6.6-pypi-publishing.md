# v0.6.6 PyPI publishing

## Trusted Publisher

| Field | Value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is
used. The workflow obtains a short-lived OIDC publishing identity.

## Manual inputs

- `version`: `0.6.6`
- `expected_sha`: exact release commit on `main`
- `confirm_pypi_publish`: `publish unlimited-skills 0.6.6 to PyPI`

The workflow checks all three values, runs the focused regression suite, frozen
contracts, package smoke, build, and `twine check`, then publishes through the
`pypi` environment.

After upload it waits for the exact PyPI version endpoint, checks that the
public filenames and SHA256 digests match the locally built release artifacts,
installs the public wheel in a clean environment, and verifies quickstart plus
the critical retrieval invariants. Only a successful public-wheel verification may create
the `v0.6.6` GitHub release on the same SHA.

If the upload succeeds but public verification fails, leave the GitHub tag
absent, capture the workflow evidence, and follow the rollback rule in
[`v0.6.6-plan.md`](v0.6.6-plan.md).
