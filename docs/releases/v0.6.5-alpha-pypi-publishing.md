# v0.6.5-alpha PyPI Trusted Publishing

Status: required release execution gate for `v0.6.5-alpha`. This document
describes the future PyPI operation; it does not perform the publish.

| Field | Value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |
| Package version | `0.6.5` |
| Tag after publish | `v0.6.5-alpha` |

Manual workflow inputs:

```yaml
version: 0.6.5
expected_sha: <selected-main-sha>
confirm_pypi_publish: publish unlimited-skills 0.6.5 to PyPI
```

The workflow must use GitHub OIDC Trusted Publishing through
`pypa/gh-action-pypi-publish@release/v1`.

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is
allowed.

Before dispatch:

```powershell
python scripts/verify-v065-retrieval-learning-release-smoke.py --json
python scripts/verify-v065-alpha-release-execution.py --json
python scripts/run-v065-alpha-package-smoke.py --json
python scripts/verify-v06-frozen-contracts.py --expected-version 0.6.5 --json
python scripts/verify-feedback-report-boundaries.py
git diff --check
```

After PyPI upload:

```powershell
python -m venv .tmp/v065-published-smoke
.tmp/v065-published-smoke/Scripts/python.exe -m pip install --no-cache-dir unlimited-skills==0.6.5
.tmp/v065-published-smoke/Scripts/unlimited-skills.exe --version
python scripts/verify-v06-frozen-contracts.py --expected-version 0.6.5 --json
```

Only the post-publish clean install may unlock the `v0.6.5-alpha` tag. Do not
create the tag before `unlimited-skills==0.6.5` installs from PyPI and the
post-publish verifier passes.

Non-goals for this release package PR:

- No PyPI publish.
- No tag.
- No GitHub release.
- No marketplace.
- No hosted rollout.
- No paid-tier work.
