# Unlimited Skills v0.2.1-alpha Release Checklist

Use this checklist after the stacked release PRs are reviewed and before tagging `v0.2.1-alpha`.

## Required Merge Order

1. Private registry PR #2: signed registry manifest publisher.
2. Public PR #13: `release/v0.2.1-alpha-hub-runtime-integration`
3. Public PR #14: `feat/remote-hub-router-installers-v1`
4. Public PR #15: `feat/capability-aware-local-install-plans-v1`
5. Public PR #16: `feat/signed-hub-manifests-v1`
6. Public PR #17: `release/v0.2.1-alpha-ci-and-merge-gate`
7. Public PR #18: `feat/trust-key-bootstrap-rotation-v1`
8. Public PR #19: `feat/hub-client-lifecycle-quota-observability-v1`
9. Finalization PR: `release/v0.2.1-alpha-finalization`

## Release Gate

- [ ] GitHub Actions CI is green on the release-gate PR.
- [ ] Full pytest suite is green.
- [ ] v0.2.x smoke suite is green.
- [ ] Release verifier is green: `python scripts/verify-v0.2.1-alpha-release.py`.
- [ ] Fresh install smoke is green: `python scripts/run-v0.2.1-alpha-fresh-install-smoke.py`.
- [ ] Upgrade smoke from a synthetic v0.2.0 layout is green: `python scripts/run-v0.2.1-alpha-upgrade-smoke.py`.
- [ ] `python -m compileall -q unlimited_skills scripts tests` is green.
- [ ] `git diff --check` is green.
- [ ] Tests use temporary HOME and temporary Unlimited Skills roots.
- [ ] Tests make no production hosted registry calls by default.
- [ ] No raw hub tokens, registration tokens, device private keys, or signing private keys are present in repo files or CI logs.
- [ ] Full catalog distribution remains disabled.
- [ ] Local Skill Hub distribution remains allowlist-only.
- [ ] Hosted remote manifests require valid signed manifest envelopes.
- [ ] Archive bytes still require SHA256 verification and safe extraction.
- [ ] `unlimited-skills serve` remains MIT/free/unregistered.
- [ ] `unlimited-skills hub serve` remains registration-gated.
- [ ] Remote CLI search/resolve/view are documented as alpha Local Skill Hub client commands, not hosted registry search.
- [ ] Machine-readable release manifest exists and is valid: `docs/releases/v0.2.1-alpha.release-manifest.json`.
- [ ] Release notes reviewed.
- [ ] Security notes reviewed.
- [ ] Tag approval received from the release owner.

## Tagging

```bash
git tag v0.2.1-alpha
git push origin v0.2.1-alpha
```

Create the GitHub release only after the release owner approves the tag.
