# v0.5.1-alpha PyPI Trusted Publishing

Status: required unblock for `v0.5.1-alpha` publication. The release tag stays
blocked until Trusted Publishing uploads `unlimited-skills==0.5.1` and the
post-publish smoke passes.

## Trusted Publisher Values

| PyPI field | Value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is
used. Publication must use GitHub OIDC through PyPI Trusted Publishing.

## Manual Workflow Inputs

```text
version: 0.5.1
expected_sha: <selected main commit sha>
confirm_pypi_publish: publish unlimited-skills 0.5.1 to PyPI
```

The workflow must fail if any value differs.

## Prepublish Gate

```bash
python scripts/run-v051-alpha-package-smoke.py --json
python scripts/verify-v051-alpha-publication.py \
  --expected-sha "$(git rev-parse HEAD)" \
  --package-availability prepublish \
  --json
```

Prepublish mode proves the local wheel/sdist and clean wheel install. It does
not prove PyPI availability and must not emit tag guidance.

## Post-publish Smoke

After the workflow uploads the package:

```bash
python -m venv /tmp/uls-v051-smoke
source /tmp/uls-v051-smoke/bin/activate
python -m pip install --upgrade pip
python -m pip install --no-cache-dir unlimited-skills==0.5.1

unlimited-skills --version
unlimited-skills quickstart --json
unlimited-skills mcp install --claude-code --dry-run
unlimited-skills feedback prepare --format json
unlimited-skills mcp savings --json

deactivate
```

Then run:

```bash
python scripts/verify-v051-alpha-publication.py \
  --expected-sha "$(git rev-parse HEAD)" \
  --package-availability published \
  --json
```

Only the published verifier may unlock the `v0.5.1-alpha` tag.
