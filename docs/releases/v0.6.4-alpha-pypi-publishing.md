# v0.6.4-alpha PyPI Trusted Publishing

Status: required release execution gate for `v0.6.4-alpha`. This document
describes the release-operator path; it does not authorize publishing by itself.

| Field | Value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |
| Package version | `0.6.4` |

Manual workflow inputs:

```yaml
version: 0.6.4
expected_sha: <selected main commit>
confirm_pypi_publish: publish unlimited-skills 0.6.4 to PyPI
```

Pre-publish gate:

```powershell
python scripts/verify-v064-money-saved-tier-smoke.py --json
python scripts/verify-v064-alpha-release-execution.py --json
python scripts/run-v064-alpha-package-smoke.py --json
python scripts/verify-v06-frozen-contracts.py --json
python scripts/verify-feedback-report-boundaries.py
git diff --check
```

Post-publish clean install smoke:

```powershell
python -m venv .tmp/v064-published-smoke
.tmp/v064-published-smoke/Scripts/python.exe -m pip install --upgrade pip
.tmp/v064-published-smoke/Scripts/python.exe -m pip install --no-cache-dir unlimited-skills==0.6.4
.tmp/v064-published-smoke/Scripts/unlimited-skills.exe --version
.tmp/v064-published-smoke/Scripts/unlimited-skills.exe money-saved meter --json --fixture-100-call
python scripts/verify-v06-frozen-contracts.py --expected-version 0.6.4 --json
```

Only the post-publish clean install may unlock the `v0.6.4-alpha` tag. Do not
create the tag before `unlimited-skills==0.6.4` installs from PyPI and the
post-publish verifier passes.

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is used.
The workflow uses GitHub OIDC Trusted Publishing through the `pypi` environment.
