# v0.5.3-alpha

Status: local event privacy hardening packaging gate; PyPI publication is
blocked until Trusted Publishing and post-publish smoke pass.

This release packages the A4.10 local event privacy hardening that landed after
`v0.5.2-alpha`. The release gate proves the installed package redacts local
learning rows before they are written: task/query/filter text becomes
`query_summary_hash`, freeform notes become `notes_present` and
`notes_length_bucket`, skill paths become safe library-relative paths, and raw
hit scores become `score_bucket`.

## What Changed

- Package, Claude plugin, and marketplace metadata versions are `0.5.3`.
- Local `events.jsonl` and `feedback.jsonl` persistence no longer stores raw
  query/task/filter text, freeform notes, absolute skill paths, or exact hit
  scores.
- Public stdout/API search, view, use, and feedback behavior remains unchanged.
- `learning-summary --events` remains aggregate-only.
- `feedback prepare` remains the redacted support surface for public-alpha
  reports.
- `scripts/run-v053-alpha-package-smoke.py` extends the v0.5.2 smoke with an
  installed-wheel local event privacy check.
- `scripts/verify-v053-alpha-publication.py` gates package metadata, release
  docs, manifest requirements, clean-install smoke, local event privacy smoke,
  and post-publish PyPI smoke before tag guidance is emitted.
