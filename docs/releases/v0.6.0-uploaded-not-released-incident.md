# PyPI 0.6.0 Uploaded-Not-Released Incident

Status: owner decision packet. No package action has been performed by this
document.

The `unlimited-skills==0.6.0` distribution was uploaded to PyPI during the
v0.6 release gate, but it failed the published verifier after upload because
the installed CLI rejected `learning-summary --events --json`. The release was
therefore not tagged, not announced, and not accepted as the valid v0.6 alpha.

`unlimited-skills==0.6.1` is the valid replacement and backs the
`v0.6.1-alpha` GitHub prerelease.

Future release operators should follow the
[v0.6.x release-operator runbook](v0.6.x-release-operator-runbook.md) so an
uploaded-but-failed package remains untagged, unreleased, and replaced by a
hotfix version instead of being retroactively tagged.

## User Guidance

Use `0.6.1` or newer:

```bash
python -m pip install --upgrade "unlimited-skills>=0.6.1"
```

For an exact reproduction of the accepted v0.6 alpha package:

```bash
python -m pip install --upgrade "unlimited-skills==0.6.1"
unlimited-skills --version
unlimited-skills learning-summary --events --json
```

Do not use `0.6.0` as release evidence for the v0.6 alpha. It was uploaded to
PyPI, but it was not tagged or released because the published verifier caught
the CLI contract drift.

If the owner later yanks or marks `0.6.0` as bad on PyPI, record the owner,
date, reason, and PyPI evidence here. Until then, the fallback is to keep
`0.6.0` untagged/unreleased and point users to `0.6.1+`.
