# v0.5.0-alpha Blocked Status

Status: `blocked_account_setup`.

Unlimited Skills v0.5.0-alpha is code-ready for the package gate, but it is
not ready to tag or publish as a GitHub prerelease. The remaining blocker is
account setup for PyPI Trusted Publishing and the protected GitHub `pypi`
environment.

## Current State

- Package version: `0.5.0`.
- Public install path before PyPI upload:
  `pip install "git+https://github.com/AI4sale/unlimited-skills.git"`.
- PyPI package availability: blocked until Trusted Publishing uploads
  `unlimited-skills==0.5.0`.
- GitHub tag `v0.5.0-alpha`: blocked until real PyPI install smoke passes.
- GitHub prerelease: blocked until the tag can point at the same SHA that
  passed the published-mode verifier.

## Blocker

Owner:

```text
PyPI account owner / GitHub repository release owner
```

Action:

```text
1. Create GitHub environment pypi for AI4sale/unlimited-skills.
2. Configure PyPI Pending Trusted Publisher:
   owner = AI4sale
   repository = unlimited-skills
   workflow filename = publish-pypi.yml
   environment = pypi
   project name = unlimited-skills
3. Approve the pypi environment when the release workflow is manually
   dispatched.
```

Fallback:

```text
Keep the public GitHub install command, do not tag v0.5.0-alpha, and do not
create a GitHub prerelease until the PyPI path is configured and verified.
```

No local PyPI token is required for the approved path. Do not use
`PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or a local `.pypirc` for this
release unless the release owner explicitly changes the publication policy.

## Why PR #127 Was Merged

PR #127 holds the public PyPI install flip while account setup is incomplete.
Without it, `main` would advertise `pip install unlimited-skills` before the
package exists on PyPI. That is a dead first-run path for new users.

After account setup is complete, the publication PR should revert #127 and
reapply the PyPI install flip in one short release operation.

## #119 Rule

PR #119 is background E19 trust-stack work. It is not a v0.5 publication
blocker and must stay out of the `v0.5.0-alpha` tag target.

## Post-Unblock Sequence

1. Confirm GitHub environment `pypi` exists and is protected.
2. Confirm PyPI Pending Trusted Publisher exists for
   `AI4sale/unlimited-skills`, workflow `publish-pypi.yml`, environment `pypi`.
3. Create `release/v0.5.0-alpha-pypi-flip-reapply`.
4. Revert the #127 merge so public docs return to `pip install
   unlimited-skills`.
5. Verify package smoke and the prepublish verifier.
6. Merge the publication PR.
7. Manually dispatch `.github/workflows/publish-pypi.yml` with:

```text
version = 0.5.0
expected_sha = <merged main SHA>
confirm_pypi_publish = publish unlimited-skills 0.5.0 to PyPI
```

8. Run a clean PyPI install smoke:

```bash
python -m pip install --no-cache-dir unlimited-skills==0.5.0
unlimited-skills --version
unlimited-skills quickstart --json
unlimited-skills suggest "Design a REST API for a service" --json
unlimited-skills mcp savings --json
```

9. Run:

```bash
python scripts/verify-v050-alpha-publication.py \
  --expected-sha "<merged main SHA>" \
  --package-availability published \
  --json
```

10. Create tag `v0.5.0-alpha` and the GitHub prerelease only after the
    published-mode verifier passes.
