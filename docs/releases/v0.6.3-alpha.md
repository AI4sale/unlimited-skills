# v0.6.3-alpha

Status: release package draft for review. Do not tag, version bump, publish, or
promote from this document alone.

`v0.6.3-alpha` ships local Learning Loop command surfaces that can be run and
verified from a clean checkout. The release claim is command-based: local
feedback can be inspected, converted into privacy-safe candidates, previewed in a
non-mutating dry run, and exported through local tier artifacts.

## User-Facing Value

The release makes the local Learning Loop visible and reproducible:

- Free: inspect feedback state with `unlimited-skills learning doctor`, list
  candidates with `unlimited-skills improvement-candidates`, and preview a
  candidate with `unlimited-skills apply-candidate --dry-run <candidate-id>`.
- Registered: write a local schema-versioned Learning Loop export with
  `unlimited-skills learning export`.
- Team: aggregate registered exports with
  `unlimited-skills learning team-rollup`.
- Business: write local admin JSON/CSV with
  `unlimited-skills learning admin-export`.
- Enterprise: write and independently verify a tamper-evident local evidence pack
  with `unlimited-skills learning evidence-pack` and
  `unlimited-skills learning verify-evidence-pack`.

All surfaces are local-only. They do not upload data, call hosted services,
enable telemetry, mutate skills automatically, or claim automatic improvement.

## v0.6.2 Compatibility Note

The v0.6.2 router-health tier-debt ladder is closed separately from the v0.6.3
Learning Loop value frame. The compatibility smoke checks the local
`unlimited-skills router-health export`, `team-rollup`, `admin-export`,
`evidence-pack`, and `verify-evidence-pack` surfaces, but those surfaces are not
the v0.6.3 Learning Loop VFP.

## Release Gate

Before any release execution:

1. Merge and verify the release package, smoke verifier, and personal procedure.
2. Obtain Opus/Hermes review PASS for the release package.
3. Run the smoke verifier and required checks from refreshed `main`.
4. Keep release execution separate from this PR.
5. Publish only with explicit owner approval and the release operator gate.
6. Tag only after the package is published and clean-install verification passes.
