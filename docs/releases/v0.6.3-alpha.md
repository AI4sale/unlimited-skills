# v0.6.3-alpha

Status: release-gate draft for human review. Do not tag or publish from this
document alone.

`v0.6.3-alpha` turns the existing local feedback and learning summaries into a
small, inspectable Learning Loop:

- diagnose local learning state with `unlimited-skills learning doctor`;
- list privacy-safe improvement candidates with
  `unlimited-skills improvement-candidates`;
- preview a candidate with
  `unlimited-skills apply-candidate --dry-run <candidate-id>`;
- keep all data local and dry-run first.

## User-Facing Value

Users can now see whether local feedback can produce a concrete next action
without uploading prompts or mutating skill files. Wrong, missed, and rejected
skill signals become reviewable candidates instead of dead-end logs.

## Release Gate

Before any tag or publish:

1. merge the accepted v0.6.3 Learning Loop PRs in order;
2. confirm no accepted open PR debt remains for v0.6.3;
3. run the smoke commands above from refreshed `main`;
4. run the project release operator process for the selected package version;
5. publish only with owner approval and PyPI Trusted Publishing;
6. tag only after clean install and published-mode verification pass.

## Boundaries

- Local-only `.learning` reads.
- No telemetry, hosted sync, model training, marketplace submission, or
  automatic skill edits.
- `apply-candidate` is dry-run only in this release.
- Candidate JSON must not include prompts, raw queries, raw notes, local paths,
  tokens, keys, or skill bodies.
