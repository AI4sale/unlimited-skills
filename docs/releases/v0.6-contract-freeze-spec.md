# v0.6 Contract Freeze Spec

Status: public-alpha contract-freeze specification for the next adoption cycle.
This is a documentation and compatibility gate, not a runtime release and not
a package publication.

The v0.6 freeze turns the working v0.5 public-alpha surfaces into explicit
contracts. The goal is to make first-value adoption safer: scripts, docs,
support replies, and external reviewers can rely on the named command shapes
while the project still keeps pre-1.0 implementation flexibility.

## Scope

The freeze covers these public, local-first surfaces:

- `unlimited-skills quickstart`;
- `unlimited-skills suggest`;
- `unlimited-skills mcp savings`;
- `unlimited-skills mcp install --claude-code`;
- `unlimited-skills feedback prepare`;
- `unlimited-skills learning-summary --events`;
- `scripts/generate-public-alpha-signal-rollup.py`;
- local event log privacy behavior after `v0.5.3-alpha`;
- feedback report schema;
- stdout JSON stability for the listed commands;
- PyPI release and Trusted Publishing expectations.

## Compatibility Window

The freeze is a compatibility promise through the v0.7 adoption cycle:

- Stable contracts may add optional fields, warnings, or enum values, but must
  not remove documented fields or change their meaning before v0.7.
- Preview contracts may add fields and tighten validation, but must keep their
  documented privacy boundary and failure mode.
- Alpha implementation details can change before 1.0 when the contract says
  they are not frozen.

See [../compatibility.md](../compatibility.md) for the user-facing
compatibility model and [../cli-contracts.md](../cli-contracts.md) for command
contract details.

## Frozen Contracts

| Surface | Stability through v0.7 | Frozen behavior |
| --- | --- | --- |
| `quickstart` | Stable public-alpha | Local golden path. Imports bundled packs only when the library is empty, runs first search, measures local MCP savings unless skipped, and emits JSON with `root`, `library`, `search`, `savings`, `savings_error`, and `next_steps`. |
| `suggest` | Stable public-alpha | Local skill probe. Text mode prints at most the requested suggestions or silence/no-match text. JSON mode emits the privacy-hardened object documented in [../cli-contracts.md](../cli-contracts.md) and must not include prompt text, local paths in card mode, skill bodies except the selected compact card when explicitly requested, or unrelated skills. |
| `mcp savings` | Stable public-alpha | Local measurement only. It reads the local Claude Code MCP config, measures configured server tool schemas locally, returns server names, tool counts, byte/token estimates, statuses, gateway bytes, savings bytes, and savings percent, and uploads nothing. |
| `mcp install --claude-code` | Stable public-alpha | Safe Claude Code gateway installer. Dry-run remains available, other MCP servers are preserved, backups are created before writes, JSON is validated before and after writes, and dry-run output redacts env values and local paths. |
| `feedback prepare` | Stable public-alpha | Paste-safe local support report. Output remains local-only, redacted, schema-versioned, and must not include prompts, tool inputs/outputs, skill bodies, MCP schemas, env values, tokens, keys, raw `.mcp.json`, raw `.claude.json`, raw learning logs, or local absolute paths. |
| `learning-summary --events` | Stable public-alpha | Aggregate-only local effectiveness summary. It may report feedback counts and aggregate effectiveness metrics, but not raw query/task text, prompt text, skill bodies, full local paths, tokens, env values, or raw notes. |
| `generate-public-alpha-signal-rollup.py` | Stable docs tooling | Offline fixture mode and public aggregate live mode. It may read public PyPI/GitHub counters and owner-provided aggregate social JSON only when explicitly supplied. It does not add telemetry, tracking pixels, analytics, private social scraping, or private user data collection. |
| Local event privacy after `v0.5.3-alpha` | Stable privacy boundary | New local rows replace raw query/task/filter text with hashes/presence flags, bucket exact scores, do not persist raw freeform notes, and remove or relativize local paths where safe. Legacy pre-v0.5.3 local logs are not automatically rewritten. |
| Feedback report schema | Stable public-alpha | Schema versioning, local-only generation, redacted sections, issue-template mapping, and usage snapshot boundaries remain stable. Optional sections can be added when redacted. |
| PyPI/Trusted Publishing | Stable release gate | PyPI publication requires the documented Trusted Publishing workflow and post-publish verifier pass before tagging. Tags must not be claimed before package availability and verifier success. |

## Still Alpha Before 1.0

These details may still change before 1.0:

- exact lexical/vector scoring and ranking internals;
- exact human-readable wording in text output;
- extra optional JSON fields;
- optional issue-template mappings and adoption labels;
- release checklist wording;
- fixture values used by docs tooling;
- internal local event file layout, as long as the privacy boundary remains
  intact;
- non-public hosted, team, enterprise, marketplace, and #119/E19 work unless
  explicitly reopened and accepted in a later gate.

## Boundaries

This freeze does not implement runtime behavior, publish a package, submit to
marketplaces, merge #119/E19, add telemetry, add analytics, add hosted query
forwarding, or create paid, hosted, team, or enterprise readiness claims.

No support promise is created. Maintainer responses remain best-effort public
alpha triage and must continue to use
[../adoption/support-response-pack.md](../adoption/support-response-pack.md)
and
[../adoption/local-event-privacy-support-runbook.md](../adoption/local-event-privacy-support-runbook.md).
