# v0.3.9-alpha

Status: integration gate for the maintainer-controlled skill improvement workflow.

`v0.3.9-alpha` proves the cross-repo remediation path: feedback and registry evals can create an improvement backlog, maintainers can accept a candidate and mark it fixed pending eval, the catalog quality report can include the improvement section, and the public client can display signed improvement status plus preview-only update recommendations.

## Scope

- Cross-repo skill improvement E2E runner.
- Fixture mode without a private registry checkout.
- External local-registry mode when `AI4sale/unlimited-skills-registry` PR #37 is available locally.
- Registry eval, feedback issue, backlog generation, maintainer triage, fixed pending eval, and catalog quality report proof.
- Public signed `catalog improvement-status`, `catalog known-issues`, `catalog update-recommendations`, `catalog update-preview`, and `catalog deprecation-status` proof.
- Deprecated/retired warning proof.
- Support bundle improvement summary counts only.

## Security Boundary

- No automatic skill rewriting.
- No auto-publish.
- No prompt upload.
- No task text upload.
- No user telemetry.
- No private skill bodies in the public repo.
- No untrusted script execution.
- No production hosted calls in tests.
- No final `v0.3.9` tag in this PR.
