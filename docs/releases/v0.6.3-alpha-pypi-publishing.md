# v0.6.3-alpha PyPI Trusted Publishing

Status: required release execution gate for `v0.6.3-alpha`. This document
prepares the exact publish path but does not upload, tag, or create a GitHub
release by itself.

The old `release_owner_GO_WITH_LIMITS_acceptance` gap is reconciled by this
release execution package once all listed gates pass. It is not a generic
blocker after C063-REL-02; it remains a historical marker for the earlier
decision-package stage.

## PyPI Trusted Publisher Values

| Field | Value |
| --- | --- |
| Project/package name | `unlimited-skills` |
| Owner | `AI4sale` |
| Repository | `unlimited-skills` |
| Workflow filename | `publish-pypi.yml` |
| Environment | `pypi` |
| Package version | `0.6.3` |

No `PYPI_TOKEN`, `TWINE_PASSWORD`, `TWINE_USERNAME`, or local `.pypirc` is used
by the release workflow. Publishing uses GitHub OIDC through PyPI Trusted
Publishing.

## Manual Workflow Inputs

```text
version: 0.6.3
expected_sha: <selected main commit>
confirm_pypi_publish: publish unlimited-skills 0.6.3 to PyPI
```

## Pre-Publish Commands

Run from the selected release commit before dispatching the workflow:

```bash
python scripts/verify-v063-tier-release-smoke.py --json
python scripts/verify-v063-alpha-release-execution.py --json
python scripts/verify-v06-frozen-contracts.py --json
python scripts/verify-feedback-report-boundaries.py
python -m pytest tests/test_v063_tier_release_smoke.py tests/test_learning_loop_cli.py tests/test_learning_export.py tests/test_learning_team_rollup.py tests/test_learning_admin_export.py tests/test_learning_evidence_pack.py tests/test_learning_verify_evidence_pack.py -q
python -m pytest tests/test_public_alpha_docs.py tests/test_pypi_trusted_publishing_workflow.py tests/test_v06_frozen_contracts.py -q
python scripts/run-v063-alpha-package-smoke.py --json
python -m py_compile scripts/run-v063-alpha-package-smoke.py scripts/verify-v063-alpha-release-execution.py scripts/verify_v063_tier_release_smoke.py
git diff --check
```

`scripts/run-v063-alpha-package-smoke.py` builds the wheel/sdist, runs metadata
checks, installs the built wheel into fresh virtual environments, and proves the
Free, Registered, Team, Business, and Enterprise Learning Loop command evidence
from the installed package.

## Workflow Dispatch

Dispatch `.github/workflows/publish-pypi.yml` only after C063-REL-02 is merged,
Opus `O063-REL-FINAL-R1` returns `PASS`, and the selected `main` SHA passes the
pre-publish commands above.

The workflow itself rechecks:

```bash
python scripts/run-v063-alpha-package-smoke.py --json
python scripts/verify-v063-alpha-release-execution.py --expected-sha <selected main commit> --json
python -m build
python -m twine check dist/*
```

## Post-Publish Commands

Run only after the PyPI workflow succeeds:

```bash
python -m venv .tmp/v063-published-smoke
.tmp/v063-published-smoke/Scripts/python.exe -m pip install --upgrade pip
.tmp/v063-published-smoke/Scripts/python.exe -m pip install --no-cache-dir unlimited-skills==0.6.3
.tmp/v063-published-smoke/Scripts/unlimited-skills.exe --version
.tmp/v063-published-smoke/Scripts/unlimited-skills.exe quickstart --json
.tmp/v063-published-smoke/Scripts/unlimited-skills.exe suggest "Design a REST API for a service" --json
.tmp/v063-published-smoke/Scripts/unlimited-skills.exe learning doctor
.tmp/v063-published-smoke/Scripts/unlimited-skills.exe improvement-candidates
.tmp/v063-published-smoke/Scripts/unlimited-skills.exe mcp savings --json
.tmp/v063-published-smoke/Scripts/unlimited-skills.exe feedback prepare --json
```

Only the post-publish clean install may unlock the `v0.6.3-alpha` tag. Do not
create the tag before `unlimited-skills==0.6.3` installs from PyPI and the
post-publish verifier passes.
