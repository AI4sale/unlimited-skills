# v0.6.4 Personal Verification Procedure

This procedure is for an owner/operator validating `v0.6.4-alpha` locally before
or during release execution. It verifies runnable commands, package build, clean
install, and post-release checks.

## 1. Confirm Version State

```powershell
python -c "import tomllib; print(tomllib.load(open('pyproject.toml','rb'))['project']['version'])"
python -c "import unlimited_skills; print(unlimited_skills.__version__)"
```

Expected: both commands print `0.6.4`.

## 2. Run The One-Command Smoke

```powershell
python scripts/verify-v064-money-saved-tier-smoke.py --json
```

Expected:

- `ok` is `true`.
- `release` is `v0.6.4-alpha`.
- rows include Free, Registered, Team, Business, Enterprise, clean verify, and
  tamper fail-closed verify.
- `privacy.no_egress_asserted` is `true`.
- exact money, exact token, and bill-reduction claims are false.

## 3. Run Release Verification Commands

```powershell
python scripts/verify-v064-alpha-release-execution.py --json
python scripts/run-v064-alpha-package-smoke.py --json
python scripts/verify-v06-frozen-contracts.py --json
python scripts/verify-feedback-report-boundaries.py
git diff --check
```

Expected: all commands pass.

## 4. Package Build And Clean Install Smoke

```powershell
python scripts/run-v064-alpha-package-smoke.py --json
```

Expected:

- `ok` is `true`.
- `version` is `0.6.4`.
- distribution metadata mentions `0.6.4`.
- clean install reports `unlimited-skills 0.6.4`.
- Free, Registered, Team, Business, and Enterprise Money Saved tier rows are
  proven from the installed wheel.
- Enterprise evidence-pack tamper check fails closed.

## 5. Trusted Publishing And Post-Publish Verification

Use `docs/releases/v0.6.4-alpha-pypi-publishing.md`. Dispatch the PyPI workflow
only after C064-REL-01 is merged, Opus O064-REL-FINAL-R1 returns PASS, and all
pre-publish checks pass on the selected SHA.

After PyPI upload, install `unlimited-skills==0.6.4` in a clean environment and
run the post-publish commands from the PyPI publishing doc. Create
`v0.6.4-alpha` only after the clean PyPI install and post-publish verifier pass.

## 6. Boundary Check

This procedure authorizes only verification and release-operator decision
readiness. Do not tag, publish to PyPI, yank, submit marketplace listings, or
start hosted/team/business/enterprise rollout inside a PR. No exact money, exact
token, or guaranteed bill-reduction claim is allowed.
