# v0.4.0-alpha Upgrade Notes

`v0.4.0-alpha` is the first SkillOps foundation milestone. It keeps the MIT local core registration-free and keeps hosted/registry behavior behind signed metadata, registration, and explicit user action.

## From v0.3.13

1. Pull the release tag after release-owner approval:

```powershell
git fetch --tags origin
git checkout v0.4.0-alpha
python -m pip install -e ".[all]"
```

2. Run local verification:

```powershell
python scripts/run-v0.2x-smoke-tests.py
python scripts/run-v040-alpha-release-smoke.py
```

3. Reinstall or resync agent integration if you use Claude Code plugin metadata:

```powershell
python -m pip install -e ".[all]"
```

The release updates the package/plugin version to `0.4.0` while the Git tag remains `v0.4.0-alpha`.

## What Changes

- Policy-aware recommendation preview can combine signed catalog metadata with quality, improvement, entitlement, and policy signals.
- Eval release workflow evidence is part of the v0.4 gate.
- Maintainer queue status is exposed as signed metadata and redacted support diagnostics.
- Governance dashboard summaries are read-only signed metadata.
- Release smoke now verifies the E01-E04 integration package plus the final publication manifest.

## What Does Not Change

- Local search, list, view, where, use, feedback, reindex, vector-reindex, adapt, serve, installers, native sync, and self-update remain local-first.
- Registration is not required for the MIT local core.
- The release does not enable production rollout, live billing, PyPI publication, full catalog distribution, automatic install/update/remove, automatic rewriting, automatic reindexing, or auto-publish.
- The client must not upload prompts, task text, search queries, local paths, repository paths, tokens, proofs, private keys, skill bodies, private-pack bodies, or customer data.

## Tagging

Codex prepares this publication package but must not push `v0.4.0-alpha`. After the PR is merged, the release owner verifies the final `main` SHA and creates the tag manually.
