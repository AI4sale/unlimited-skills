# v0.3.4-alpha Known Issues

`v0.3.4-alpha` is a developer preview release gate for plan, entitlement, and sandbox billing diagnostics. It is not a production SLA.

## Distribution

- GitHub clone is the supported alpha distribution path.
- PyPI remains deferred because router skills, installers, schemas, docs, scripts, and bundled repo assets must be available beside the checkout.

## Hosted Registry

- Public MIT local commands remain usable without registration.
- `plan refresh` requires a registered installation and hosted `/v1/hub/entitlements` support.
- `billing refresh` requires a registered installation and hosted `/v1/hub/billing-status` support.
- Billing lifecycle diagnostics are `sandbox_only`.
- Live payment providers are not enabled.
- Checkout sessions are not created.
- Payment data is not collected.
- Full catalog distribution remains disabled.
- Local Skill Hub remains allowlist-only.

## Signing Status

Production release signing status is `blocked_no_production_signing_key` for this gate. Fixture manifests and local test keys are used only for reproducible test evidence. The release owner must replace this with `production_signed` or an explicit release-owner override before a broad production release.

## Privacy

- Support bundle and diagnostics are local and redacted by default.
- Diagnostics must not include checkout URLs, payment links, invoice URLs, card data, bank data, private skill bodies, private skill names, private pack names by default, raw archive URLs, local paths, hosted tokens, device proofs, or device private keys.
- Users decide whether to share support bundle output with support.
