# v0.6.5-alpha Upgrade Notes

Status: operator-facing upgrade notes for the `v0.6.5-alpha` reliability
package. These notes are inert until the release operator publishes
`unlimited-skills==0.6.5` through PyPI Trusted Publishing.

## Install

After the release is published:

```powershell
python -m pip install --upgrade "unlimited-skills==0.6.5"
unlimited-skills --version
```

Expected version output:

```text
unlimited-skills 0.6.5
```

## What Users Should Notice

- `suggest` and the UserPromptSubmit hook are less likely to deliver zero
  candidates when retrieval has usable candidates.
- `search`, `suggest --card`, and hook delivery now share the same candidate
  family and source metadata.
- Accepted skill usage can boost future exact/similar queries.
- Wrong/rejected skill feedback can demote future exact/similar queries.
- Manual `search -> view -> use` can learn even when `use` does not repeat
  `--query`, using privacy-safe query fingerprints.

## Post-Upgrade Checks

```powershell
unlimited-skills suggest "напиши пост для линкедин" --json --card --limit 5
unlimited-skills search "напиши пост для линкедин" --mode hybrid --json --limit 5
```

Expected: both commands return usable LinkedIn/social/content candidates when
the local skill library contains those skills. The output may include
low-confidence candidates; low confidence is a label, not a zero-candidate
failure.
