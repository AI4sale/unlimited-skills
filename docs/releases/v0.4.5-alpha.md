# v0.4.5-alpha

Status: MCP audit inspector integration gate.

`v0.4.5-alpha` integrates the E11 read-only MCP audit inspector after the
`v0.4.4-alpha` publication tag. It proves that local operators can inspect the
redacted MCP JSONL audit log without enabling hosted gateway mode, OAuth,
remote upstreams, resources, prompts, production hosted calls, automatic
telemetry, or full catalog distribution.

This is an integration gate. Codex must not create or push `v0.4.5-alpha` from
this branch. A later final publication gate must select the `main` SHA and run
the publication verifier before any tag is created.

## Integrated Work

- Public PR #97 published and tagged `v0.4.4-alpha`.
- Public PR #95 added the E11 MCP audit inspector v1.
- `unlimited-skills mcp audit-report` renders summary, refusals, upstream
  health, profiles, and redaction self-check sections.
- `--json` output validates against `schemas/mcp-audit-report.schema.json`.
- The inspector reads the active audit log plus rotated generations in
  chronological order.
- Recent refusals include timestamp, tool, upstream, code, and name only: no
  argument values and no error text.
- Profile evidence includes `profile_loaded` rows and profile SHA-256 values
  without requiring older logs to carry profile fields.
- The redaction self-check reports only suspect file, line, field, and reason.
  Suspect values are never printed.
- Missing audit logs return a clear exit code 1 message instead of a traceback.

## Security Boundary

- No audit log writes from the inspector.
- No production rollout.
- No production hosted calls.
- No hosted gateway.
- No automatic telemetry.
- No OAuth implementation.
- No remote upstream implementation.
- No MCP resources.
- No MCP prompts.
- No shell execution.
- No full catalog distribution.
- No auto-publish.
- No live billing.
- No PyPI publication.
- Codex must not create or push `v0.4.5-alpha` in this integration gate.
