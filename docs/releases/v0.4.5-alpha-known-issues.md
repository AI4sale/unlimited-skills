# v0.4.5-alpha Known Issues

`v0.4.5-alpha` is an alpha integration gate for the MCP audit inspector.

## Intentional Limitations

- The inspector is local and read-only. It does not upload audit logs, forward
  hosted queries, or provide hosted gateway reporting.
- The inspector reads redacted JSONL audit rows. It cannot recover argument
  values or original error text once the audit writer has scrubbed them.
- The redaction self-check uses secret-shape and local-path heuristics. It is a
  leak detector, not a proof that every possible sensitive string class is
  impossible.
- Recent refusals intentionally omit argument values and error text. Use the
  local audit log and local operator context for deeper debugging.
- The JSON schema is locked for v1 reports but remains alpha and may change
  before v0.6.
- No OAuth, remote upstreams, MCP resources, MCP prompts, hosted gateway mode,
  production hosted calls, automatic telemetry, full catalog distribution,
  auto-publish, live billing, or PyPI publication are included.
- Codex may create and push `v0.4.5-alpha` only after the final publication
  verifier passes on the selected `main` SHA.

## Operator Notes

- Missing audit logs return exit code 1 with a clear message. Start the gateway
  first or pass `--audit-log` to inspect a specific file.
- Rotated logs are read oldest first, then the active audit log.
- Profile sections are present only when the log includes profile fields such
  as `profile_loaded`; older logs are still valid.
