# v0.4.4-alpha Known Issues

`v0.4.4-alpha` is an MCP permissioned tool profile integration gate, not a
production SLA.

- Codex creates and pushes the final `v0.4.4-alpha` release tag only after the
  publication verifier passes on the selected `main` SHA.
- Hosted production calls are intentionally disabled in tests.
- The MCP gateway is local stdio only; there is no hosted gateway.
- OAuth and remote upstreams are out of scope.
- MCP resources and MCP prompts are out of scope.
- Profile files are loaded once at gateway startup; changing a file requires a
  gateway restart.
- Profile signing remains shape-only; signature verification is a later gate.
- No live profile reload is implemented.
- MCP v1 schemas/configs/profiles are alpha and may break before v0.6.
- Full catalog distribution, live billing, PyPI publication, automatic
  telemetry, automatic rewriting, and auto-publish remain disabled.
