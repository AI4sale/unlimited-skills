# v0.6.6 product precision plan

Status: implementation and release execution

## Capability

An agent using Unlimited Skills should receive one useful skill when confidence
is sufficient, silence or an explicit recovery action when it is not, and a
first-run path that reaches that behavior without overwriting local content.
A release is complete only when the exact public PyPI wheel has passed the same
critical-path checks as the source tree.

## Audit baseline

The review started from `origin/main` at `159205b` (`v0.6.5.post1-alpha`). The
official PyPI project still exposed `0.6.4.post1`, while GitHub already exposed
0.6.5 and 0.6.5.post1 prereleases. Focused local reproduction found:

| Priority | Defect | User-visible failure | Required invariant |
| --- | --- | --- | --- |
| P0 | One score floor implicitly controlled diagnostics, name hints, and body injection | weak candidates either leaked through plain text or disappeared from the model entirely | raw retrieval, recall-safe hints, and card/body eligibility are separate surfaces |
| P0 | Mixed-language weak overlaps suppressed rescue | a Russian prompt containing `python api` received a weak English skill hint | any non-English result below the floor requests English-keyword rescue |
| P0 | GitHub release state could advance without PyPI | users installed an older package than release docs described | tag/release creation follows verified public-wheel smoke |
| P1 | Card mode imported the full CLI before knowing whether vector rescue was needed | baseline hook probe used about 1.8 seconds of a 2-second budget | English/confident lexical routing stays on the lightweight module graph |
| P1 | Native-language retrieval depended on an operator noticing and starting the daemon | repeated prompts stayed on lexical rescue even though the warm runtime was the designed fast path | the hook keeps one compatible local daemon running by default, without blocking the prompt |
| P1 | Quickstart treated any non-empty library or collection as complete | one local skill or a partial managed collection prevented bundled skills from being imported | missing managed skills are added without touching existing files or `library/local` |
| P1 | Doctor collapsed three states into "dead" | fallback, cold vector search, and warm daemon readiness were misreported | diagnosis names fallback, index, and daemon states separately |
| P2 | Wheel output pointed at relative repository docs | `docs/...` next steps were not usable from an installed wheel | CLI guidance uses executable commands and absolute documentation URLs |

## Implementation contract

### Retrieval precision

- Preserve the raw family as `retrieval_candidates`; keep the legacy
  `top_3_skill_candidates` aligned with recall-safe model hints in card mode.
- Keep plain-text `--floor` strict, but allow lower-confidence NAME hints only
  when exact identity, discriminative name/description evidence, or a strong
  compatible vector hit qualifies them. Body-only overlap never qualifies.
- Require a stricter lexical/card floor and confidence margin before any skill
  body can be injected. A vector-only or mixed-language-uncertain result can be
  hinted, never card-injected.
- For non-English weak matches, deliver safe names plus an English-keyword
  rescue instruction; when no safe name exists, deliver rescue only.
- Fuse lexical and vector ranks with RRF; never treat the fused rank as proof
  that a candidate is safe to hint.
- Calibrate discriminative evidence on the frozen 30-positive/12-negative
  corpus: top-1 0.900, top-3 0.933, false-positive rate 0.000, card precision
  1.000, and cold indexed p95 1.686 s on Windows.

### Fast and consent-safe hooks

- Check a small vector manifest for schema, model, dimensions, inventory
  generation, count, and completeness without importing the heavy CLI.
- Attempt vector rescue only when lexical evidence is not safely decisive and
  a compatible in-budget runtime is available; preserve semantic-only English
  matches while exact/strong lexical queries stay lightweight.
- Always request tier metadata even when card body injection is disabled.
- Keep `unlimited-skills serve` available by default: probe the local health
  identity, launch a hidden detached instance only when the port is absent,
  and use a cross-process cooldown to prevent prompt-hook spawn storms.
- Never replace an occupied/incompatible port or start a remote endpoint.
  `UNLIMITED_SKILLS_NO_AUTOSERVE=1` remains an emergency escape hatch for
  restricted runtimes, and launch failure stays fail-open for the prompt.
- Start proactively at `SessionStart` and retain `UserPromptSubmit` as a
  watchdog. Persist local launch/PID status beside the install root and derive
  the loopback port from root + embedding model for multi-library isolation.

### First value and diagnosis

- Import only missing bundled skills; preserve existing local skills and
  existing managed skill files, and migrate a legacy index in place.
- Apply the production floor to quickstart's proof search.
- Lead post-quickstart actions with the dry-run MCP installer.
- Report native-language fallback, vector-index readiness, and optional daemon
  readiness independently.
- Version lexical token indexes with tokenizer/stopword/expansion revisions and
  a library generation hash; stale or interrupted indexes fail open to a fresh
  filesystem scan.

### Release truth

- Publish package version `0.6.6` through PyPI Trusted Publishing.
- Wait for `https://pypi.org/pypi/unlimited-skills/0.6.6/json`.
- Clean-install `unlimited-skills==0.6.6` from the public index and prove the
  version, bundled quickstart, floor silence, and mixed-language rescue.
- Create `v0.6.6` only after that public-wheel smoke passes.

## Non-goals

- No hosted registry/backend changes.
- No operating-system service installation. Prompt-hook autoserve is limited to
  the existing local `unlimited-skills serve` process and never binds LAN/remote.
- No deletion, replacement, or migration of `library/local`.
- No new paid/hosted tier claims.
- No breaking removal of the existing JSON diagnostic fields.

## Release gates

1. Focused retrieval, hook, quickstart, doctor, workflow, and release tests.
2. Full pytest suite, compile check, and the 42-scenario effectiveness record.
3. Frozen v0.6 contract verifier for `0.6.6`.
4. Build wheel/sdist once, `twine check`, clean-install that exact wheel, and
   prove an in-place upgrade from the official PyPI `0.6.4.post1` package while
   preserving a local sentinel skill and migrating its legacy index.
5. CI green on the exact release commit.
6. PyPI Trusted Publishing succeeds.
7. Public PyPI clean-install verifier succeeds.
8. GitHub release tag targets the same commit.

Rollback is a new patch release; published PyPI files are immutable. If public
verification fails after upload, do not create the GitHub release and either
yank the broken version with an explicit reason or immediately prepare the
smallest corrective version.
