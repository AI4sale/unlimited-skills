# v0.4.8-alpha Known Issues

`v0.4.8-alpha` is an alpha release for the managed MCP profile trust store. The
known boundaries are intentional:

- No hosted trust fetch.
- No registry sync.
- No production signing keys.
- No private key storage.
- No OAuth.
- No MCP resources.
- No MCP prompts.
- No production rollout.
- No PyPI publication.
- No live billing.
- No E16 rollout simulator.
- The release tag is created only after the final publication verifier passes
  on the selected `main` SHA.
- The trust-store metadata sidecar is informational only. It cannot widen
  bundle scope, audience, or profile permissions.
- Missing trust store behavior is explicit: raw local profiles may still work
  when policy allows them, but signed bundle verification fails clearly without
  a trusted key.
- Corrupt trust store files are reported by `trust doctor` and must be fixed
  manually.
