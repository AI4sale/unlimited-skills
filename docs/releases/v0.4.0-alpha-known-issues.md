# v0.4.0-alpha Known Issues

`v0.4.0-alpha` is an alpha SkillOps foundation milestone, not a production SLA.

- Codex does not create or push the final release tag. The release owner must verify the final `main` SHA and run the human tag command.
- External local-registry E2E requires a private registry checkout that contains E02, E03, and E04 validators.
- Hosted production calls are intentionally disabled in tests.
- Production signing keys are not required by this gate.
- Full catalog distribution remains disabled.
- Live billing remains disabled.
- PyPI publication remains disabled.
- Automatic install, update, remove, rewrite, and auto-publish remain disabled.
- Local usage snapshot and recommendation-context preview work must wait until the v0.4.0-alpha publication gate is merged, tagged by the release owner, or explicitly blocked with owner, action, and fallback.
