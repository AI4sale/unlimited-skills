# v0.4.7-alpha Upgrade Notes

`v0.4.7-alpha` is a GitHub-clone alpha release. It publishes the signed MCP
profile bundle verification integration for the local stdio Unlimited Tools
gateway.

## Upgrade

From a local clone:

```powershell
git fetch --tags origin
git checkout v0.4.7-alpha
python -m pip install -e ".[all]"
```

For a working branch that tracks `main`:

```powershell
git fetch origin
git checkout main
git pull --ff-only
python -m pip install -e ".[all]"
```

## What Changes

- `unlimited-skills mcp gateway` can load a signed profile bundle with
  `--profile-bundle`, `--trusted-keys`, and one or more `--audience-id`
  values.
- The bundle is verified once at startup before profile enforcement.
- Fail-closed refusal covers bad signatures, unknown or expired keys, expired
  or not-yet-valid bundles, revoked bundles or keys, wrong audience, and
  namespace-ceiling violations.
- `profile_loaded` audit rows include signed-bundle provenance: bundle SHA-256,
  issuer key id/display, audience, expiry, and verification status.
- Raw local profile files keep the existing local MIT core behavior unless a
  host explicitly enables `--require-signed-profiles`.

## Compatibility

This release does not add hosted trust fetch, registry sync, production signing
keys, private key storage, OAuth, remote upstreams, MCP resources, or MCP
prompts.

MCP v1 schemas/configs/profiles/bundles are alpha and may break before v0.6.
Registered/business signed-required behavior is future-gated unless explicitly
implemented in a later gate.

## Boundaries

- No hosted trust fetch.
- No registry sync.
- No production signing keys.
- No private key storage.
- No OAuth.
- No remote upstreams.
- No MCP resources or prompts.
- No hosted gateway.
- No production hosted calls.
- No automatic telemetry.
- No arbitrary shell execution.
- No full catalog distribution.
- No auto-publish.
- No live billing.
- No PyPI publication.

## Publication

After the final publication verifier passes on the selected `main` SHA:

```powershell
git tag -a v0.4.7-alpha <verified-main-sha> -m "v0.4.7-alpha"
git push origin v0.4.7-alpha
```
