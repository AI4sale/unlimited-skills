# v0.3.6-alpha

Status: final publication gate for registered catalog browser discovery.

`v0.3.6-alpha` adds a signed, metadata-only catalog browser UX for registered installations. Users can browse, search, inspect filters, preview reviewed catalog items, and dry-run install checks without loading skill bodies into agent context.

## Highlights

- Package version is `0.3.6`.
- GitHub clone remains the supported alpha distribution path.
- `catalog browse`, `catalog search`, `catalog filters`, `catalog preview`, and `catalog install --dry-run` are registered hosted operations.
- Catalog browser responses must be signed by trusted manifest keys.
- Results are approved-or-published by default; pending, rejected, and withdrawn items are hidden even if a server returns them.
- Preview is metadata-only and refuses body-including responses.
- Dry-run install verifies signed approved/published metadata and writes no files.
- Community-source non-dry-run installs delegate to the existing Community Skills install flow after signed browser metadata verification.
- Official and private-visible browser items remain metadata/dry-run only until dedicated install-plan capability checks are implemented.
- Fixture tests make no production hosted calls.
- Final publication adds upgrade notes, known issues, release smoke alias, and publication verifier.

## Security Boundary

- MIT local commands remain unregistered.
- No billing, marketplace storefront, checkout, or payment provider is enabled by this release.
- Catalog browser commands require registration, hosted token, and signed device proof.
- Hosted browser metadata calls must not upload skill bodies, prompts, source code, full local paths, repository paths, customer names, environment variable values, tokens, secrets, or device private keys.
- Catalog browser responses must be signed.
- Preview and search responses are metadata-only.
- Support bundle catalog browser diagnostics include only redacted booleans and no queries, item names, skill bodies, or private paths.
- Production hosted calls are blocked in the release smoke.

`--local-registry` requires a local private registry checkout, defaulting to `D:\git\unlimited-skills-registry`. `--fixture-mode` does not require that checkout.
