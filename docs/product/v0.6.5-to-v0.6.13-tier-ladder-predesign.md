# Tier-Ladder Pre-Design: v0.6.5 → v0.6.13

**Status:** forward product pre-design (no code). **Companion to:** `tier-ladder-coverage.md`
(the coverage table + release gate this file fills). **Source of themes:** the private
roadmap `UNLIMITED-SKILLS-PRIVATE-ROADMAP-v0.6.3-to-v0.6.13.md` (private registry) — each
release's **Free core** below is summarized from that roadmap's VFPs; the tier bonuses are
designed on top of that real capability, not invented.

**Purpose.** Satisfy the owner directive *ahead of build* for the whole line: every release
gets a concrete Free / Registered / Team / Business / Enterprise bonus, designed before the
release is implemented. At build time each release's section here expands into the full
per-tier contract set (the v0.6.3/v0.6.4 shape) + a cross-tier matrix + the TIER-06 review
forms. This file is the **design**, not the final contracts.

## The fixed tier ladder (applied to every release)

Each release's Free core is a **local capability**. The four above-Free tiers are bounded,
**local-first** add-ons on top of it, following one repeatable pattern:

- **Registered** — one bounded, **schema-versioned local export/profile** of the feature's
  safe aggregates. Produced locally, stays local. No upload/sync/hosted submit. No machine/install id.
- **Team** — a **local rollup** that aggregates multiple members' Registered exports, gathered
  out of band (no network fetch). Member-local aliases (never OS identity), dedup on repeated
  imports, reject incompatible `schema_version`. No live sync / dashboard / central server.
- **Business** — an **admin CSV + JSON export** with admin-supplied local labels
  (team/project/workspace/agent-class), measured facts kept separate from estimates,
  import-ready for a *future* dashboard. No hosted dashboard, billing, or entitlement.
- **Enterprise** — an **audit-safe local evidence pack**: machine-readable manifest + written
  method/assumptions statement + privacy-proof + schema/version-proof, stable hash for identical
  content. No data egress, no SSO/SCIM, no enforced governance, signature-ready ≠ signature-enforced.

### Cross-tier claim boundary (every release, every tier)

Forbidden anywhere (code, CLI, docs, release notes) unless the code for it actually ships in
that release: *exact money saved*, *guaranteed bill reduction*, *hosted telemetry*, *live
dashboard*, *enterprise governance enforced*, *provider reconciliation*, *automatic
deletion/mutation*, *marketplace ranking/endorsement*. Privacy invariant: aggregates and safe
labels only — never raw prompts, task text, skill bodies, secrets, local/private-repo paths,
server/MCP names, or raw MCP payloads. Free core behaviour is never changed by a tier add-on.

## v0.6.5 — Skill Library Quality Control
**Free core:** local skill quality signal model + duplicate/overlap detector + unused/low-signal
report + quality report (JSON: pass/warn/blocked/duplicate/stale counts, names without bodies).

- **Registered:** schema-versioned **quality-report export** (`quality-report-v1.json`) — the
  counts + affected skill **names** (never bodies) + per-signal severity; local, no upload.
- **Team:** **library-health rollup** across members' quality exports — aggregate duplicate/
  stale/broken/low-signal counts per member-alias; dedup repeated imports.
- **Business:** admin **library-health export** (CSV+JSON) grouped by team/project/pack +
  "where quality debt concentrates" prioritization; import-ready, no hosted.
- **Enterprise:** **quality evidence pack** — manifest + method statement (signal definitions,
  severity, default action) + privacy-proof (no bodies) + schema-proof; no egress.
- **Claim line:** "local skill-library quality signals and reports." Not live: hosted quality
  service, paid quality score, automatic deletion.

## v0.6.6 — Personal Router Tuning
**Free core:** per-user router preference model (domain boosts, ignored skills, preferred packs,
strictness) + config validation + applied to local suggest/search (defaults reversible).

- **Registered:** portable **tuning-profile export** (`tuning-profile-v1.json`) — the user's
  preference set; local, reversible; no cloud profile.
- **Team:** **shared tuning preset** — team curates a recommended preset from members' profiles;
  manual import; defaults preserved; no central policy engine.
- **Business:** admin **tuning-baseline export** (CSV+JSON) + advisory drift report (which agent
  classes deviate from the baseline); advisory only, no enforcement.
- **Enterprise:** **tuning governance evidence pack** — documents the org's recommended router
  posture + method/assumptions; no policy-enforcement claim, no SSO.
- **Claim line:** "local, reversible personal router tuning." Not live: hosted profile, team
  policy engine, enforced tuning.

## v0.6.7 — Auto Skill Draft from Repeated Work
**Free core:** local repeated-task pattern detection (redacted) + draft `SKILL.md` skeleton
(saved to review location, "not installed") + draft privacy guard + review docs.

- **Registered:** **draft-bundle export** (`draft-bundle-v1.json` + skeleton) — redacted,
  privacy-scanned, marked not-installed; local only.
- **Team:** **shared draft candidate pool** — aggregate members' redacted draft candidates;
  dedup overlapping drafts; manual share; no auto-publish.
- **Business:** admin **reusable-work report** (CSV+JSON) — where repeated work concentrates
  across teams/agent classes (ROI signal for skill creation); no auto-install.
- **Enterprise:** **draft-provenance evidence pack** — manifest + privacy-proof (no raw
  prompts/secrets/paths) + method statement; audit-safe; no egress.
- **Claim line:** "local draft skills from repeated work, reviewed before install." Not live:
  auto-install, auto-publish, hosted telemetry.

## v0.6.8 — Task-to-Skill Recommendation
**Free core:** recommendation explanation contract (matched intent/source/confidence/caveat;
exact/semantic/fallback) + recommendation command output + negative fixtures + docs.

- **Registered:** **recommendation-receipt export** (`rec-receipt-v1.json`) — per-session
  matches + explanations; local; no hosted recommender.
- **Team:** **recommendation-coverage rollup** — aggregate hit/miss + false-positive tallies by
  member-alias; dedup; no hosted eval.
- **Business:** admin **recommendation-quality export** (CSV+JSON) grouped by team/project —
  match-rate, false-positive rate, coverage gaps; advisory, no paid quality claim.
- **Enterprise:** **recommendation evidence pack** — manifest + method (match types,
  thresholds, caveats) + privacy-proof; no hosted recommender/training claim.
- **Claim line:** "local, explainable task-to-skill recommendations." Not live: hosted
  recommender, model training, zero-false-positive guarantee.

## v0.6.9 — Skill Pack Quality / Managed Pack Recommendations
**Free core:** pack quality scorecard (coverage/lint/duplication/freshness/safety) + managed-pack
recommendation report (recommended/review/blocked/unknown) + pack conflict detection + docs.

- **Registered:** **pack-scorecard export** (`pack-scorecard-v1.json`) — per-pack scores +
  statuses; local; no marketplace ranking claim.
- **Team:** **pack-adoption rollup** — which packs the team uses + aggregate scorecards +
  cross-member conflicts; manual; no auto-install.
- **Business:** admin **pack-portfolio export** (CSV+JSON) — recommended/review/blocked packs by
  team + conflict hotspots; import-ready; no hosted catalog mutation.
- **Enterprise:** **pack-governance evidence pack** — manifest + method (scoring rules) +
  safety/conflict proof + privacy-proof; no endorsement/marketplace claim.
- **Claim line:** "local skill-pack quality scorecards and recommendations." Not live: hosted
  scoring, paid endorsement, automatic install.

## v0.6.10 — Context Budget Optimizer
**Free core:** context budget model (router / loaded skills / selected skill / MCP meta-tools /
upstream schemas) + before/after report (skill-preload savings + MCP-schema savings separated) +
optimizer recommendations + docs.

- **Registered:** **context-budget export** (`context-budget-v1.json`) — standing vs on-demand
  context with savings separated; local.
- **Team:** **budget rollup** — aggregate context spend/savings across members/agent instances by
  alias; dedup; no hosted analytics.
- **Business:** admin **context-ROI export** (CSV+JSON) grouped by team/agent class — where
  context waste concentrates + savings opportunity; no pricing claim.
- **Enterprise:** **context-budget evidence pack** — manifest + method (categories, units,
  caveats) + privacy-proof; reproducible; no hosted/benchmark promise.
- **Claim line:** "local, reproducible context-budget measurement and optimization." Not live:
  hosted analytics, universal benchmark promise, pricing claim.

## v0.6.11 — Workflow Skill Chains
**Free core:** skill-chain manifest (chain id, steps, inputs, outputs, stop conditions; skills by
stable id) + chain validation (missing/cycles/dup/unsafe-stop fail) + suggestion surface (no
auto-run) + docs.

- **Registered:** **chain export/import bundle** (`chain-bundle-v1.json`) — manifest by stable
  skill ids; local; validates without private bodies; no execution engine.
- **Team:** **shared chain library** — curate validated chains from members; dedup; manual share;
  no hosted orchestration.
- **Business:** admin **chain-usage export** (CSV+JSON) — which chains are used / where they fit
  across teams; advisory; no automation claim.
- **Enterprise:** **chain-governance evidence pack** — manifest + validation proof (no cycles/
  unsafe stops) + method statement; manual-execution-only stated; no hosted runner.
- **Claim line:** "local, reviewable workflow skill chains (manual execution)." Not live:
  autonomous workflow runner, hosted orchestration, paid automation.

## v0.6.12 — Skill Library Cleanup Autopilot
**Free core:** cleanup recommendation policy (keep/review/duplicate/stale/broken/blocked; local/
private never auto-deleted; approval required) + cleanup scan report + safe cleanup-plan export
(no side effects) + docs.

- **Registered:** **cleanup-plan export** (`cleanup-plan-v1.json`) — statuses + evidence +
  proposed action; no side effects; local/private never auto-deleted.
- **Team:** **cleanup rollup** — aggregate cleanup backlog across members; dedup; manual; no
  destructive action.
- **Business:** admin **library-debt export** (CSV+JSON) grouped by team/pack — cleanup backlog +
  projected context/quality gain; advisory; no auto-cleanup.
- **Enterprise:** **cleanup-governance evidence pack** — manifest + safety-policy proof (no
  auto-delete, approval required) + method + privacy-proof; no hosted mutation.
- **Claim line:** "local, non-destructive cleanup recommendations and plans." Not live: automatic
  deletion, hosted mutation, registry publish.

## v0.6.13 — Portable Skill Workspace
**Free core:** portable workspace layout (separates local/private skills, registry/managed packs,
indexes, tuning, feedback, drafts; secrets/machine paths excluded) + export dry-run + import/
restore plan (no overwrite without approval) + docs.

- **Registered:** **portable workspace bundle** (`workspace-bundle-v1`) — included/excluded
  manifest; secrets and machine paths excluded; local; no cloud sync.
- **Team:** **team workspace template** — shareable baseline (registry/managed packs + recommended
  tuning/chains, **no private skills**); manual; no hosted transfer.
- **Business:** admin **workspace-provisioning export** (CSV+JSON) — standardized workspace
  definition for onboarding agents/teams + restore plan; no auto-overwrite, no upload.
- **Enterprise:** **workspace evidence/governance pack** — manifest + restore-safety proof (no
  overwrite / no secret import) + provenance + method. *E19 signed-bundle integration is the one
  optional, roadmap-sanctioned hook here, but remains **parked** — signature-ready ≠ signature
  enforced; no claim unless code ships.*
- **Claim line:** "local, portable skill workspace (export/restore, no private-data egress)." Not
  live: cloud sync, hosted transfer, automatic overwrite, enforced signing.

## Build sequencing (per release, when it reaches build)

1. Pull the release theme (above) and expand its section into full per-tier contracts +
   cross-tier matrix in `docs/product/<version>/`, following the v0.6.3/v0.6.4 shape.
2. Opus reviews the tier contracts pre-build; Codex implements **Free core first**, then each
   tier bonus in **separate** PRs (never one giant PR).
3. Opus reviews each tier PR against the TIER-06 review forms
   (`docs/reviews/templates/o064-money-saved-meter-tier-review-forms.md`, generalized per theme).
4. Flip the release's row in `tier-ladder-coverage.md` to ✅ and only then let the release gate pass.

This keeps the paid-tier value proposition designed at every stage across the entire v0.6.x
line — the explicit owner requirement — with no tier left inert and no claim outrunning the code.
