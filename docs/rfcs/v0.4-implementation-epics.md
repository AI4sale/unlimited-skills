# v0.4 Implementation Epics

Status: approved to start by the v0.4 go/no-go decision package after review and merge.

These epics define the first implementation layer only. They do not authorize production rollout, automatic skill mutation, live billing, PyPI publication, or full catalog distribution.

## V04-E01: Policy-Aware Recommendation Runtime Preview

VFP: users can request recommendation previews that combine signed catalog metadata, quality status, improvement status, team/private-pack entitlement, and enterprise policy without applying changes automatically.

Repositories:

- `AI4sale/unlimited-skills`
- `AI4sale/unlimited-skills-registry`

Required boundaries:

- preview/decision-only output;
- no automatic install, update, remove, rewrite, or publish;
- signed hosted metadata for any hosted decision input;
- MIT local core remains registration-free;
- support diagnostics expose counts and status only.

Review gates:

- runtime review;
- privacy review;
- security review;
- release owner approval.

## V04-E02: Eval-Driven Release Gate Operator Workflow

VFP: registry operators can run signed eval gates, inspect pass/warning/block/override/rollback recommendations, and prepare human release decisions without auto-promotion or rollback.

Repositories:

- `AI4sale/unlimited-skills-registry`

Required boundaries:

- no automatic channel promotion;
- no automatic production rollback;
- overrides require release-owner scope, expiry, risk acceptance, and evidence;
- public summaries remain metadata-only and redacted.

Review gates:

- registry operator review;
- security review;
- release owner approval.

## V04-E03: Maintainer Improvement Queue Runtime Lifecycle

VFP: maintainers can expose redacted queue status and fixed-pending-eval evidence to public clients while keeping all queue mutations human-reviewed.

Repositories:

- `AI4sale/unlimited-skills-registry`
- `AI4sale/unlimited-skills`

Required boundaries:

- no automatic skill rewriting;
- no auto-publish;
- queue transitions require owner/reviewer metadata and evidence;
- public client receives signed summaries, not private queue payloads or skill bodies.

Review gates:

- maintainer review;
- privacy review;
- security review.

## V04-E04: Governance Dashboard And Support Diagnostics Contract

VFP: operators can see signed, redacted governance status, entitlement/policy summaries, queue health, eval gate state, and support diagnostics without uploading prompts, skill bodies, private paths, tokens, or secrets.

Repositories:

- `AI4sale/unlimited-skills`
- `AI4sale/unlimited-skills-registry`

Required boundaries:

- no automatic telemetry;
- no prompt upload;
- no skill body upload;
- no private path, token, proof, or private-key exposure;
- no live billing;
- dashboard actions remain role-gated and reviewed.

Review gates:

- product review;
- privacy review;
- security review;
- enterprise review.

## Release Hygiene

Each epic must ship in reviewable layers. When a layer is release-ready, merge or close completed PRs before requesting the next task batch. Do not accumulate another large open PR stack.
