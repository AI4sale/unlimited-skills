# v0.6.4 Money Saved Truth Audit

Status: emergency post-publish correction for `v0.6.4-alpha`.

Tracking issue: https://github.com/AI4sale/unlimited-skills/issues/218

## Finding

`v0.6.4-alpha` shipped a local Money Saved Meter release package, but the
published package does not compute dollar savings. It reports local aggregate
counts, measured MCP schema bytes when available, and estimated tokens from
context bytes. Dollar value remains disabled, provider/model price binding is
absent, and skill savings are not separated from MCP savings in the Free meter
output.

The accurate product description for `v0.6.4-alpha` is:

> Context bytes saved and estimated tokens avoided meter with local tier
> artifacts.

It is not yet a complete Money Saved Meter because it cannot produce non-null
estimated money saved from a model-bound pricing basis.

## Evidence

The current `money-saved meter --json` contract exposes:

- measured MCP context bytes avoided when `mcp savings` data exists;
- estimated tokens avoided using the `bytes_divided_by_4` fallback;
- `estimated_dollar_value.enabled=false`;
- `estimated_dollar_value.value=null`;
- `disabled_by_default_no_local_price_config`;
- no provider/model/currency/price table binding;
- no separate Free-level `skills_savings` versus `mcp_savings` money sections.

The release smoke gates verified local command availability, privacy boundaries,
tamper behavior, and no exact-money overclaim. They did not verify that Money
Saved computes estimated money.

## Product Gap

The missing acceptance target is:

> A user can run the local Money Saved Meter with an explicit model/pricing
> basis and receive non-null estimated money saved, with skills and MCP savings
> separated and with clear "not provider bill reconciliation" labeling.

## Hotfix Acceptance Criteria

The corrective release must include:

- explicit provider/model/currency/price basis in the Free meter JSON;
- public or locally configured input-token price source metadata;
- separate `skills_savings`, `mcp_savings`, and total sections;
- estimated input money formula:
  `estimated_input_tokens_saved / 1_000_000 * input_price_per_1m_tokens`;
- output money savings set to zero unless a future artifact measures output
  reduction;
- no provider bill reconciliation claim;
- Registered export preserves pricing basis and money fields;
- Team rollup aggregates money only across compatible basis groups;
- Business CSV/JSON includes money columns and pricing source metadata;
- Enterprise evidence pack includes pricing/model/formula proof;
- Enterprise verifier recomputes money and fails closed on model, price,
  formula, or total tampering.

## Release Note Correction

The GitHub release notes for `v0.6.4-alpha` must carry this known issue:

> Known issue: v0.6.4 reports context/token proxy savings only; it does not
> compute dollar savings.

## Non-Goals

This correction does not authorize v0.6.5 feature work, hosted telemetry,
marketplace rollout, provider billing reconciliation, exact token claims, exact
money claims, or bill reduction guarantees.
