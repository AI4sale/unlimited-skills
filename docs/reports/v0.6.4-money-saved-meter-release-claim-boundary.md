# v0.6.4-alpha Money Saved Meter Release Claim Boundary

This boundary controls release notes, README edits, marketplace drafts, and
operator summaries for the v0.6.4-alpha Money Saved Meter. It does not authorize
release execution, version bump, tag creation, PyPI publish, marketplace
submission, or hosted/team/business/enterprise rollout.

## Allowed Claims

- v0.6.4-alpha adds a local Money Saved Meter.
- Users can run local before/after measurement commands.
- Users can run a deterministic 100-call fixture and verify the report.
- Exact counts remain counts.
- Measured bytes remain measured bytes when local source artifacts expose byte
  values.
- Token and dollar values are estimates.
- Tokens and dollars are estimates.
- Dollar estimates are disabled by default unless the user supplies local
  pricing configuration.
- Dollars are disabled by default.
- The 100-call frame is a local reporting cadence, not billing math.
- Reports are local-only and do not upload raw prompts, task text, secrets,
  private repository paths, skill bodies, raw MCP payloads, or customer data.

## Blocked Claims

Do not claim any of the following:

- Blocked: exact token savings;
- Blocked: exact money savings;
- Blocked: guaranteed bill reduction;
- Blocked: provider billing reconciliation;
- Blocked: hosted telemetry-backed savings;
- Blocked: live hosted dashboard;
- Blocked: automatic upload or sync;
- Blocked: persistent meter state writer;
- Blocked: push nudge or ambient daemon reminder;
- Blocked: paid-tier exports;
- Blocked: team, business, or enterprise rollout;
- Blocked: enterprise governance enforcement;
- Blocked: marketplace availability or acceptance;
- Blocked: PyPI availability for v0.6.4-alpha before the release operator publishes and
  verifies it.

## Required Wording Guardrails

Use "estimates" for token and dollar values. Use "measured bytes" only for byte
fields produced from local artifacts. Use "local-only" for privacy posture. Say
"GO_WITH_LIMITS" when summarizing release readiness.

Review-support docs (#188 and #193) and tier-ladder planning (#190) are not live
paid-tier functionality. They may be described as planning or review support
only.
