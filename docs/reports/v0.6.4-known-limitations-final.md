# v0.6.4-alpha Money Saved Meter Final Known Limitations

This final limitations note is the release-readiness companion for
[`v0.6.4-money-saved-meter-known-limitations.md`](v0.6.4-money-saved-meter-known-limitations.md).
It summarizes the limits that must remain visible in release notes and support
responses.

## Measurement Limits

- Counts are exact only when the local source records exact counts.
- Byte values are measured only when local source artifacts expose byte fields.
- Token values are estimates.
- Dollar values are disabled by default and remain estimates when enabled by a
  local pricing configuration.
- The 100-call frame is a reporting cadence, not billing math.
- Partial windows must stay partial; missing data must not be fabricated.

## Feature Limits

v0.6.4-alpha does not include:

- persistent meter state writer;
- push nudge or ambient daemon reminder;
- paid-tier export implementation;
- hosted telemetry;
- hosted dashboard;
- billing-provider reconciliation;
- team, business, or enterprise rollout;
- enterprise governance enforcement;
- marketplace submission;
- release execution by this package alone.

## Privacy Limits

The Money Saved Meter remains local-only. It must not ask users to paste or
upload raw prompts, raw task text, secrets, private repository paths, skill
bodies, customer names, local absolute paths, raw MCP schemas, or raw MCP
payloads.

## Operator Limit

This package recommends `GO_WITH_LIMITS`; it does not perform version bump, tag,
PyPI publish, yanking, marketplace submission, or hosted/team/business/enterprise
work.
