# v0.6.4 Money Saved Meter Known Limitations

**Roadmap items:** US-064-001 and US-064-002
**Status:** the value model and local before/after pull command exist. The
runtime push nudge, state writer, paid-tier exports, release gate, version bump,
tag, publish, and hosted surfaces are still not implemented here.

## Summary

The v0.6.4 Money Saved Meter value model is an honest local estimate model. It
does not promise exact tokens, exact money, provider-bill reduction, or hosted
telemetry-backed analytics.

It does not promise exact tokens. It does not promise exact money.

It does not promise hosted telemetry-backed analytics.

## What Can Be Exact

The model can exactly count local aggregate events when source artifacts exist:

- router call count;
- suggested skill count;
- injected skill card count;
- gateway/MCP call count where an audit summary exists;
- per-window counted calls.

These are event counts only. They are not billing units.

## What Can Be Measured

The model can measure bytes when local artifacts expose byte sizes:

- upstream MCP schema bytes;
- gateway meta-tool schema bytes;
- derived context bytes avoided from those measured values;
- injected skill-card bytes only when a local delivery artifact exposes the card
  size.

MCP schema savings may be measured. Skill-body savings are not measured exactly
unless a future artifact explicitly measures skill-body bytes.

## What Is Estimated

The model estimates:

- tokens avoided, normally from `bytes_divided_by_4`;
- context bytes avoided when derived from body or schema size assumptions;
- dollars only if the user configures a local price input.

Tokens and dollars are estimates. They must always be labeled as estimates.

## What Is Disabled By Default

These capabilities are disabled by default:

- dollar value;
- provider-specific price assumptions;
- hosted telemetry;
- billing-provider integration.

Dollar value remains unavailable unless local user configuration supplies a
price. This task does not define or implement that configuration.

## What Must Never Be Claimed

Forbidden claims:

- "exact tokens saved"
- "exact money saved"
- "bill reduction guaranteed"
- "hosted telemetry-backed savings"
- "all skill-body savings measured exactly"
- "provider billing reconciliation"

Allowed claims remain limited to:

- "Unlimited Skills estimates local context savings from routed skill/tool usage."
- "Bytes may be measured when local artifacts expose sizes."
- "Tokens and dollars are estimates."

## What Must Never Be Output

The meter must not output:

- raw prompts;
- raw task text;
- skill bodies;
- local absolute paths;
- tokens, keys, secrets;
- customer names;
- hosted uploads;
- private repo paths;
- raw MCP tool input/output payloads;
- raw MCP schemas;
- MCP server command lines or env values.

## Per-100-Call Limitation

The per-100-call frame is a local reporting cadence. It is not billing math. A
partial window reports counts-so-far and must not extrapolate to exact tokens,
exact money, or guaranteed bill reduction.

## Before/After Command Limitation

US-064-002 adds `unlimited-skills money-saved meter`, a pull command for
reproducible local before/after install measurement. It reads an optional
`mcp savings --json` artifact and optional gateway audit summary, strips server
names/raw schemas/paths, and emits safe aggregate Markdown or JSON.

It does not mutate local meter state, emit an ambient nudge, submit telemetry,
upload reports, or authorize release/publish activity. The before/after delta is
available only when both local reports contain measured context bytes.

## Runtime Limitation

US-064-002 does not add a state writer, push nudge, or daemon behavior. A later
implementation task must create runtime nudge/state code and repeat the privacy
scan against real serialized output.
