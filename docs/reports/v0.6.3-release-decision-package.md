## Decision Summary

The v0.6.3 Learning Loop may be released as an alpha for local diagnostics,
privacy-safe candidate review, and non-mutating dry-run preview.

Allowed public claim:

```text
Unlimited Skills v0.6.3-alpha adds a local Learning Loop that lets users record
feedback, inspect learning diagnostics, review privacy-safe improvement
candidates, and preview a candidate with a non-mutating dry-run.
```

Blocked public claims:

- "actually improves skills";
- "automatically improves skills";
- "auto-applies fixes";
- "mutates or rewrites skill files";
- "publishes skill updates";
- hosted sync, telemetry, model training, marketplace submission, paid-tier
  rollout, dashboard, billing, or enterprise governance claims.

## Required Release Limits

`GO_WITH_LIMITS` means:

- release notes may claim local inspect/review/dry-run preview;
- release notes must state that closed-loop proof is dry-run only;
- release notes must not claim automatic skill improvement or mutation;
- release execution still requires owner approval, accepted final PR state, PyPI
  Trusted Publishing, clean package install, and post-publish verification;
- no tag/version/publish step is authorized by this package alone.

## Release Execution Reconciliation

The old `release_owner_GO_WITH_LIMITS_acceptance` marker must not be treated as
a vague blocker after C063-REL-02 passes. Stop only for failing gates or for
external destructive/production actions that require separate authorization.

## Open Release Conditions

Before any tag or publish:

1. C063-REL-02 must be reviewed and merged;
2. Opus O063-REL-FINAL-R1 must return PASS;
3. package release operator must rerun the selected release flow from refreshed
   `main`;
4. PyPI publish must use Trusted Publishing only;
5. tag creation must wait until clean install and published-mode verification
   pass.

## Final Recommendation

`GO_WITH_LIMITS`.

The local Learning Loop is releaseable as an alpha inspect/review/dry-run
preview feature. The stronger "actually improves skills" claim remains blocked
because there is no non-dry-run skill mutation path in v0.6.3.
