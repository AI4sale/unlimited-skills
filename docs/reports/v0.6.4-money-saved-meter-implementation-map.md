## Summary

The repository already has several reusable value-reporting surfaces:

- `unlimited-skills roi receipt` is a local, aggregate-only pull report.
- `unlimited-skills mcp savings` is a local, pull-based MCP context savings
  measurement.
- `suggest` writes local event rows and router metrics for every router probe.
- The MCP gateway writes redacted per-meta-tool audit rows and has a read-only
  audit inspector with call counts, duration stats, refusal breakdowns, and
  redaction checks.
- v0.6.3 Learning Loop commands can inspect local `.learning` state and
  feedback-derived candidates without mutating skill files.

The Money Saved Meter / per-call value nudge is not implemented:

- no `money saved` command or module exists;
- no push-based nudge is emitted after 100 gateway or router calls;
- no persisted nudge state, debounce, dismissal, or opt-out exists;
- no code joins latest MCP savings numbers to gateway call counts;
- no dollar or token-price model exists;
- ROI receipt remains a pull report and does not trigger ambient nudges.

## Privacy-Sensitive Fields

The existing code treats these as forbidden or sensitive and v0.6.4 should keep
that boundary:

- raw prompts, tasks, queries, filters, and free-form notes;
- tool inputs, tool outputs, raw MCP schemas, stdout, and stderr;
- skill bodies, except the already sanctioned high-confidence `suggest --card`
  channel;
- raw `.mcp.json`, `.claude.json`, gateway configs, env names, env values,
  commands, args, tokens, proofs, private keys, auth headers, cookies, device
  keys, and checkout/payment data;
- local absolute paths, repo paths, home paths, temp paths, and UNC paths;
- raw session ids or tracking identifiers.

Fields that are locally useful but sensitive enough to avoid in a nudge:

- MCP server names from `mcp savings` snapshots;
- gateway upstream names and profile names from audit rows;
- skill names for private/local skills;
- exact top scores, exact task hashes, and exact per-session correlation ids.

Safe or reusable aggregate fields:

- total call counts and threshold counters;
- per-tool counts for the gateway meta-tools when names are limited to
  `tools_search`, `tools_schema`, and `tools_call`;
- measured/skipped server counts without server names;
- total bytes/tokens, gateway bytes/tokens, savings bytes/tokens, savings
  percent, and token heuristic;
- coarse buckets for score, margin, confidence, and status;
- booleans for local-only, upload=false, telemetry=false, schemas included=false,
  server names included=false, paths included=false.

## Minimal Data Model for a Per-100-Call Nudge

The smallest useful v0.6.4 model can be local-only and derived from existing
files:

```json

  "schema_version": 1,
  "report_type": "money_saved_meter",
  "generated_at": "2026-06-15T00:00:00Z",
  "source": {

    "gateway_calls": "mcp_audit_report",
    "router_calls": "router_metrics"
  },
  "window": {
    "threshold_calls": 100,
    "calls_since_last_nudge": 100,
    "counted_call_kind": "gateway_meta_tool_call"
  },
  "savings": {
    "source": "local_mcp_savings",
    "measured_servers": 0,
    "skipped_servers": 0,
    "total_bytes": 0,
    "total_est_tokens": 0,
    "gateway_bytes": 1268,

    "available": false,

    "currency": null,

  },

    "eligible": true,

    "local_only": true,

    "tool_outputs_included": false,
    "mcp_schemas_included": false,
    "server_names_included": false,
    "upstream_names_included": false,
    "skill_bodies_included": false,
    "local_paths_included": false,
    "tokens_included": false,
    "keys_included": false
  }

```

Open design choices before implementation:

- Count gateway meta-tool calls, router `suggest` calls, or both. Gateway calls
  best match MCP savings; router calls best match skill adoption.
- Avoid dollar claims by default. A money amount should require an explicit,
  local user-configured input token price and must remain labeled as an
  estimate.
- Persist nudge state locally, for example in
  `<root>/.learning/money-saved-meter.json`, with threshold, last counted audit
  row cursor, last nudge timestamp, dismissed state, and privacy flags.
- Avoid storing full audit file paths in the meter. Use basenames, line counts,
  timestamps, or total counts.
- Decide whether the nudge is emitted by the CLI, the gateway process, or the
  Claude Code hook. The hook is already a push surface, but it is prompt-driven;
  the gateway has better call data but currently communicates only via MCP
  tool responses/audit rows.

## Tests Needed for v0.6.4

Minimum focused test coverage:

1. Meter builder with no local MCP savings snapshot returns `money.available`
   false and does not claim local savings.
2. Meter builder with a local `mcp_savings` event strips server names, commands,
   schemas, env, and local paths.
3. Meter builder reads gateway audit report totals and emits no nudge at 99
   counted calls.
4. Meter builder emits one eligible nudge at 100 counted calls.
5. Nudge state is idempotent: the same 100 calls do not produce repeated nudges.
6. Dismissal/disable state suppresses push output but still allows a pull report.
7. Rotated audit logs and malformed audit lines are tolerated through the
   existing audit inspector path.
8. Minimal and standard audit levels both work; argument shapes are never needed
   for the meter.
9. Router metrics can be included as a secondary adoption counter without query
   text, path, or raw session id leakage.
10. JSON stdout remains valid and existing `roi receipt`, `mcp savings`, and
    `mcp audit-report` contracts do not drift.
11. Hook-delivered nudge, if used, does not echo prompt text and respects
    `UNLIMITED_SKILLS_NO_INJECT` or a new explicit meter opt-out.
12. Forbidden-needle privacy scan covers Windows paths, POSIX paths, tokens,
    auth headers, private keys, raw tool text, and raw schema fragments.

## Grounded Next Tasks

1. Add a local `money_saved_meter` module that consumes sanitized ROI receipt
   savings, MCP audit-report call counts, and router metrics without reading
   raw prompts, schemas, tool results, or full paths.
2. Add a pull command first, for example `unlimited-skills money-saved meter
   --json`, so the JSON contract and privacy scan can stabilize before push
   injection.
3. Add local nudge state and threshold logic for per-100 counted gateway calls.
4. Add a push delivery path only after the pull report is covered. Prefer a
   short context-saving nudge over a dollar claim unless the user has configured
   a local token-price model.
5. Extend ROI receipt only with optional fields after the Money Saved Meter
   schema is stable, preserving existing v0.6 JSON fields.
